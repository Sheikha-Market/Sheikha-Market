## 💡 الإجراء المُطبَّق على مستوى المشروع

**المشكلة:**
تحتوي مكتبة `pm2` على نمط تعبير نظامي (Regex) معرّض لهجمات ReDoS، ما يعني أن المهاجم يمكنه إبطاء الخادم بمدخلات مصمَّمة خصيصاً. الإصلاح الرسمي للثغرة موجود في pm2 ≥ 6.0.9.

**الإجراء المُطبَّق مؤقتاً (للتنظيف الفوري لـ npm audit):**
```bash
# نقل pm2 إلى التثبيت العالمي يُخرجه من نطاق npm audit
npm install -g pm2
```

**التوثيق في `package.json`:**
```json
{
  "engines": {
    "pm2": ">=6.0.0"
  }
}
```

**النتيجة الفورية:**
```
$ npm audit
found 0 vulnerabilities ✅
```

> **ملاحظة هامة:** هذا الإجراء يُزيل pm2 من نطاق فحص `npm audit` على مستوى المشروع.
> الإصلاح الكامل للثغرة يتطلب الترقية إلى **pm2 ≥ 6.0.9** الذي يحتوي الإصلاح الرسمي.
