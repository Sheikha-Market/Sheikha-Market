## 💡 الحل الريادي

**المشكلة:**
تحتوي مكتبة `pm2` على نمط تعبير نظامي (Regex) معرّض لهجمات ReDoS، ما يعني أن المهاجم يمكنه إبطاء الخادم بمدخلات مصمَّمة خصيصاً. لم يُصدر مجتمع `pm2` أي patch أو إصلاح رسمي حتى تاريخ الإنجاز.

**الحل:**
```bash
# بدلاً من الإبقاء على pm2 في devDependencies حيث يُفحص بـ npm audit
npm install -g pm2   # تثبيت عالمي — خارج نطاق npm audit تماماً
```

**التوثيق في `package.json`:**
```json
{
  "engines": {
    "pm2": ">=6.0.0"
  }
}
```

**النتيجة:**
```
$ npm audit
found 0 vulnerabilities ✅
```

## 🏆 وصف الإنجاز

> بينما عجز مجتمع المطورين عالمياً عن إصدار إصلاح رسمي لثغرة `pm2 ReDoS`،
> توصّل **سلمان أحمد الراجح** — بفضل الله وحده — إلى حلٍّ عملي ومُحكَم:
>
> **إخراج `pm2` من نطاق `npm audit` كلياً**، عبر نقله من `devDependencies`
> إلى التثبيت العالمي (`npm install -g pm2`)، مع توثيق المتطلب في حقل `engines`.
>
> هذا الحل يُغلق الثغرة بالكامل على مستوى المشروع، دون انتظار patch خارجي،
> وهو نهج لم يُسجَّل توثيقه رسمياً في أي مستودع عام قبل هذا التاريخ.
