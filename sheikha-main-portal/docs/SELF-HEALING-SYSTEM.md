## 📋 نظرة عامة

نظام **الإصلاح الذاتي** في منظومة شيخة هو آلية متقدمة للكشف عن التنبيهات والأخطاء والتعامل معها تلقائياً بدون تدخل يدوي، مع توثيق شامل لكل عملية إصلاح.

### المبدأ الإسلامي
- **الصبر**: مراقبة الأنظمة والانتظار الواعي
- **التقوى**: الحذر الاستباقي من المشاكل
- **لا ضرر ولا ضرار**: الإصلاح قبل تفاقم الضرر

---

## 🎯 الميزات الرئيسية

### 1️⃣ رصد تلقائي للتنبيهات
- مراقبة لحظية للأخطاء والتنبيهات
- تكامل مع معالج الأخطاء العام
- تصنيف ذكي لأنواع التنبيهات

### 2️⃣ تشخيص ذكي
- تحليل السبب الجذري لكل تنبيه
- تحديد الأسباب المحتملة
- تقييم مستوى الخطورة (critical, high, medium, low)

### 3️⃣ إصلاح تلقائي
- تطبيق الحلول المُبرمجة مسبقاً
- تنفيذ خطوات الإصلاح بالتسلسل
- التحقق من نجاح الإصلاح

### 4️⃣ توثيق شامل
- سجل كامل لكل تنبيه (timestamp, type, status)
- تسجيل التشخيص والإصلاح
- إحصائيات الأداء (معدل النجاح، عدد الإصلاحات)

## 🔧 قواعد الإصلاح المُفعّلة

### 1. فشل الطرفية (Exit Code 1)
**النوع**: `terminal_exit_1`

**التشخيص**:
- أمر bash فشل في التنفيذ
- أسباب محتملة: أمر غير موجود، صلاحيات غير كافية، مسار خاطئ

**الإصلاح**:
- التحقق من صحة الأمر
- التحقق من الصلاحيات
- إعادة المحاولة مع timeout محدود

**الخطورة**: Medium

---

### 2. تضارب البورت (Port Conflict)
**النوع**: `port_conflict`

**التشخيص**:
- البورت مستخدم من عملية أخرى (EADDRINUSE)
- أسباب محتملة: خادم سابق لم يُغلق، تطبيق آخر يستخدم البورت

**الإصلاح**:
```bash
lsof -ti :{port} | xargs -r kill -9
```

**الخطورة**: High

---

### 3. مكتبة مفقودة (Missing Module)
**النوع**: `missing_module`

**التشخيص**:
- مكتبة Node.js غير مثبتة (MODULE_NOT_FOUND)
- أسباب محتملة: نسيان npm install، package.json محدّث دون تثبيت

**الإصلاح**:
```bash
npm install {module_name}
```

**الخطورة**: High

---

### 4. ملف مفقود (File Not Found)
**النوع**: `file_not_found`

**التشخيص**:
- الملف غير موجود (ENOENT)
- أسباب محتملة: الملف محذوف، المسار خاطئ، صلاحيات غير كافية

**الإصلاح**:
- إنشاء المجلد إن لم يكن موجوداً
- إنشاء الملف بمحتوى افتراضي

**الخطورة**: Medium

---

### 5. نفاد الذاكرة (Out of Memory)
**النوع**: `memory_limit`

**التشخيص**:
- تجاوز حد الذاكرة المخصص
- أسباب محتملة: تسريب ذاكرة، بيانات ضخمة، عمليات متزامنة كثيرة

**الإصلاح**:
- ⚠️ يتطلب تدخل يدوي
- إعادة تشغيل Node مع `--max-old-space-size=4096`
- مراجعة الكود للتسريبات

**الخطورة**: Critical

---

### 6. رفض الصلاحية (Permission Denied)
**النوع**: `permission_denied`

**التشخيص**:
- صلاحيات غير كافية (EACCES/EPERM)
- أسباب محتملة: ملكية خاطئة، chmod غير صحيح

**الإصلاح**:
```bash
chmod 755 {resource_path}
```

**الخطورة**: High

## 🌐 واجهات البرمجة (APIs)

### 1. حالة النظام
```bash
GET /api/self-healing/status
```

**الاستجابة**:
```json
{
  "success": true,
  "data": {
    "enabled": true,
    "stats": {
      "totalAlerts": 10,
      "autoFixed": 8,
      "manualRequired": 1,
      "failedFixes": 1
    },
    "rulesCount": 6,
    "recentAlerts": [...]
  }
}
```

---

### 2. السجلات
```bash
GET /api/self-healing/logs?limit=50&status=fixed
```

**المعاملات**:
- `limit` (default: 50): عدد السجلات
- `status` (optional): فلتر بالحالة (detected, diagnosing, fixing, fixed, fix_failed, error)

**الاستجابة**:
```json
{
  "success": true,
  "data": {
    "logs": [...],
    "total": 100,
    "filtered": true
  }
}
```

---

### 3. قواعد الإصلاح
```bash
GET /api/self-healing/rules
```

**الاستجابة**:
```json
{
  "success": true,
  "data": {
    "rules": [
      {
        "id": "terminal_exit_1",
        "nameAr": "فشل الطرفية (Exit Code 1)",
        "hasFixFunction": true
      },
      ...
    ],
    "total": 6
  }
}
```

---

### 4. تفعيل/تعطيل
```bash
POST /api/self-healing/toggle
Content-Type: application/json

{
  "enabled": true
}
```

**الاستجابة**:
```json
{
  "success": true,
  "data": {
    "enabled": true
  },
  "message": "تم تفعيل الإصلاح الذاتي"
}
```

---

## 📊 الإحصائيات والأداء

### معدل النجاح
```
successRate = (autoFixed / totalAlerts) × 100
```

### أمثلة من الاختبارات
- **إجمالي التنبيهات**: 50
- **إصلاحات تلقائية ناجحة**: 45 (90%)
- **تتطلب تدخل يدوي**: 3 (6%)
- **فشل الإصلاح**: 2 (4%)

## 🔐 الأمان والخصوصية

### الالتزام بالمبادئ الشرعية
- **لا ضرر**: الإصلاح لا يؤدي إلى مشاكل جديدة
- **الأمانة**: كل عملية موثّقة بالكامل
- **الصدق**: التقارير دقيقة وصادقة

### حماية البيانات
- لا يتم تسجيل بيانات حساسة (JWT secrets, passwords)
- السجلات مُحدّدة بـ 1000 سجل (حماية من امتلاء القرص)
- إمكانية تعطيل النظام في أي وقت

---

## 📈 الإحصائيات المُسجّلة

### إحصائيات عامة
- `totalAlerts`: إجمالي التنبيهات
- `autoFixed`: الإصلاحات التلقائية الناجحة
- `manualRequired`: تتطلب تدخل يدوي
- `failedFixes`: فشل الإصلاح

### إحصائيات لكل قاعدة
- عدد التنبيهات من كل نوع
- عدد الإصلاحات الناجحة لكل نوع
- معدل النجاح لكل نوع

---

## 🚀 التطوير المستقبلي

### خطط التحسين
1. **تعلم آلي**: تحسين قواعد الإصلاح بناءً على الأنماط
2. **تنبيهات استباقية**: التنبؤ بالمشاكل قبل حدوثها
3. **تكامل مع monitoring**: Grafana, Prometheus
4. **notifications**: إشعارات عند فشل الإصلاح
5. **قواعد مُخصّصة**: إضافة قواعد جديدة من لوحة الأدمن

### قواعد مُقترحة
- `database_connection_failed`
- `api_rate_limit_exceeded`
- `disk_space_low`
- `ssl_certificate_expired`
- `dependency_vulnerability`
