# سوق شيخه للأسواق كلها — تسجيل التاجر وعرض البضائع (أفضل التقنيات وأجددها)

## 1) الهدف

تفعيل **سوق شيخه للأسواق كلها** — نظام يشمل **كل الأسواق**، مع إمكانية **تسجيل أي تاجر** و**عرض بضائع** داخل السوق، باستخدام **أفضل التقنيات وأجددها** المتاحة — الأفضل بالكون.

---

## 2) المكونات

| المكون | الوصف |
|--------|--------|
| **سوق شيخه للأسواق كلها** | نظام موحد يشمل كل الأسواق — عروض بضائع من متاجر نموذجية وتجار مسجّلين. |
| **تسجيل التاجر** | أي تاجر يمكنه التسجيل (الاسم، الجوال، البريد، المنطقة، المؤسسة) عبر نموذج ويب وواجهة API. |
| **عرض البضائع** | أي تاجر مسجّل (أو برقم جوال) يمكنه إضافة عرض بضاعة (الاسم، الكمية، الوحدة، السعر، العملة). |

---

## 3) التقنيات المستخدمة (أفضل وأجدد متاحة)

| الطبقة | التقنية | الوصف |
|--------|----------|--------|
| **الخادم** | Node.js + Express | خادم HTTP حديث، دعم CORS، JSON، REST API. |
| **التخزين** | JSON + ملفات (data/traders.json, data/listings.json) | تخزين بسيط وقابل للاستبدال بقاعدة بيانات (PostgreSQL، MongoDB، إلخ) لاحقاً. |
| **الهوية** | UUID (uuid v4) | معرف فريد لكل تاجر وعرض. |
| **الواجهة الأمامية** | HTML5 + CSS3 + JavaScript (Vanilla) | نماذج تفاعلية، Fetch API، لا اعتماد على إطارات ثقيلة. |
| **التصميم** | Tajawal، RTL، متجاوب | خط عربي حديث، دعم كامل للعربية، يعمل على الجوال والحاسب. |
| **التطبيق التقدمي** | PWA + Service Worker | التخزين المؤقت، إمكانية العمل دون اتصال جزئي. |
| **الواجهات** | REST API (JSON) | POST/GET قياسي، سهل التكامل مع أي عميل أو تطبيق. |

---

## 4) واجهات API

| الطريقة | المسار | الوصف |
|---------|--------|--------|
| **POST** | `/api/traders/register` | تسجيل تاجر جديد. الجسم: `{ name, phone, email?, region?, company? }`. |
| **GET** | `/api/traders` | قائمة كل التجار المسجّلين. |
| **POST** | `/api/listings` | إضافة عرض بضائع. الجسم: `{ traderId? أو traderPhone?, name, quantity?, unit?, price?, currency? }`. |
| **GET** | `/api/stores` | قائمة المتاجر (نموذج + تجار لديهم عروض). |
| **GET** | `/api/market-listings` | كل عروض البضائع (نموذج + مسجّل). |

---

## 5) الصفحات

| الصفحة | الوصف |
|--------|--------|
| **تسجيل-تاجر.html** | نموذج تسجيل تاجر (الاسم، الجوال، البريد، المنطقة، المؤسسة). |
| **عرض-بضائع.html** | نموذج إضافة عرض بضائع (اختيار تاجر أو إدخال جوال، اسم البضاعة، الكمية، الوحدة، السعر، العملة). |
| **index.html** | قسم «سوق شيخه للأسواق كلها» مع روابط تسجيل تاجر وعرض بضائع، وقسم نظام المتاجر وعرض البضائع. |

---

## 6) حالة الأنظمة

في `GET /api/systems-status`:

- **globalMarkets:** سوق شيخه للأسواق كلها — نظام يشمل كل الأسواق.
- **traderRegistration:** تسجيل التاجر وعرض البضائع — أي تاجر يمكنه التسجيل وعرض بضائعه.

---

## 7) التطوير المستقبلي (اختياري)

- استبدال تخزين JSON بقاعدة بيانات (PostgreSQL، MongoDB) للقدرة والبحث.
- إضافة مصادقة (JWT أو جلسات) لحسابات التجار.
- إضافة تحقق من البريد أو الجوال (OTP).
- إضافة لوحة تحكم للتاجر (عرض وتعديل عروضه).
- إضافة بحث وفلترة في عروض البضائع.
