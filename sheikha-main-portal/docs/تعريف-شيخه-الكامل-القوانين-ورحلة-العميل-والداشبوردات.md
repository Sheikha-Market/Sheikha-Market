# تعريف موقع شيخه الكامل — الخواص، القوانين، رحلة العميل، والداشبوردات

وثيقة مرجعية شاملة لتعريف المنصة، قوانينها، رحلة العميل، تجربة المستخدم، آلية العمل، التسجيل، الواجهات، لوحات التحكم، والتقارير.

## الجزء الأول: التعريفات والخواص

### 1. تعريف موقع شيخه

| البند | الوصف |
|-------|--------|
| **الاسم** | شيخه / Sheikha |
| **النوع** | منصة سوق متخصصة للمعادن والسكراب — B2B و B2C |
| **الشعار** | «أفضل تقنية في بيع السكراب بالكون» — «الذكاء • الجودة • الحلال» |
| **الهدف** | حل متكامل لإعادة تدوير المعادن من المادة الخام إلى المنتج النهائي؛ ربط كل مصدر سكراب بكل مشترٍ عبر سلسلة واحدة شفافة وحلال. |
| **المستخدم المستهدف** | تاجر سكراب، مصنع، جامع، ناقل، مصهر، تاجر معادن، شركات نقل، مستهلك نهائي. |
| **الموضع الرسمي** | **سوق شيخه أفضل من أي سوق إلكتروني في الكون والدنيا كلها، وأفضل من المتاجر كلها** — في نطاق المعادن والسكراب وإعادة التدوير (انظر: سوق-شيخه-أفضل-سوق-في-الكون.md). |

### 2. الخواص الأساسية للموقع

| الخاصية | الوصف |
|---------|--------|
| **سوق متخصص 100%** | سكراب ومعادن فقط؛ سلسلة توريد كاملة: مصدر → تجميع → نقل → مصهر → تصنيع → طرف نهائي. |
| **B2B + B2C** | نفس المنصة: تاجر يبيع لمصنع (B2B) أو لطرف نهائي (B2C)； التمييز بمرحلة التوريد ونوع العرض. |
| **متاجر حسب التخصص** | كل بائع = متجر بتخصص (مصدر، جامع، ناقل، مصهر، معادن، شاحنات)； عرض السوق حسب المرحلة ونوع السكراب. |
| **أسعار مرجعية** | أسعار عالمية وسعودية مرجعية (مع تنبيه أنها ليست أسعار تداول حية)؛ جاهز لربط مصادر حية لاحقاً. |
| **FBS — التخزين والشحن بواسطة شيخه** | إيداع سكراب (حاويات، أوزان، مواقع)； عند البيع، شيخه أو الشريك يتولى التجهيز والنقل. |
| **Sheikha Partner API** | ربط متجرك أو نظامك (ERP، Shopify، إلخ) بسوق شيخه — مزامنة عروض، طلبات، أسعار. |
| **ذكاء مخصص للسكراب** | اقتراح أسعار، توصيات «من قد يشتري منك»، لوجستيات (أوزان، مسافات، شاحنات). |
| **متوافق مع الشريعة** | تجارة حلال، محتوى حلال (لا صور ذوات أرواح)، مدفوعات وعقود واضحة. |
| **الهوية المتكاملة والعلامة التجارية** | العلامة التجارية **الأفضل بالكون والأعلى قيمة** — قياس قيمتها بكل المعايير. في الموقع والتطبيق وفي كل مكان: الشاحنات، أماكن تجميع السكراب، الحاويات، إدارة مشاريع المقاولات، الطرق، المصاهر، والمنتجات النهائية المعاد صهر السكراب واستخدامها (هوية-شيخه-المتكاملة-والعلامة-التجارية). |
| **منظومة الشهادات البيئية والقياس** | منظومة تعمل بالخلفية — شهادات نظام للبيئة، قياس حفظ البيئة وإبعاد الضرر (بالنسبة والكمية)، أفضل من الأنظمة الخضراء (منظومة-الشهادات-البيئية-والقياس-والمحافظة-على-البيئة). |

### 3. القيم والمعايير

| القيمة | التطبيق في الموقع |
|--------|---------------------|
| **الشفافية** | تتبع من المصدر إلى المنتج؛ أسعار معلنة؛ مصدر الأسعار (مرجعي/حي) واضح. |
| **الحلال** | لا صور محرمة؛ زخارف هندسية ونخيل ونجوم فقط؛ تجارة ومدفوعات بضوابط الشريعة. |
| **الجودة** | تصنيف حسب HS Code؛ إثبات ملكية؛ معايير واضحة للمنتجات. |
| **الوحدة** | واجهة واحدة للمستخدم؛ مسار واضح: تسجيل → عرض → بيع؛ تواصل واحد (واتساب). |

### 4. أهداف المنظومة الأخلاقية والقيم — الجامع وما يعمل في الخلفية

- **الأهداف الأخلاقية والقيمية** من أهداف المنظومة الرسمية.
- **الجامع** (ما نُظهِره): **(١) بلا ضرر ولا ضرار** — **(٢) حب العلامة وأمان** — **(٣) المحافظة على البيئة والصحة (وقاية) والبشر والمخلوقات.**
- **الباقي** (الشفافية، الحلال، الجودة، الوحدة، قياس الأثر، التكامل، التأهيل، الخصوصية، إلخ) **يعمل في الخلفية** — مُطبَّق في القرارات والتصميم والتشغيل دون أن يكون كله في واجهة العرض (انظر: أهداف-المنظومة-الأخلاقية-والقيم-والجامع.md).

### 1. القوانين الإدارية

| الرقم | القانون | التفاصيل |
|-------|----------|----------|
| أ-1 | **وضوح الرسالة** | الصفحة الرئيسية تعرض رسالة واحدة وهدفاً واحداً: أفضل تقنية في بيع السكراب وحل إعادة التدوير من المادة إلى المنتج. |
| أ-2 | **مسار واحد للبيع** | ثلاث خطوات فقط: (1) سجّل وتعرّف على السوق، (2) اعرض بضاعتك، (3) اتّفق وبيع. لا مسارات متوازية مربكة. آلية وشكل تسجيل المتجر وعرض البضائع — الأفضل بالكون (انظر: آلية-تسجيل-المتجر-وعرض-البضائع-الأفضل-بالكون.md). |

## الجزء الخامس: كيف يعمل الموقع (آلية العمل)

### 1. من الناحية الوظيفية

1. **الزائر** يفتح الموقع → يرى محتوى تسويقياً وواضحاً (لا تفاصيل تقنية).
2. **التسجيل**: يختار «تاجر» أو «شركة نقل» أو «مشتري» → يملأ النموذج → يُنشأ حساب وداشبورد.
3. **التاجر** يضيف منتجات من داشبورده → تظهر في «السوق» وعروض المتاجر حسب التخصص والمرحلة.
4. **المشتري** يبحث في السوق أو المتاجر → يختار عرضاً → يتواصل (واتساب أو رسالة داخل المنصة) → الصفقة تتم خارج المنصة أو (لاحقاً) عبر آلية دفع وحماية.
5. **شركة النقل** تسجّل → تظهر في خيارات النقل؛ عند طلب شحن (FBS أو طلب مستخدم) تُعرض للاختيار وتستقبل الطلب.
6. **الأدمن** يراقب ويدير المستخدمين والعروض وشركات النقل ويستخرج التقارير.

### 2. من الناحية التقنية (مبسطة)

- **الواجهة (Frontend):** صفحات HTML/CSS/JS؛ متجاوبة؛ ألوان وخطوط الهوية.
- **الخادم (Backend):** Express؛ APIs للتسجيل، العروض، المتاجر، الأسعار، النموذج المبتكر (FBS، Partner API، AI).
- **البيانات:** JSON (traders، listings، إلخ) أو قاعدة بيانات لاحقاً؛ لا عرض تفاصيل داخلية للمستخدم العادي.
- **التسجيل والدخول:** جوال/بريد + كلمة مرور؛ جلسة أو توكن؛ صلاحيات (user / admin / transporter).

| **المستخدمون** | جدول: معرف، اسم، نوع (تاجر/مشتري/نقل/أدمن)، تاريخ التسجيل، حالة (نشط/معطّل). بحث، تصفية، تفعيل/تعطيل. |
| **المتاجر والتجار** | قائمة المتاجر مع التخصص والمنطقة وعدد المنتجات؛ مراجعة طلبات «تسجيل كتاجر» إن وُجدت. |
| **العروض والمنتجات** | كل العروض في السوق؛ فلترة حسب التاجر، التصنيف، التاريخ؛ إخفاء أو حذف مخالف. |
| **شركات النقل** | قائمة شركات النقل؛ الموافقة على التسجيل؛ تفعيل/تعطيل؛ ربط مع FBS. |
| **التقارير** | تقارير جاهزة: مبيعات، نشاط، سلسلة توريد؛ فترات زمنية؛ تصدير PDF/Excel. |
| **الإعدادات** | إعدادات عامة للمنصة، مصدر الأسعار المرجعية، رسائل النظام، الصلاحيات. |

### 2. التقارير في داشبورد الأدمن

| التقرير | الوصف | الاستخدام |
|---------|--------|------------|
| تقرير المستخدمين | عدد المسجلين حسب النوع والفترة | نمو، تحليل. |
| تقرير المتاجر والعروض | عدد المتاجر والعروض حسب التخصص والمرحلة | حالة السوق. |
| تقرير النشاط | تسجيلات، إضافات منتجات، طلبات (إن وُجدت) | نشاط يومي/أسبوعي/شهري. |
| تقرير شركات النقل | عددها، طلبات النقل، حالة التنفيذ | لوجستيات. |
| تقرير مبيعات (لاحقاً) | إجمالي صفقات وقيم (إن وُجدت آلية دفع) | إيرادات وعمولات. |

## الجزء التاسع: داشبورد المستخدم — تفصيل

### 1. الهيكل المقترح

| القسم | المحتويات |
|-------|------------|
| **متجري** | اسم المتجر، التخصص، المنطقة، عدد المنتجات؛ زر «تعديل بيانات المتجر». |
| **منتجاتي** | جدول أو بطاقات: اسم المنتج، تصنيف HS، كمية، سعر، حالة (نشط/مخفي)； أزرار تعديل، حذف؛ زر «إضافة منتج». |
| **إضافة منتج** | نموذج: الاسم، رمز HS، الوصف، الكمية، الوحدة، الوزن، السعر، العملة، الموقع (عنوان + خريطة)، إثبات ملكية (رابط/مرفق). |
| **طلباتي** | الطلبات الواردة على عروضي (معرف الطالب، المنتج، التاريخ، الحالة)؛ والطلبات الصادرة (ما طلبته أنا). |
| **التقارير** | تقاريري: عدد المشاهدات، عدد الطلبات، ملخص مبيعات (إن وُجد)； تصدير بسيط. |
| **الإعدادات** | تعديل الاسم، الجوال، البريد، كلمة المرور؛ تفعيل التسويق (إبراز متجر، ظهور في الصدارة). |

### 2. التقارير في داشبورد المستخدم

| التقرير | الوصف |
|---------|--------|
| ملخص متجري | عدد المنتجات، عدد المشاهدات (إن تُسجّل)، عدد الطلبات. |
| تقرير مبيعاتي | صفقاتي في فترة (إن وُجدت آلية تتبع صفقات). |
| تصدير منتجاتي | قائمة منتجاتي بتنسيق Excel أو PDF. |

## الجزء العاشر: التقارير — ملخص شامل

### 1. للأدمن

- تقارير المستخدمين والمتاجر وشركات النقل.
- تقارير النشاط والعروض.
- تقارير مبيعات وإيرادات (عند تفعيل الدفع والعمولات).
- تصدير البيانات (PDF، Excel) لفترات محددة.

### 2. للمستخدم (تاجر / شركة نقل)

- ملخص متجري وملخص طلبات النقل.
- تقارير مبسطة للمبيعات والمشاهدات.
- تصدير قائمة المنتجات أو الطلبات.

### 3. معايير التقارير

- **اللغة:** عربي مع إمكانية إنجليزي.
- **التواريخ:** دعم نطاق تاريخ (من – إلى).
- **التصدير:** PDF و/أو Excel حسب النوع.
- **الخصوصية:** المستخدم يرى بياناته فقط؛ الأدمن يرى الإجماليات والتفاصيل حسب الصلاحية.

## الخلاصة

هذه الوثيقة تجمع:

1. **التعريفات والخواص** — ما هو موقع شيخه وقيمته وقوانينه.
2. **القوانين الكاملة** — إدارية، هندسية، تجارة.
3. **رحلة العميل** — من الزائر إلى العميل الدائم.
4. **تجربة العميل** — لكل شخصية (زائر، تاجر، شركة نقل، أدمن).
5. **كيف يعمل الموقع** — وظيفياً وتقنياً.
6. **تسجيل المستخدم وتسجيل شركات النقل** — النماذج والصلاحيات.
7. **كيف تظهر الواجهات** — حسب نوع المستخدم.
8. **داشبورد الأدمن** — أقسامه والتقارير.
9. **داشبورد المستخدم** — أقسامه والتقارير.
10. **التقارير** — للأدمن وللمستخدم ومعاييرها.

هي المرجع لبناء وتطوير الواجهات والمنطق؛ أي تغيير في المسارات أو الصلاحيات أو القوانين يُقيّس بهذا المستند.
