# دعم لوحات التحكم للمستخدم بالذكاء الصناعي والشات بصلاحيات محدودة ونظام ERP

## النطاق والهدف

- **دعم لوحات التحكم للمستخدم بالذكاء الصناعي والشات** — مساعد ذكي (شات) مدمج في لوحة تحكم المستخدم — إجابات عن متجره، منتجاته، المؤشرات، آلية البيع، والتكامل مع ERP عند التفعيل.
- **صلاحيات محدودة** — المستخدم (تاجر) لا يرى ولا يطلب بيانات مستخدمين آخرين ولا إدارة المنصة؛ السياق المُمرَّر للمساعد يقتصر على دوره وصلاحياته.
- **نظام ERP** — الربط مع نظام تخطيط موارد المؤسسة (ERP) وSCM عند التفعيل — عرض حالة ERP في لوحة التحكم والتكامل مع الشات ضمن حدود الصلاحيات.

---

## 1. الشات والذكاء في لوحة التحكم

### 1.1 الواجهة

| البند | الوصف |
|--------|--------|
| **قسم «الذكاء والشات»** | في الشريط الجانبي للوحة التحكم — رابط «مساعد شيخه (صلاحيات محدودة)». |
| **لوحة المساعد** | منطقة محادثة (رسائل المستخدم والردود)، حقل إدخال، زر إرسال. |
| **الصلاحيات المحدودة** | نص توضيحي: الإجابة فقط عن متجرك، منتجاتك، المؤشرات، آلية البيع، والتكامل مع ERP/SCM عند التفعيل. |

### 1.2 API الشات

| البند | الوصف |
|--------|--------|
| **المسار** | `POST /api/ai/chat` — يتطلب مصادقة (Bearer token). |
| **الجسم** | `{ "message": "نص الرسالة", "history": [ { "role": "user"|"assistant", "content": "..." } ] }`. |
| **الرد** | `{ "success": true, "reply": "نص الرد", "usage": {...} }` أو رسالة خطأ. |
| **الصلاحيات المحدودة (خادم)** | عند وجود مستخدم مسجّل: يُضاف توجيه للنظام (system prompt) يحدّ نطاق الإجابات: متجر المستخدم، منتجاته، المؤشرات، آلية البيع، التكامل وERP فقط — دون بيانات مستخدمين آخرين أو إدارة المنصة. |

---

## 2. صلاحيات محدودة — التطبيق

### 2.1 حسب الدور (مثال: تاجر)

| الصلاحية | التطبيق |
|----------|----------|
| **ما يُسمح به** | الإجابة عن: تسجيل المتجر، عرض البضائع، لوحة التحكم، منتجاتي، المؤشرات، آلية البيع، التكامل مع ERP/SCM (عند التفعيل)، روابط الصفحات (خريطة الموقع، آلية البيع، التكامل). |
| **ما لا يُسمح به** | بيانات متاجر أخرى، بيانات مستخدمين آخرين، إدارة المنصة، تقارير إدارية شاملة، أو أي محتوى خارج نطاق دور التاجر. |

### 2.2 تعزيز النظام (System Prompt)

- في الخادم: إضافة جملة توجيهية عند استدعاء الشات من مستخدم مصادق:  
  «صلاحيات المستخدم محدودة (داشبورد تاجر): أجب فقط عن متجره، منتجاته، المؤشرات، آلية البيع، التكامل وERP عند التفعيل. لا تقدم بيانات مستخدمين آخرين ولا إدارة المنصة.»

## 3. نظام ERP والربط مع لوحة التحكم

### 3.1 حالة ERP في المنصة

| البند | الوصف |
|--------|--------|
| **واجهة ERP** | حالة النظام عبر `GET /api/erp/status` — مفعّل/غير مفعّل؛ وحدات ERP عبر `GET /api/erp/modules` (مخزون، مبيعات، مشتريات، أطراف، حاويات، مالية). |
| **في لوحة التحكم** | يمكن عرض «نظام ERP مفعّل» أو رابط صفحة التكامل في قسم المساعد أو في قسم الإعدادات. |
| **الشات والـ ERP** | المساعد (بصلاحيات محدودة) يوضح للمستخدم كيف يفعّل التكامل مع ERP، وصفحة التكامل، وعدم إتاحة بيانات ERP لشركات أخرى. |

### 3.2 التكامل مع ERP الشركة

| البند | الوصف |
|--------|--------|
| **Sheikha Partner API** | تكامل مع ERP الشركة (SAP، Oracle، Microsoft، إلخ) — مزامنة عروض، أوامر، كميات — حسب العقد والصلاحيات. |
| **صلاحيات المستخدم** | التاجر يرى فقط ما يخصه من تكامل (إن وُجد)؛ إعدادات التكامل الكاملة لمسؤول الشركة — انظر: نظام-الصلاحيات-والمستخدمين-والشركات-والتكامل-والتقارير-الاحترافية.md. |
