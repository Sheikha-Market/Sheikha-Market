# آلية البيع بالموقع وخطة مبيعات باسم شيخه

## 1. آلية البيع — أدوات المستخدم بالموقع فقط

الهدف: أن يستخدم المستخدم **أدوات الموقع فقط** لعرض المنتجات، عرض الشاحنات/الحاويات، والبيع — دون الحاجة للخروج إلى قنوات أخرى للاتفاق.

### 1.1 الأدوات المتاحة (بالموقع)

| الأداة | الوصف | الصفحة / API |
|--------|--------|--------------|
| **عرض منتجات / بضائع** | التاجر يعرض عروض السكراب (حديد، نحاس، ألومنيوم، إلخ) مع الكمية والسعر والمنطقة | `عرض-بضائع.html` — `POST /api/listings` |
| **عرض شاحنات / حاويات** | التاجر يعرض حاويات أو شاحنات متاحة للنقل أو التأجير | لوحة التحكم — `GET/POST /api/containers` |
| **بيع لموقع سكراب** | المشتري يطلب أو يختار عرضاً من السوق — العملية كلها تتم داخل الموقع (طلب، قبول، تأكيد) | عرض البضائع + لوحة التحكم — طلبات وعروض داخل المنصة |

### 1.2 تفعيل آلية البيع

- **عرض المنتجات:** مفعّل — أي تاجر يمكنه إضافة عرض عبر `عرض-بضائع.html` أو API.
- **عرض الشاحنات/الحاويات:** مفعّل — للمستخدمين المسجلين من لوحة التحكم (`/api/containers`, ` /api/my-containers`).
- **البيع بالموقع فقط:**
  - الخطوة الحالية: المستخدم يعرض بضاعته، والمشترون يتواصلون (مثلاً واتساب أو عبر المنصة لاحقاً).
  - التفعيل الكامل: إضافة **طلبات شراء** داخل الموقع (مشتري يضغط «اطلب» على عرض → يظهر طلب عند البائع → قبول/رفض → تأكيد البيع). كل ذلك داخل سوق شيخه دون كشف بيانات الطرف الآخر إلا ما تسمح به السياسة (انظر خطة مبيعات باسم شيخه).

### 1.3 روابط سريعة للمستخدم

- **أريد عرض بضاعتي** → [عرض بضائع](public/عرض-بضائع.html)
- **أريد عرض شاحنات/حاويات** → تسجيل الدخول → لوحة التحكم → حاوياتي
- **أريد شراء من السوق** → [السوق / المتاجر المتخصصة](public/index.html#market) أو عرض البضائع

## 2. خطة مبيعات باسم شيخه — إخفاء أسماء الأطراف

الهدف: أن تظهر **جميع العمليات باسم «سوق شيخه»** ولا تظهر أسماء البائع والمشتري لبعضهما (أو تظهر بشكل محدود حسب السياسة).

### 2.1 المبدأ

- **الظهور للعميل:** الفواتير، التأكيدات، وإشعارات البيع تظهر كأنها من «سوق شيخه» (أو «Sheikha Market»).
- **عدم كشف الهوية بين الأطراف:** البائع لا يرى اسم المشتري الحقيقي (أو يرى اسماً مستعاراً أو «مشتري عبر شيخه»)، والمشتري لا يرى اسم البائع الحقيقي (أو يرى «بائع عبر شيخه» أو «سوق شيخه»).
- **البيانات الداخلية:** إدارة شيخه لديها معرفة الطرفين للدعم والامتثال والمنازعات، لكن الواجهة للمستخدم تعكس سياسة «العمليات باسم شيخه».

### 2.2 تطبيق تقني (مختصر)

| العنصر | التطبيق |
|--------|----------|
| **عرض البائع في السوق** | لا يعرض «اسم التاجر» للمشتري — يعرض «عرض من سوق شيخه» أو «بائع معتمد عبر شيخه». |
| **عرض المشتري للبائع** | عند إنشاء طلب، لا يرسل للبائع «اسم المشتري» — يرسل «طلب شراء عبر سوق شيخه» مع رقم الطلب والكمية والسعر. |
| **الفواتير والتأكيدات** | الشعار والاسم الظاهر: «سوق شيخه» — بدون أسماء الطرفين أو مع «مشتري عبر شيخه» / «بائع عبر شيخه». |
| **الرسائل داخل المنصة** | إن وُجدت رسائل بين طرفين: يمكن عرض «مستخدم عبر شيخه» أو عدم إظهار الاسم حتى يتم الاتفاق على كشف الهوية. |

### 2.3 مراحل التنفيذ

1. **الآن:** توثيق السياسة (هذه الوثيقة) واعتماد مبدأ «العمليات باسم شيخه» في المحتوى والواجهة.
2. **قريب:** تعديل واجهة عرض البضائع والسوق لعدم إظهار اسم التاجر الحقيقي للمشتري (استبدال بـ «سوق شيخه» أو «بائع عبر شيخه»).
3. **مع تفعيل الطلبات:** عند إضافة نظام طلبات الشراء، عدم إرسال اسم المشتري للبائع في الواجهة، وإظهار الفواتير والتأكيدات باسم «سوق شيخه».

---

## 3. ملخص

- **آلية البيع:** المستخدم يستخدم أدوات الموقع فقط — عرض منتجات، عرض شاحنات/حاويات، والبيع (مع تفعيل طلبات الشراء لاحقاً) داخل سوق شيخه.
- **مبيعات باسم شيخه:** جميع العمليات تظهر باسم «سوق شيخه»، ولا تظهر أسماء الأطراف لبعضهم (أو تظهر بشكل محايد مثل «بائع/مشتري عبر شيخه») مع الاحتفاظ بالبيانات داخلياً للإدارة والامتثال.

هذه الوثيقة مرجع لـ **آلية البيع بالموقع** و**خطة مبيعات باسم شيخه**.
