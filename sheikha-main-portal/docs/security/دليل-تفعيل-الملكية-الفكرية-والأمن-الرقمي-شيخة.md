# دليل تفعيل الملكية الفكرية والأمن الرقمي — شيخة

## بسم الله الرحمن الرحيم

هذا الدليل يفعّل حماية الملكية الفكرية والأمن الرقمي بشكل تشغيلي داخل المشروع، وليس كنصوص فقط.

---

## 1) الهدف

- توثيق الأصول الرقمية الحساسة.
- إثبات نسبة الملفات للمنظومة عبر بصمات SHA-256.
- اكتشاف أي تغيير غير مصرح به عبر المقارنة الدورية.
- دعم ملف الملكية أمام أي إجراء نظامي.

---

## 2) أوامر التفعيل

### تشغيل إثبات الملكية (الوضع القياسي)

```bash
npm run ops:ip:proof
```

### تشغيل إثبات الملكية (الوضع الصارم)

```bash
npm run ops:ip:proof:strict
```

> الوضع الصارم يفشل إذا كانت ملفات حرجة غير موجودة.

### التشغيل الدوري التلقائي (PM2)

```bash
npm run ops:ip:proof:pm2:start
```

إيقاف التشغيل الدوري:

```bash
npm run ops:ip:proof:pm2:stop
```

وضع دوري صارم (اختياري):

```bash
npm run ops:ip:proof:loop:strict
```

> الوضع الدوري ينشئ تقريرًا تلقائيًا كل 24 ساعة (قابل للتعديل عبر `SHEIKHA_IP_PROOF_INTERVAL`).

### تفعيل تنبيه تغيّر Root Hash (اختياري)

يمكن تفعيل التنبيه عند تغيّر البصمة الجذرية عبر:

- `SHEIKHA_IP_ALERT_WEBHOOK_URL`
- `SHEIKHA_IP_TELEGRAM_BOT_TOKEN`
- `SHEIKHA_IP_TELEGRAM_CHAT_ID`
- `SHEIKHA_IP_ALERT_COOLDOWN_SEC` (الافتراضي: `300` ثانية)

> إذا كانت هذه القيم غير مضبوطة، سيبقى الرصد والتسجيل فعالًا محليًا بدون إرسال خارجي.

---

## 3) نواتج الرقمنة

عند التشغيل يتم إنشاء:

- `reports/ip/ip-proof-latest.json`
- `reports/ip/ip-proof-<timestamp>.json`
- `reports/ip/ip-proof-<timestamp>.txt`

وتشمل:
- بصمة SHA-256 لكل ملف.
- بصمة جذرية واحدة (Root Hash) للمنظومة.
- سجل الملفات الحرجة المفقودة إن وجدت.

---

## 4) دورة الحوكمة المقترحة

1. تشغيل `ops:ip:proof` بعد كل تعديل كبير.
2. حفظ التقرير الدوري في أرشيف آمن منفصل.
3. توقيع تقارير الإثبات من المسؤول المعتمد.
4. مقارنة `ip-proof-latest.json` أسبوعيًا لرصد أي انحراف.

---

## 5) ضوابط أمنية مرافقة (إلزامية)

- تفعيل أقل صلاحية (Least Privilege) لكل حساب.
- عدم تخزين أي أسرار في المستودع.
- تدوير المفاتيح الحساسة دوريًا.
- مراقبة محاولات العبث بالسجلات.
- نسخ احتياطي مشفّر وخارج بيئة التشغيل.

## 7) تنبيه نظامي

هذا النظام يقدم **إثباتًا تقنيًا** قويًا (سلامة وتاريخ وتغيّر)، لكنه لا يغني عن:
- تسجيل العلامة والأعمال حسب الأنظمة المعمول بها.
- الإجراءات القانونية الرسمية عند النزاع.

---

## 8) تعريف الجاهزية

تعتبر الحماية مفعلة تشغيليًا عندما:
- تنجح أوامر الإثبات.
- تتوفر التقارير في `reports/ip/`.
- يوجد تشغيل دوري ومراجعة ثابتة للتقارير.
- يكون `sheikha-ip-proof` في حالة `online` عند استخدام PM2.
- يظهر سجل رصد/تنبيه تغيّر Root Hash في `logs/ip-proof-guard.log` عند حدوث تغير.
