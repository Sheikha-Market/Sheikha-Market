# نظام التعليمات والإجراءات والقوانين — شيخة

## 1) التعليمات الحاكمة
- المرجعية: الكتاب والسنة.
- القاعدة التشغيلية: لا ضرر ولا ضرار.
- أي إجراء تقني أو تجاري يجب أن يكون موثقًا وقابلًا للمراجعة.

## 2) إجراءات التشغيل القياسية (SOP)
1. **قبل النشر**
   - مراجعة تغييرات الكود.
   - تشغيل فحوصات CI الأمنية.
   - التحقق من عدم تسرب أسرار.
2. **أثناء النشر**
   - استخدام قناة نشر رسمية فقط.
   - تسجيل الإصدار ووقت النشر.
3. **بعد النشر**
   - فحص `/api/health`.
   - مراقبة سجل الأمان.
   - توثيق أي حادث/استثناء.

## 3) نظام الحوكمة القانونية
- حماية الملكية الفكرية إلزامية.
- منع إعادة التوزيع غير المصرح.
- تقييد الصلاحيات حسب أقل امتياز.
- حفظ سجل تدقيق للأحداث الحساسة.

## 4) إجراءات الحوادث
- احتواء فوري.
- توثيق فني وقانوني.
- إصلاح السبب الجذري.
- مراجعة ضوابط المنع بعد الإغلاق.

## 5) نظام الامتثال الرقمي
- التشفير أثناء النقل والتخزين.
- إدارة أسرار مركزية.
- فحص ملفات مرفوعة ضد البرمجيات الضارة.
- بوابة تحقق أمنية قبل أي إصدار رسمي.

## 6) مؤشرات القياس
- نسبة نجاح فحوصات CI الأمنية.
- زمن الاستجابة للحوادث.
- عدد تنبيهات الأمان المغلقة.
- عدد الإصدارات الموثقة مع checksum وSBOM.
