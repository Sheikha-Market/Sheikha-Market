# دليل الاستجابة للحوادث الأمنية — شيخة

## الهدف
توحيد التعامل مع أي حادث أمني بسرعة ودقة لتقليل الأثر وحماية العملاء والبيانات.

## تعريف الحادث
أي نشاط يهدد السرية أو السلامة أو الإتاحة، مثل:
- تسرب مفتاح API.
- وصول غير مصرح.
- ملف ضار مرفوع.
- توقف خدمة بسبب هجوم.

## مستويات الخطورة
- **حرج (P0):** تسرب أسرار أو اختراق فعلي.
- **عالٍ (P1):** محاولة اختراق موثقة أو ضرر متوقع كبير.
- **متوسط (P2):** سلوك مشبوه دون أثر مباشر.
- **منخفض (P3):** إنذار غير مؤكد.

## إجراءات الاستجابة
1. **الاحتواء الفوري**
   - إيقاف المفتاح المتسرب وتدويره فورًا.
   - حظر المصدر المهاجم إن لزم.
   - عزل الملفات المشبوهة في quarantine.

2. **التحقق والتحليل**
   - مراجعة `logs/security-audit.ndjson`.
   - توثيق وقت البداية، المؤشرات، ونطاق الأثر.
   - حفظ الأدلة الفنية بدون تعديل.

3. **المعالجة**
   - إصلاح الثغرة الجذرية.
   - تحديث الإعدادات أو القواعد الأمنية.
   - تنفيذ تصحيح آمن مع rollback جاهز.

4. **الاستعادة**
   - إعادة الخدمة تدريجيًا.
   - مراقبة مشددة لمدة 24-72 ساعة.
   - التأكد من عدم تكرار المؤشرات.

5. **ما بعد الحادث**
   - تقرير حادث رسمي خلال 48 ساعة.
   - تحديث السياسات وخطة المنع.
   - تحديث قائمة التحقق قبل الإصدار.

## SLA داخلي
- P0: بدء الاستجابة خلال 15 دقيقة.
- P1: خلال 60 دقيقة.
- P2/P3: خلال يوم عمل.

## قائمة تحقق سريعة
- [ ] احتواء فوري.
- [ ] تدوير أسرار متأثرة.
- [ ] توثيق كامل للحادث.
- [ ] تحليل السبب الجذري.
- [ ] تنفيذ ضوابط منع التكرار.

# دليل الاستجابة للحوادث الأمنية — منظومة شيخة

## الهدف
ضمان استجابة سريعة ومنضبطة لأي حادث أمني مع الحد الأدنى من الضرر.

## مستويات الحوادث
- **P1 حرج:** تسريب مفاتيح/اختراق/تعطل خدمة أساسية.
- **P2 عالي:** نشاط ضار مؤثر أو محاولات هجوم متكررة.
- **P3 متوسط:** أخطاء أمنية دون أثر مباشر على البيانات.

## خطوات الاستجابة
1. **الاكتشاف:** رصد الإنذار عبر السجلات/التنبيهات.
2. **الاحتواء:** عزل المسار أو الخدمة المتأثرة فورًا.
3. **التقييم:** تحديد نطاق الأثر (بيانات/مستخدمين/زمن).
4. **المعالجة:** إصلاح السبب الجذري وإغلاق الثغرة.
5. **الاسترجاع:** إعادة الخدمة مع مراقبة مشددة.
6. **ما بعد الحادثة:** تقرير رسمي + إجراءات منع تكرار.

## إجراءات فورية للحوادث الحرجة
- تدوير جميع الأسرار المحتمل تسريبها:
  - `JWT_SECRET`
  - `OPENAI_API_KEY` و`NVIDIA_API_KEY` و`LINEAR_API_KEY`
  - مفاتيح الدفع والتكاملات الأخرى
- إيقاف نقاط الوصول الحساسة مؤقتًا إذا لزم.
- فرض تسجيل دخول جديد للجلسات الحساسة.

## توثيق الحادثة
- وقت الاكتشاف، وقت الاحتواء، وقت الاسترجاع.
- السبب الجذري.
- الأنظمة/البيانات المتأثرة.
- الإجراءات المتخذة.
- الدروس المستفادة.

## قنوات التواصل
- إشعار فريق التشغيل والأمن فورًا.
- إشعار المالك بتقرير تنفيذي مختصر.
- إشعار العملاء المتأثرين عند وجود أثر مباشر.

## اختبارات الجاهزية
- تمرين محاكاة حادثة (Tabletop) كل ربع سنة.
- اختبار استرجاع نسخ احتياطي مرة شهريًا.
