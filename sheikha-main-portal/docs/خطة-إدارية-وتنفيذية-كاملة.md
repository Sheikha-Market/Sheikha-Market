# خطة إدارية وتنفيذية كاملة — منظومة وشبكة شيخه

## النطاق

خطة إدارية وتنفيذية لتنفيذ **المنظومة والشبكة كاملة** لسوق شيخه — مرتبطة بالرسومات الهندسية والفنية والخواص والقوانين الكاملة (انظر: تعريف-شيخه-الكامل-القوانين، الهندسة-الكاملة-للموقع-وقوانينه، رسومات-هندسية-وفنية.html، نظام-متابعة-وتنفيذ-وتحليل.html).

---

# الجزء الأول — الخطة الإدارية

## 1.1 الأهداف الإدارية

| الرقم | الهدف | المؤشر |
|-------|--------|--------|
| م-1 | وضوح الرسالة والمسار (قانون أ-1، أ-2) | رسالة واحدة في الرئيسية؛ مسار بيع بثلاث خطوات فقط |
| م-2 | تواصل واحد (قانون أ-3) | رقم واتساب واحد في الهيدر والهيرو والفوتر |
| م-3 | المحتوى حلال (قانون أ-4) | لا صور ذوات أرواح؛ زخارف هندسية فقط |
| م-4 | البيانات والخصوصية (قانون أ-6) | سياسة خصوصية؛ لا نقل بيانات دون موافقة |
| م-5 | منع الهندسة العكسية (قانون ت-5) | وثائق وشروط تمنع التفكيك والاستنساخ |

## 1.2 المراحل الإدارية والمسؤوليات

| المرحلة | المهام | المسؤول (مقترح) | الحالة |
|---------|--------|-------------------|--------|
| **إ-1** | اعتماد القوانين والخواص (أ، هـ، ت) وإبلاغ الفريق | الإدارة | لم يبدأ |
| **إ-2** | تحديد سياسة الخصوصية والاشتراكات والعمولات | الإدارة + قانوني | لم يبدأ |
| **إ-3** | اعتماد خطة التسويق والظهور (رابط التكامل، السعودية أولاً) | الإدارة + تسويق | لم يبدأ |
| **إ-4** | اعتماد نموذج «من يدفع» (مقدم الخدمة / المستخدم النهائي) ووضع الأسعار | الإدارة | لم يبدأ |
| **إ-5** | مراجعة دورية للخطة (ربع سنوية) وتحديث نظام المتابعة | الإدارة | لم يبدأ |

# الجزء الثاني — الخطة الهندسية

## 2.1 الأهداف الهندسية (مرتبطة بالقوانين هـ-1 إلى هـ-6)

| الرقم | الهدف | القانون | المؤشر |
|-------|--------|---------|--------|
| هـ-1 | هيكل ثابت للصفحة الرئيسية | ه-1 | هيدر → هيرو → مسار 3 خطوات → أسعار/لماذا شيخه → تواصل → فوتر |
| هـ-2 | هيدر موحد (شعار، روابط، واتساب، Integration) | ه-2 | موجود في index.html وكل الصفحات الرئيسية |
| هـ-3 | تسلسل هرمي H1/H2/H3 وتباين | ه-3، ه-4 | حجم خط ≥16px؛ ألوان الهوية |
| هـ-4 | استجابة (موبايل، تابلت) | ه-5 | هيدر يتحول لقائمة على الشاشات الصغيرة |
| هـ-5 | الخادم والبيانات (8080، data/) | ه-6 | APIs مسجّلة قبل الملفات الثابتة؛ traders.json، listings.json |

## 2.2 المراحل الهندسية والمهام

| المرحلة | المهام | المسؤول | الحالة |
|---------|--------|---------|--------|
| **ه-1** | التأكد من هيكل الصفحة الرئيسية مطابق لـ ه-1 | هندسة واجهة | قيد التنفيذ |
| **ه-2** | توحيد الهيدر والفوتر في كل الصفحات (تسجيل، عرض بضائع، لوحة تحكم، تكامل) | هندسة واجهة | قيد التنفيذ |
| **ه-3** | إعداد الرسومات الهندسية والفنية (معمارية، شبكة، سلسلة توريد) — حلال فقط | هندسة | لم يبدأ |
| **ه-4** | توثيق APIs وخريطة الشبكة في وثيقة/صفحة واحدة | هندسة | لم يبدأ |
| **ه-5** | مراجعة أمان (HTTPS، عدم تخزين كلمات مرور واضحة، API Key) | هندسة | لم يبدأ |

# الجزء الثالث — الخطة الفنية

## 3.1 الأهداف الفنية

| الرقم | الهدف | المؤشر |
|-------|--------|--------|
| ف-1 | الخادم (Node/Express) يعمل على 8080؛ CORS، JSON، data/ | server.js؛ مجلد data/ |
| ف-2 | APIs: تسجيل تجار، عروض، سوق، أسعار، تخصصات، سلسلة توريد | GET/POST مسجّلة في server.js |
| ف-3 | التكامل (Sheikha Partner API) — وثائق ومفتاح API حسب الصلاحيات | صفحة تكامل.html؛ وثيقة أساس التكامل |
| ف-4 | لوحة تحكم: عروضي، طلباتي، حاويات، تقارير | لوحة-التحكم.html + APIs |
| ف-5 | PWA (manifest، Service Worker) للتثبيت من المتصفح | manifest.json، sw.js |
| ف-6 | نظام متابعة وتنفيذ وتحليل (مهام إدارية، هندسية، فنية) | نظام-متابعة-وتنفيذ-وتحليل.html |

## 3.2 المراحل الفنية والمهام

| المرحلة | المهام | المسؤول | الحالة |
|---------|--------|---------|--------|
| **ف-1** | تفعيل سجل الطلبات (orders) وربطها بلوحة التحكم | تطوير | لم يبدأ |
| **ف-2** | تفعيل مفتاح API وصلاحيات محدودة (حزم صلاحيات) في الخادم | تطوير | لم يبدأ |
| **ف-3** | إعداد صفحة «نظام متابعة وتنفيذ وتحليل» وربطها بالخطة | تطوير | لم يبدأ |
| **ف-4** | إعداد صفحة «رسومات هندسية وفنية» (حلال) وربطها بالخواص والقوانين | تطوير | لم يبدأ |
| **ف-5** | تحسين SEO (meta، عناوين، روابط داخلية) واختبار الأداء | تطوير | لم يبدأ |

# الجزء الخامس — الربط بالخواص والقوانين

## 5.1 الخواص (من تعريف شيخه)

سوق متخصص 100% | B2B + B2C | متاجر حسب التخصص | أسعار مرجعية | FBS | Sheikha Partner API | ذكاء مخصص للسكراب | متوافق مع الشريعة | الهوية المتكاملة (علامة على الشاحنات ومنتجات إعادة التدوير) | منظومة الشهادات البيئية والقياس | تأهيل المستخدمين وتقييم الأداء وقياس الإنتاجية

## 4.2 القوانين (مرجع سريع)

- **إدارية (أ-1 إلى أ-7):** وضوح الرسالة، مسار واحد، تواصل واحد، حلال، لغة، خصوصية، عدم إظهار تفاصيل تقنية.
- **هندسية (ه-1 إلى ه-6):** هيكل الصفحة، هيدر، تسلسل هرمي، ألوان وخط، استجابة، خادم وبيانات.
- **تجارة ومنصة (ت-1 إلى ت-5):** تخصص، تصنيف منتجات، مصدر أسعار، عمولة واشتراكات، منع الهندسة العكسية.
