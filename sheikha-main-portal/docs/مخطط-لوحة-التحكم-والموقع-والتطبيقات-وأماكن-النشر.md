# مخطط لوحة التحكم الرئيسية، مخطط الموقع، لوحات التحكم للمستخدمين، مخطط التطبيقات الإلكترونية، مخطط ERP/SCM، مخطط التكامل، وأماكن النشر والظهور

## النطاق والهدف

- **مخطط لوحة التحكم الرئيسية** — لوحة تحكم الأدمن (نظرة عامة، إدارة مستخدمين، متاجر، عروض، تقارير، إعدادات).
- **مخطط الموقع** — الصفحات الأساسية والداعمة، التنقل، خريطة الموقع — مرجع: مخطط-الموقع-والنظام-المتكامل.md.
- **مخطط لوحات التحكم للمستخدمين** — لوحة التاجر، المشتري، شركة النقل، وأي أدوار أخرى.
- **مخطط التطبيقات الإلكترونية** — التطبيقات للأجهزة (موبايل، تابلت، ويب) وأنظمة التشغيل (iOS، Android، Web، Windows) وأماكن نشرها في المتاجر الإلكترونية.
- **مخطط ERP و SCM** — نظام ERP (تخطيط موارد المؤسسة) ونظام SCM (إدارة سلسلة التوريد) — وحدات، APIs، ربط مع المنصة.
- **مخطط التكامل (Integration)** — تكامل مع الأنظمة والشركات (Sheikha Partner API، ERP/SCM خارجي)، أماكن ظهور التكامل.
- **أماكن نشر وظهور كل مكون** — التطبيقات (متاجر إلكترونية)، ERP/SCM والتكامل (صفحة تكامل، وثائق API، متاجر تكامل، أدلة مطورين).

# الجزء الرابع — مخطط التطبيقات الإلكترونية (الأجهزة وأنظمة التشغيل)

## 4.1 الأجهزة المستهدفة

| الجهاز | الوصف | الأولوية |
|--------|--------|----------|
| **موبايل (هاتف ذكي)** | تطبيق أو ويب متجاوب للاستخدام على الهاتف — تسجيل، عرض بضائع، لوحة تحكم، بحث. | عالية |
| **تابلت** | نفس الوظائف — واجهة متكيفة للشاشة المتوسطة. | متوسطة |
| **ويب (سطح المكتب)** | الموقع الحالي (HTML/CSS/JS) — يعمل على أي متصفح (Chrome، Safari، Edge، Firefox). | مُنجَز |
| **سطح المكتب (Desktop)** | تطبيق سطح مكتب (Windows، macOS) — اختياري؛ يمكن الاعتماد على الويب أو PWA. | منخفضة |

## 4.2 أنظمة التشغيل المستهدفة

| نظام التشغيل | المنصة | نوع التطبيق |
|---------------|--------|-------------|
| **iOS** | أبل (iPhone، iPad) | تطبيق أصلي (Swift) أو هجين (React Native، Flutter) أو PWA |
| **Android** | جوجل (هواتف، أجهزة لوحية) | تطبيق أصلي (Kotlin) أو هجين أو PWA |
| **Web** | أي متصفح (Chrome، Safari، Edge، Firefox) | موقع ويب متجاوب — مُنجَز (public/) |
| **Windows** | مايكروسوفت (سطح المكتب) | تطبيق UWP/Windows أو PWA أو ويب |
| **macOS** | أبل (سطح المكتب) | تطبيق Mac أو PWA أو ويب |

## 4.3 خيارات بناء التطبيقات

| الخيار | الوصف | مزايا |
|--------|--------|--------|
| **ويب متجاوب (Responsive Web)** | نفس الموقع يعمل على الموبايل والتابلت والمكتب — مُنجَز جزئياً | لا حاجة لتطبيق منفصل؛ تحديث واحد. |
| **PWA (Progressive Web App)** | موقع ويب + manifest + Service Worker — يُضاف إلى الشاشة الرئيسية ويعمل دون اتصال جزئي | تجربة شبيهة بالتطبيق؛ نشر عبر الويب دون متجر. |
| **تطبيق هجين (Hybrid)** | React Native أو Flutter أو Capacitor — كود واحد لـ iOS و Android | كود واحد؛ نشر في App Store و Google Play. |
| **تطبيق أصلي (Native)** | Swift (iOS)، Kotlin (Android) — منفصل لكل منصة | أداء وتجربة أفضل؛ جهد أعلى. |

## 4.4 الملفات الحالية ذات العلاقة

| الملف | الغرض |
|--------|--------|
| public/user-app.html | تطبيق/واجهة مستخدم |
| public/scrap-specialist.html | واجهة متخصص سكراب |
| public/admin-app.html | واجهة أدمن |
| public/manifest.json | PWA — اسم التطبيق، أيقونات، عرض (standalone/fullscreen) |

# الجزء الخامس — أماكن نشر التطبيقات في المتاجر الإلكترونية

## 5.1 المتاجر الإلكترونية الرئيسية

| المتجر | المنصة | نظام التشغيل | الرابط النموذجي |
|--------|--------|---------------|------------------|
| **App Store** | أبل | iOS، iPadOS، macOS (تطبيقات Mac) | https://apps.apple.com |
| **Google Play** | جوجل | Android | https://play.google.com/store |
| **Microsoft Store (متجر Microsoft)** | مايكروسوفت | Windows، Xbox (إن وُجد) | https://www.microsoft.com/store/apps |
| **ويب (الموقع الرسمي)** | شيخه | أي متصفح — لا متجر | https://sheikha.top أو النطاق المعتمد |

## 5.2 شروط وأماكن النشر (ملخص)

| المتجر | متطلبات النشر (ملخص) | ملاحظات |
|--------|------------------------|----------|
| **App Store** | حساب Apple Developer (مدفوع)، تطبيق iOS/IPadOS، مراجعة أبل، التزام بإرشادات أبل | ضروري للنشر على أجهزة أبل؛ مراجعة قد تستغرق أياماً. |
| **Google Play** | حساب مطور Google Play (رسوم مرة واحدة)، تطبيق Android (APK/AAB)، سياسة خصوصية، مراجعة جوجل | ضروري للنشر على أندرويد. |
| **Microsoft Store** | حساب Microsoft Partner Center، تطبيق UWP أو PWA مُعبّأ، مراجعة مايكروسوفت | اختياري — جزء من المستخدمين يستخدمون Windows. |
| **ويب / PWA** | لا متجر — النشر على الموقع الرسمي؛ المستخدم يفتح الرابط ويُضيف إلى الشاشة الرئيسية (إن كان PWA) | بدون رسوم متجر؛ يصل لجميع الأجهزة عبر المتصفح. |

## 5.3 خريطة النشر (أين يُنشر كل نوع)

| نوع التطبيق | أين يُنشر |
|-------------|-----------|
| **موقع ويب (متجاوب)** | النطاق الرسمي (sheikha.top أو المعتمد) — لا متجر. |
| **PWA** | نفس النطاق؛ المستخدم يزور الموقع ويختار «إضافة إلى الشاشة الرئيسية» — لا متجر ضروري. |
| **تطبيق iOS (أصلي أو هجين)** | App Store (أبل) — بعد إنشاء حساب مطور ومراجعة أبل. |
| **تطبيق Android (أصلي أو هجين)** | Google Play — بعد إنشاء حساب مطور جوجل ومراجعة جوجل. |
| **تطبيق Windows** | Microsoft Store — اختياري. |

## 5.4 أولوية النشر (اقتراح)

| الأولوية | المكان | الوصف |
|----------|--------|--------|
| 1 | **الموقع الرسمي (ويب متجاوب)** | يعمل الآن على كل الأجهزة عبر المتصفح — لا حاجة لمتجر. |
| 2 | **PWA على نفس الموقع** | إكمال manifest و Service Worker — المستخدم يضيف «شيخه» إلى الشاشة الرئيسية كتطبيق. |
| 3 | **Google Play** | عند توفر نسخة Android (هجين أو أصلي) — وصول لمستخدمي أندرويد عبر المتجر. |
| 4 | **App Store** | عند توفر نسخة iOS — وصول لمستخدمي أبل. |
| 5 | **Microsoft Store** | اختياري — عند توفر نسخة Windows أو PWA مُعبّأة لـ Microsoft. |

# الجزء السابع — مخطط التكامل (Integration)

## 7.1 الغرض ومكونات التكامل

| البند | الوصف |
|--------|--------|
| **التكامل** | ربط سوق شيخه مع أنظمة الشركات الخارجية (ERP، SCM، منصات أخرى) — عروض، طلبات، كميات، أسعار، تقارير. |
| **Sheikha Partner API** | واجهة برمجية للشركاء — مزامنة عروض، طلبات، كميات؛ تكامل مع SAP، Oracle، Odoo، Microsoft Dynamics، أو أي ERP/SCM. |
| **مكونات التكامل** | APIs REST (تسجيل، عروض، متاجر، كميات، مؤشرات، عمليات)； توثيق؛ أمثلة (استيراد/تصدير CSV، استدعاء API). |

| الجهة | الوصف |

| **منصات سحابية** | حفظ سحابي، مؤشرات، (اختياري) جوجل شيتس، أوفيس 365. |
| **الدولة والمنصات الحكومية** | ربط مع منصات حكومية سعودية أو برامج دولة (ندلب) — عند التوفر والبروتوكولات. |

# الجزء الثامن — أماكن نشر وظهور ERP و SCM والتكامل

## 8.1 جدول أماكن النشر والظهور

| المكون | أين يُنشر / يظهر |
|--------|-------------------|
| **التطبيقات الإلكترونية** | انظر الجزء الخامس — الموقع الرسمي، PWA، App Store، Google Play، Microsoft Store. |
| **ERP** | الخادم (APIs)； صفحة تكامل.html؛ لوحة التحكم؛ وثائق المطورين؛ (لاحقاً) متاجر تكامل (SAP App Center، إلخ). |
| **SCM** | الخادم (APIs)； صفحة تكامل.html؛ لوحة التحكم؛ وثائق المطورين؛ صفحة سلسلة التوريد. |
| **التكامل (Sheikha Partner API)** | صفحة تكامل.html؛ وثائق API (موقع أو مستودع توثيق)； خطة تسويقية (جوجل، بينغ، محتوى «تكامل ERP معادن»)؛ (لاحقاً) SAP App Center، Microsoft AppSource، أدلة B2B. |

## 8.2 متاجر تكامل وحلول (أماكن ظهور للتكامل)

| المتجر / المنصة | الوصف | ملاحظات |
|------------------|--------|----------|
| **صفحة التكامل (sheikha.top/تكامل.html)** | الصفحة الرسمية للتكامل — تظهر في الموقع وفي نتائج البحث (SEO). | أساسي |
| **وثائق API (داخل الموقع أو GitHub)** | توثيق Sheikha Partner API — للمطورين والشركات. | أساسي |
| **SAP App Center** | إدراج حل «شيخه» أو تكامل شيخه مع SAP — عند التوفر. | لاحقاً |
| **Microsoft AppSource** | إدراج تطبيق/تكامل لـ Microsoft 365 أو Dynamics — عند التوفر. | لاحقاً |
| **أدلة B2B / قطاع المعادن** | إدراج شيخه في أدلة تكامل قطاع المعادن والسكراب وإعادة التدوير. | مستمر |
| **خطة تسويقية (بحث، محتوى)** | كلمات «تكامل ERP معادن»، «سوق سكراب API» — ظهور في جوجل وبينغ. | انظر: نظام-ومحرك-الامتياز-في-محركات-البحث. |
