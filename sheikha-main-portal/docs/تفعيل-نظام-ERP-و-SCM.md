# تفعيل نظام ERP و SCM في سوق شيخه

## 1. نظام ERP (تخطيط موارد المؤسسة)

تم تفعيل **نظام ERP** وربطه ببوابة سوق شيخه.

### الحالة

| البند | الوصف |
|-------|--------|
| **الحالة** | مفعّل |
| **الاسم بالعربي** | نظام تخطيط موارد المؤسسة |
| **الاسم بالإنجليزي** | Enterprise Resource Planning (ERP) |
| **الإصدار** | 1.0 |

### الوحدات (Modules) المفعّلة

| الوحدة | الاسم بالعربي | الوصف | واجهة API |
|--------|---------------|--------|-----------|
| inventory | المخزون | عروض المنتجات والمخزون | GET /api/market-listings |
| sales | المبيعات | تسجيل وعرض البضائع للبيع | POST /api/listings، GET /api/my-products |
| procurement | المشتريات | عروض الشراء من السوق | GET /api/market-listings |
| parties | الأطراف والتجار | التجار والمتاجر | GET /api/traders، GET /api/stores |
| containers | الحاويات | إدارة حاويات السكراب | GET/POST/PUT/DELETE /api/containers |
| finance | المالية | لوحة التحكم والإحصائيات | GET /api/dashboard |

### واجهات API الخاصة بـ ERP

| الطريقة | المسار | الوظيفة |
|---------|--------|----------|
| GET | /api/erp/status | حالة نظام ERP وقائمة الوحدات المفعّلة |
| GET | /api/erp/modules | قائمة وحدات ERP ووصف كل وحدة |

## 2. نظام SCM (إدارة سلسلة التوريد)

تم تفعيل **نظام SCM** وربطه بسلسلة التوريد الحالية (مصدر → تجميع → نقل → مصهر → تصنيع → طرف نهائي).

### الحالة

| البند | الوصف |
|-------|--------|
| **الحالة** | مفعّل |
| **الاسم بالعربي** | نظام إدارة سلسلة التوريد |
| **الاسم بالإنجليزي** | Supply Chain Management (SCM) |
| **الإصدار** | 1.0 |

### مراحل سلسلة التوريد (SCM Flow)

1. **مصدر** (source) — مصدر السكراب (أفراد، مصانع، مناجم، تجار)
2. **تجميع** (collection) — جمع السكراب وفرزه
3. **نقل** (transport) — نقل السكراب
4. **مصهر** (smelter) — صهر وإعادة تدوير المعادن
5. **تصنيع** (manufacturing) — تصنيع أو إعادة تدوير
6. **طرف نهائي** (end) — تاجر تجزئة أو مستهلك

### واجهات API الخاصة بـ SCM

| الطريقة | المسار | الوظيفة |
|---------|--------|----------|
| GET | /api/scm/status | حالة نظام SCM وملخص العروض حسب المرحلة |
| GET | /api/scm/supply-chain | مراحل سلسلة التوريد وأنواع المصدر |
| GET | /api/scm/flow | تدفق العروض حسب كل مرحلة (مع تفاصيل العروض) |

### التكامل مع البيانات الحالية

- العروض (listings) تحتوي على حقل **supplyChainStage** يحدد مرحلة التوريد.
- واجهات السوق تدعم الفلترة حسب المرحلة: `/api/market-by-stage?stage=source`، `/api/market-listings?stage=collection`، إلخ.

---

## 3. حالة الأنظمة (/api/systems-status)

في واجهة **حالة الأنظمة** (`GET /api/systems-status`) يظهر الآن:

- **erp:** نظام ERP (تخطيط موارد المؤسسة) — مفعّل، API: /api/erp/status
- **scm:** نظام SCM (إدارة سلسلة التوريد) — مفعّل، API: /api/scm/status

---

## 4. ملخص

| النظام | الحالة | الواجهات الرئيسية |
|--------|--------|-------------------|
| **ERP** | مفعّل | /api/erp/status، /api/erp/modules |
| **SCM** | مفعّل | /api/scm/status، /api/scm/supply-chain، /api/scm/flow |
