# نظام اللوائح والقوانين والإجراءات الإدارية — تنفيذها وتطويرها والنماذج الإدارية والرقمية

## النطاق

نظام موحد لل**لوائح** و**القوانين** و**الإجراءات الإدارية** — تنفيذها، تطويرها، **النماذج الإدارية** و**النماذج الرقمية** (سواء نموذج كتابي أو تعبئته رقمياً أو توثيقه بأفضل الطرق الرقمية: نفاذ، أجهزة لوجستيات، تطبيقات الجوال لإدارة النقل وتتبع المواد، تطبيق للمستخدم السائق وشركات النقل)، مع **إنترنت الأشياء** و**نظام الذكاء التشغيلي والرقمي** وتكاملها مع **نظام المدن الذكية**. المراجع: تعريف-شيخه-الكامل-القوانين، نظام-المدن-الذكية-والتكامل.

# الجزء الأول — اللوائح والقوانين والإجراءات الإدارية

## 1.1 التعريف والهيكل

| المصطلح | التعريف |
|---------|---------|
| **اللوائح** | قواعد وإطار عام يضبط عمل المنصة (التسجيل، العرض، البيع، التكامل، الخصوصية، منع الهندسة العكسية) — قابلة للتحديث والمراجعة. |
| **القوانين** | قواعد ثابتة أو شبه ثابتة (قوانين إدارية أ-1 إلى أ-7، هندسية ه-1 إلى ه-6، تجارة ومنصة ت-1 إلى ت-5) — مُدرجة في تعريف شيخه الكامل. |
| **الإجراءات الإدارية** | خطوات تنفيذية محددة (كيف تسجّل تاجراً، كيف تضيف عرضاً، كيف تطلب حاوية، كيف توثّق صفقة) — قابلة للنماذج الكتابية أو الرقمية. |

## 1.2 وضع اللوائح والقوانين والإجراءات في المنظومة

| البند | الوصف |
|-------|--------|
| **مصدر اللوائح والقوانين** | وثيقة «تعريف شيخه الكامل — الخواص، القوانين، رحلة العميل»؛ وثيقة «الهندسة الكاملة لموقع شيخه وقوانينه»؛ وثيقة «نظام اللوائح والقوانين والإجراءات الإدارية» (هذه الوثيقة). |
| **مراجعة وتطوير** | مراجعة دورية (ربع سنوية أو سنوية) للوائح والإجراءات؛ تحديث الوثائق والنماذج؛ إبلاغ المستخدمين بالتغييرات الجوهرية. |
| **النشر** | اللوائح والقوانين والإجراءات متاحة في الموقع (صفحة اللوائح والقوانين والإجراءات؛ روابط للوثائق)؛ للمسجّلين: إشعار عند تحديث جوهري. |

# الجزء الثاني — النماذج الإدارية والنماذج الرقمية

## 2.1 النماذج الإدارية

| النموذج | الوصف | الشكل |
|---------|--------|--------|
| **تسجيل تاجر** | نموذج بيانات التاجر (اسم، جوال، بريد، منطقة، تخصص، اسم المتجر) | ورق أو PDF أو نموذج ويب (تسجيل-تاجر.html) |
| **عرض بضائع** | نموذج إضافة عرض (اسم البضاعة، كمية، وحدة، سعر، عملة، تاجر) | ورق أو PDF أو نموذج ويب (عرض-بضائع.html) |
| **طلب حاوية / نقل** | نموذج طلب حاوية أو خدمة نقل (نوع، موقع، تاريخ، جهة اتصال) | ورق أو PDF أو نموذج ويب (لوحة التحكم، /api/containers) |
| **توثيق صفقة** | نموذج توثيق اتفاق بيع/شراء (أطراف، بضاعة، كمية، سعر، تاريخ) | ورق أو PDF أو رقمي (عند التفعيل) |
| **طلب تكامل / API** | نموذج طلب مفتاح API أو تكامل (جهة، استخدام، صلاحيات مطلوبة) | ويب (صفحة التكامل) أو بريد |

## 2.2 النماذج الرقمية — أفضل الطرق الرقمية

| الطريقة | الوصف | التطبيق في شيخه |
|---------|--------|-------------------|
| **تعبئة رقمية على الموقع** | المستخدم يملأ النموذج من المتصفح (سطح المكتب أو الجوال) — البيانات تُرسل للخادم وتُخزّن. | تسجيل-تاجر.html، عرض-بضائع.html، لوحة التحكم، تسجيل-الدخول.html |
| **نفاذ (Nafath)** | توثيق الهوية أو التوقيع عبر تطبيق نفاذ — للتحقق من الشخص أو الموافقة. | ربط-نفاذ-وجوجل-ومايكروسوفت-وأبل-لتسجيل-الدخول؛ (لاحقاً) توثيق صفقات أو عقود بنفاذ |
| **أجهزة اللوجستيات** | أجهزة أو حساسات (وزن، موقع، درجة حرارة) ترسل بيانات تلقائياً — إنترنت الأشياء (IoT). | حاويات ذكية، تتبع شحن؛ تكامل مع /api/containers أو نظام نقل |
| **تطبيقات الجوال** | تطبيق للمستخدم (السائق) أو لشركات النقل — إدارة النقل، استقبال الطلبات، تتبع المواد، تحديث حالة الشحنة. | (مرحلة لاحقة) تطبيق سائق؛ تطبيق شركة نقل؛ تكامل مع Sheikha Partner API |
| **توثيق رقمي** | حفظ نسخة رقمية من النموذج أو العقد مع تاريخ وطرف — للرجوع والمراجعة. | لوحة التحكم → تقارير؛ تصدير CSV/JSON؛ (لاحقاً) توقيع إلكتروني أو نفاذ |

## 2.3 تطبيق السائق وشركات النقل

| البند | الوصف |
|-------|--------|
| **المستخدم السائق** | تطبيق جوال (أو واجهة ويب متجاوبة) يسمح للسائق: استقبال طلبات النقل، عرض تفاصيل الشحنة والموقع، تحديث حالة الرحلة (قيد التحميل، منقول، تم التسليم)، تتبع الموقع (إن وُجد). |
| **شركات النقل** | تطبيق أو لوحة تحكم لشركة النقل: إدارة الأسطول، تعيين السائقين، استقبال طلبات النقل من شيخه (عبر API أو منصة)، تقارير وتتبع. |
| **التكامل مع شيخه** | طلبات النقل من سوق شيخه تصل تلقائياً لشركة النقل أو السائق (عبر API أو إشعار)； تحديث الحالة يُعاد لشيخه — المشتري/البائع يرى حالة الشحنة. |
| **تتبع المواد** | رقم تتبع واحد للشحنة — من طلب النقل إلى التسليم؛ يظهر في لوحة تحكم المشتري/البائع و(لاحقاً) في تطبيق السائق. |

# الجزء الثالث — إنترنت الأشياء والذكاء التشغيلي والرقمي

## 3.1 إنترنت الأشياء (IoT)

| البند | الوصف |
|-------|--------|
| **الحاويات والأجهزة** | حاويات أو بوابات مزودة بحساسات (وزن، موقع GPS، رطوبة إن لزم) — بيانات تُرسل لخادم شيخه أو لنظام الشريك. |
| **تتبع المواد** | تتبع حركة الحاوية أو الشحنة دون تدخل يدوي كامل — تحديث الموقع والحالة تلقائياً. |
| **التكامل مع المدن الذكية** | (انظر نظام-المدن-الذكية-والتكامل) — بيانات التتبع أو الحركة قابلة للربط مع منصات المدن الذكية (مرور، لوجستيات، بيئة). |

## 3.2 نظام الذكاء التشغيلي والرقمي

| البند | الوصف |
|-------|--------|
| **الذكاء التشغيلي** | استخدام البيانات (عروض، طلبات، نقل، تتبع) لتحسين العمليات: اقتراح مسار نقل، تقدير تكلفة، توقع الطلب، تنبيه تأخير. |
| **الذكاء الرقمي** | ذكاء اصطناعي مدمج في المنصة: اقتراح أسعار (/api/ai/pricing-suggestion)، محادثة ذكية (مساعد شيخه)، توصيات «من قد يشتري منك» — قابل للتوسع. |
| **التكامل** | الذكاء التشغيلي والرقمي يتكامل مع اللوجستيات وتطبيق السائق وشركات النقل والمدن الذكية — بيانات موحدة، قرارات أو اقتراحات مبنية على البيانات. |

# الجزء الرابع — الإجراءات الإدارية (قائمة مرجعية)

| الرقم | الإجراء | النموذج / الواجهة | التوثيق الرقمي |
|-------|---------|-------------------|-----------------|
| إج-1 | تسجيل تاجر | تسجيل-تاجر.html؛ POST /api/traders/register | بيانات في data/traders.json؛ (لاحقاً) نفاذ للتحقق |
| إج-2 | إضافة عرض بضائع | عرض-بضائع.html؛ POST /api/listings | بيانات في data/listings.json |
| إج-3 | طلب حاوية / نقل | لوحة التحكم؛ POST /api/containers (أو نموذج طلب نقل) | سجل حاويات؛ (لاحقاً) تطبيق سائق |
| إج-4 | تسجيل الدخول | تسجيل-الدخول.html؛ نفاذ/جوجل/مايكروسوفت/أبل | JWT؛ OAuth |
| إج-5 | طلب مفتاح API / تكامل | صفحة التكامل؛ بريد أو نموذج | سجل مفاتيح API؛ صلاحيات محدودة |
| إج-6 | توثيق صفقة | (عند التفعيل) لوحة التحكم أو نموذج صفقة | سجل صفقات؛ (لاحقاً) توقيع نفاذ أو إلكتروني |
