# خطة تفعيل تكامل أنظمة الدولة — الأهم والأفضل

## الهدف
تفعيل تكامل حكومي نظامي عالي الاعتمادية لمنظومة شيخة، يبدأ بـ `National Address` ثم يتوسع إلى `واثق` و`اعتماد` وفق الصلاحيات الرسمية.

## ما تم تفعيله الآن
- إعداد باقات وحدود الاستخدام في `config/gov-api-plans.json`.
- إنشاء موصل جاهز لـ `National Address` في `lib/connectors/national-address-client.js`.
- إنشاء فحص جاهزية تشغيلي عبر:
  - `npm run ops:gov:readiness`
- إضافة متغيرات البيئة الرسمية في `.env.example`:
  - `NATIONAL_ADDRESS_*`
  - `WATHQ_*`
  - `ETIMAD_*`

## واجهات National Address المفعلة في الموصل
- `NationalAddress.Address`
- `NationalAddress.Lookups`
- `NationalAddress.Maps`
- `NationalAddress.NationalAddressByShortAddress`

## الباقات المعتمدة
- `development`: 100 نداء/شهر، نداء كل 5 ثوانٍ.
- `business`: 4,000,000 نداء/شهر، حتى 200 نداء/ثانية.
- `unlimited`: 6,000,000 نداء/شهر، حتى 400 نداء/ثانية.

## خطوات التشغيل الفعلي
1. نسخ `.env.example` إلى `.env`.
2. تعبئة قيم `NATIONAL_ADDRESS_BASE_URL` والمفاتيح الرسمية.
3. اختيار الباقة عبر `NATIONAL_ADDRESS_PLAN`.
4. تشغيل فحص الجاهزية:
   - `npm run ops:gov:readiness`
5. عند نجاح الفحص، ربط الموصل مع APIs الأعمال داخل `server.js` حسب تدفق التحقق/العقود/اللوجستيات.

## ضوابط شرعية ونظامية
- التكامل يتم فقط عبر مفاتيح مصرح بها من الجهة المالكة للبيانات.
- لا استخدام لأي طريقة تجاوز أو وصول غير مصرح.
- تطبيق مبدأ: لا ضرر ولا ضرار، مع توثيق جميع عمليات الاستعلام.
