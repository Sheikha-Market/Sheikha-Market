# التفعيل الآلي لخدمات الدولة — Government Auto Activation

## ما الذي تم تفعيله؟

تم إنشاء مسار تفعيل آلي يقرأ مفاتيح `.env` ويُفعّل الخدمات الجاهزة تلقائيًا:

- كتالوج الخدمات: `config/gov-services-catalog.json`
- سكربت التشغيل الآلي: `scripts/gov-auto-activation.js`
- ملف الحالة الناتج: `data/gov-services-state.json`

## الأوامر

- فحص آلي بدون تعديل:
```bash
npm run ops:gov:auto
```

- تفعيل آلي مع كتابة أعلام التفعيل إلى `.env`:
```bash
npm run ops:gov:auto:fix
```

## آلية القرار (لكل خدمة)

1. يتحقق من وجود المفاتيح المطلوبة (`requiredEnv`).
2. إذا كانت الخدمة تحتاج موافقة تنظيمية:
   - يقرأ المتغير: `<SERVICE_ID>_APPROVED`
3. النتيجة:
   - `active` إذا كانت المفاتيح موجودة + الموافقة متاحة
   - `pending_credentials` إذا المفاتيح ناقصة
   - `pending_approval` إذا المفاتيح موجودة لكن الموافقة غير مفعلة

## مثال سريع

لخدمة `YAQEEN`:
- مطلوب:
  - `YAQEEN_BASE_URL`
  - `YAQEEN_API_KEY`
- وموافقة:
  - `YAQEEN_APPROVED=true`

عند تحققها، ستصبح `YAQEEN_ENABLED=true` (في وضع auto:fix).

## ملاحظة امتثال

يتم التفعيل فقط عبر مفاتيح رسمية وتصاريح نظامية.
لا يوجد أي تجاوز أو محاولة وصول غير مصرح.
