# بسم الله الرحمن الرحيم
# رقمنة الإجراءات الكاملة لخدمات الدولة

## الهدف
تحويل إجراءات التكامل من خطوات يدوية متفرقة إلى مسار رقمي موحد قابل للقياس والتدقيق.

## ما تم إضافته
- تعريف سير الإجراءات: `config/gov-procedures-workflow.json`
- محرك رقمنة الإجراءات: `scripts/gov-procedures-digitalization.js`
- مخرجات الحالة الرقمية: `data/gov-procedures-state.json`

## الأوامر

```bash
# 1) تحديث حالة التفعيل الأساسية
npm run ops:gov:auto

# 2) رقمنة الإجراءات وحساب التقدم لكل خدمة
npm run ops:gov:procedures

# 3) نفس الخطوات مع كتابة تقدم الإجراء في .env
npm run ops:gov:procedures:fix
```

## مراحل الإجراء الرقمية
1. التأهيل النظامي
2. الإعداد التقني
3. اختبار Sandbox
4. الإطلاق التشغيلي
5. المراقبة والامتثال

## كيف تقرأ الحالة؟
- `procedureProgress`: نسبة الإنجاز لكل خدمة.
- `nextAction`: الإجراء التالي الموصى به آليًا.
- `stages[]`: حالة كل مرحلة (`pending | in_progress | done`).

## مبدأ الحوكمة
- التفعيل لا يكون إلا ببيانات وموافقات رسمية.
- كل خطوة موثقة رقميًا في ملفات الحالة.
- لا تجاوز، لا وصول غير مصرح، ولا ضرر ولا ضرار.
