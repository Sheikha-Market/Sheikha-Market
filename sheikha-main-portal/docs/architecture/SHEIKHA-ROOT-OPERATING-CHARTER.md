## 8) خارطة التحول التدريجي (من Monolith إلى Modular)

### Phase 1 — Stabilize Core (P0)
- تثبيت Runtime Core
- فصل الهوية والحوكمة كخدمات أساسية
- قياس مؤشرات الاستقرار الأساسية

### Phase 2 — Extract High-Impact Domains (P1)
- فصل السوق، التوريد، وإعادة التدوير كوحدات مستقلة
- توحيد عقود البيانات والأحداث

### Phase 3 — Operational Intelligence (P1/P2)
- ربط الفضاء والاتصالات بسير اللوجستيات
- تفعيل AI Routing وتحليلات التنبؤ

### Phase 4 — Global Control (P2)
- توحيد لوحة التحكم العالمية
- إدارة الأداء الجزئي والكلي من مركز واحد

## 9) واجهات التكامل الرسمية

### معايير API
- Versioned REST (`/api/v1/...`)
- Schema-first contracts (OpenAPI + JSON Schema)
- Idempotency للمسارات الحرجة

### معايير Events
- Event naming: `{domain}.{entity}.{action}.v1`
- Payload موحّد: `meta + data + compliance`
- تتبع إلزامي: `traceId`, `source`, `timestamp`

### معايير الحوكمة على العقود
- أي تغيير Contract = نسخة جديدة + خطة ترحيل + موافقة Architecture Board

## 10) مركز التحكم الموحد (Control Center)

### نطاق المراقبة
- الأسواق
- سلاسل الإمداد
- الاتصالات والفضاء
- الذكاء الصناعي
- الامتثال والحوكمة

### KPIs على مستويين
- **جزئي (Micro):** أداء سوق/مورد/ميناء/خدمة
- **كلي (Macro):** أداء المدن/الدول/الشبكة الشاملة

### لوحة القياس الموحدة (حد أدنى)
- Market Liquidity Index
- Supply Continuity Index
- Logistics On-Time Index
- Recycling Re-Entry Ratio
- Compliance Breach Rate
- AI Decision Utility Score
- Sustainability Impact Index
