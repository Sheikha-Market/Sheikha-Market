# بسم الله الرحمن الرحيم
# CHECKLIST طوارئ شيخة — بنية جاهزة لا تنهار

هذا الدليل لتشغيل المنظومة عند أي حالة طارئة، مع أولويات واضحة وسرعة قرار.

## 1) قبل الطوارئ (جاهزية دائمة)

- تشغيل فحص يومي: `scripts/sheikha-daily-ops-check.sh`
- تشغيل فحص الطوارئ: `scripts/sheikha-emergency-drill.sh`
- نسخ احتياطي يومي لمجلد `data/` + اختبار استرجاع أسبوعي
- حفظ المفاتيح الحساسة خارج الجهاز (Vault / Secret Manager)
- توفير مفتاح إداري بديل للطوارئ وتدويره شهريا
- توثيق جهات الاتصال (تقني، أمني، تشغيلي، قانوني)

## 2) عند بداية الطوارئ (أول 15 دقيقة)

- تثبيت الحالة: إيقاف أي نشر جديد
- جمع حالة فورية:
  - `curl -fsS http://localhost:8080/health`
  - `curl -fsS http://localhost:8080/api/comms/readiness`
  - `bash scripts/sheikha-emergency-drill.sh`
- تحديد نوع الحادث:
  - انقطاع خدمة
  - تسريب سرية/مفاتيح
  - ارتفاع أخطاء/بطء شديد
  - فساد بيانات
- إعلان حالة INCIDENT داخليًا

## 3) احتواء الحادث (Containment)

- تفعيل وضع تشغيل محافظ:
  - تعطيل المسارات غير الضرورية
  - تقييد العمليات عالية المخاطر
- في حال تسريب سرية:
  - تدوير جميع المفاتيح فورًا
  - إبطال JWT secrets القديمة
  - مراجعة سجلات الوصول
- في حال فساد بيانات:
  - إيقاف الكتابة المؤقت
  - استرجاع نسخة سليمة

## 4) استعادة التشغيل (Recovery)

- استعادة الخدمات الأساسية بالترتيب:
  1. `health`
  2. `auth`
  3. `comms`
  4. `billing`
- التحقق من الجاهزية:
  - `readinessScore`
  - صلاحية JWT الإداري
  - webhook key
- إعادة فتح الخدمات تدريجيًا وليس دفعة واحدة

## 5) بعد الحادث (Postmortem)

- تقرير خلال 24 ساعة:
  - السبب الجذري
  - زمن التوقف
  - البيانات المتأثرة
  - ما تم تدويره من أسرار
- خطة منع التكرار مع مواعيد تنفيذ واضحة

## 6) مستويات الحالة

- `READY`: لا أعطال حرجة
- `DEGRADED`: الخدمة تعمل مع مخاطر
- `INCIDENT`: توقف أو خطر أمني/بياني

## 7) أوامر تشغيل سريعة

```bash

bash /home/sheikha/Projects/sheikha/sheikha-main-portal/scripts/sheikha-daily-ops-check.sh

# فحص طوارئ
bash /home/sheikha/Projects/sheikha/sheikha-main-portal/scripts/sheikha-emergency-drill.sh
```
