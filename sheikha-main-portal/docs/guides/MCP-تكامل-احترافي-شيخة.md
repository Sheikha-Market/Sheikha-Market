# دليل تفعيل أفضل تكامل MCP في شيخة

هذا الدليل يفعّل بيئة MCP قوية وآمنة للمشروع بالكامل، وليس للملف المفتوح فقط.

## 1) الهدف

- فهرسة المشروع كاملًا وربط الملفات بذكاء.
- تنفيذ أدوات التطوير من داخل الوكيل مباشرة.
- تشغيل آمن بدون كشف الأسرار.

## 2) ملف القالب الجاهز

تم تجهيز ملف:

- `config/mcp-servers.template.json`

ويحتوي تكاملات أساسية:

- `sheikha-local`: خادم MCP المحلي الخاص بالمشروع.
- `github`: إدارة المستودعات والقضايا وطلبات الدمج.
- `fetch`: جلب محتوى الروابط والصفحات.
- `filesystem-sheikha`: وصول مقيد لمسار المشروع فقط.

## 3) خطوات التفعيل

1. انسخ محتوى `config/mcp-servers.template.json`.
2. ضع المحتوى داخل إعداد MCP في بيئتك (Cursor/عميل MCP).
3. بدّل `GITHUB_PERSONAL_ACCESS_TOKEN` بقيمة حقيقية من أسرارك المحلية فقط.
4. أعد تشغيل العميل.

## 4) ضوابط الأمان

- لا تضع أي مفتاح حقيقي داخل Git.
- احتفظ بالمفاتيح في `.env` المحلي فقط.
- فعّل أقل صلاحيات لازمة لكل تكامل.

## 5) أفضل إضافات VS Code المقترحة للمشروع

- `NVIDIA.nsight-vscode-edition`
- `ms-vscode.cpptools`
- `ms-python.python`
- `ms-vscode-remote.remote-ssh`
- `ms-vscode-remote.remote-containers`
- `eamodio.gitlens`
- `usernamehw.errorlens`
- `humao.rest-client`

## 6) ملاحظة تشغيلية مهمة

التكامل القوي لا يعني منح صلاحيات غير محدودة. الأفضل تشغيل تدريجي:

1. Local MCP + Fetch
2. GitHub
3. بقية الخدمات حسب الحاجة
