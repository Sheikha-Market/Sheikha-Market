# الجزء الأول — أنواع المستخدمين والواجهات الاحترافية

## 1.1 أنواع المستخدمين في سلاسل الإمداد (حسب الدور والتخصص)

| النوع | المعرف | الوصف | الواجهة والتقارير |
|-------|--------|--------|-------------------|
| **مصنع** | factory | مصنع أو منشأة إنتاج — مشتريات، إنتاج، لوجستيات | تقارير مصانع (مشتريات، مخزون، توريد، جودة)؛ داشبورد مصنع. |
| **إدارة المشتريات / مختص مشتريات** | procurement | قسم المشتريات أو مختص مشتريات في الشركة | تقارير مشتريات؛ عروض أسعار؛ أوامر شراء؛ مقارنات موردين. |
| **المخازن / مختص مخازن** | warehouse | قسم المخازن أو مختص مخازن — مخزون، جرد، تدفق مواد | تقارير مخزون؛ جرد؛ حركة مواد؛ تقارير مخازن. |
| **النقل البحري** | maritime | نقل بحري — شحن بحري، حاويات، موانئ | تقارير شحن بحري؛ موانئ؛ حاويات؛ مسافات بحرية. |
| **النقل الجوي** | air_freight | نقل جوي — شحن جوي | تقارير شحن جوي؛ أوزان؛ تواريخ. |
| **موانئ** | ports | إدارة موانئ أو تشغيل موانئ | تقارير موانئ؛ حركة بضائع؛ شحن وتفريغ. |
| **لوجستيات / نقل** | logistics | قسم اللوجستيات والنقل (عام) | تقارير نقل؛ شحنات؛ أوزان؛ مسافات؛ تكاليف. |
| **إداري / دوائر إدارية** | admin | إداري أو دوائر إدارية — إدارة عامة | داشبورد إداري؛ تقارير إدارية؛ مؤشرات. |
| **دوائر سوقية / تسويقية** | marketing | دوائر سوقية أو تسويقية | تقارير تسويق؛ مؤشرات سوق؛ ظهور وعروض. |
| **إدارة وهندسة في المجال** | engineering | إدارة مشاريع، هندسة، تصميم — في سلاسل الإمداد أو المعادن أو اللوجستيات | تقارير هندسية؛ مشاريع؛ مواصفات؛ تكامل مع أنظمة. |
| **مستشار (CISCC وغيره)** | consultant | مستشار سلاسل إمداد وتوريد — CISCC أو CSCP أو CIPS أو CPIM أو أي اعتماد/تخصص | واجهة تحليلية؛ تقارير استشارية؛ مؤشرات سلاسل التوريد؛ تغذية الذكاء (انظر: قاعدة-المعرفة-والذكاء-والاشتراكات-وأساسات-البيئة-المتكاملة.md). |
| **دكتوراه / باحث** | phd_researcher | باحث أو أكاديمي متخصص في سلاسل الإمداد أو المعادن | واجهة بيانات وتقارير بحثية؛ تصدير بيانات؛ تحليلات متقدمة. |
| **مالية** | finance | قسم المالية | تقارير مالية؛ التزامات؛ مدفوعات؛ تكلفة التوريد. |
| **إنتاج** | production | قسم الإنتاج | تقارير إنتاج؛ استهلاك مواد؛ توريد خام. |
| **جودة** | quality | قسم الجودة | تقارير جودة؛ مواصفات؛ HS Code؛ شهادات. |
| **تاجر / متجر** | trader | تاجر في سوق شيخه (متجر) | داشبورد تاجر؛ منتجاتي؛ تسويق؛ مؤشرات. |
| **قطاع حكومي / جهة حكومية** | government | وزارة، هيئة، مؤسسة حكومية | داشبورد القطاع الحكومي؛ مؤشرات سلاسل إمداد؛ تقارير مجمّعة؛ مؤشرات بيئية؛ دعم أهداف الدولة. |
| **شركة حكومية** | government_company | شركة مملوكة للدولة أو ذات صلة | داشبورد شركات الحكومية؛ مشتريات؛ مخازن؛ نقل؛ موانئ؛ تقارير احترافية؛ تكامل مع أهداف الدولة. |

المستخدم يمكن أن يكون مرتبطاً **بقسم واحد أو أكثر** (مثال: مشتريات + مخازن؛ نقل بحري + موانئ؛ إداري + دوائر سوقية). **كل ما يخص الإدارة والهندسة في نفس المجال** — سلاسل إمداد، معادن، سكراب، نقل، لوجستيات، مصاهر — مُشمَل في الأدوار والواجهات والتقارير.

## 1.2 الواجهات الاحترافية حسب المستخدم

| نوع المستخدم | ما يُعرض في الداشبورد | التقارير الاحترافية |
|--------------|------------------------|----------------------|
| **مصنع** | ملخص توريد، أوامر شراء، موردين، مخزون سكراب/معادن، تكاليف، جودة | تقرير مشتريات شهري؛ تقرير توريد حسب المورد؛ تقرير مخزون؛ تقرير جودة ومواصفات. |
| **مستشار (أي نوع)** | تحليلات سلاسل التوريد؛ مؤشرات أداء (KPI)؛ مقارنات أسواق؛ توصيات؛ استعلام على قاعدة المعرفة | تقرير تحليل سلسلة التوريد؛ تقرير مقارنة موردين؛ تقرير مخاطر توريد؛ تصدير لأغراض استشارية. |
| **دكتوراه / باحث** | بيانات مجمّعة (بدون تفاصيل خاصة)؛ رسوم بيانية؛ تصدير CSV/Excel | تقارير بحثية قابلة للتصدير؛ إحصائيات سوق؛ سلاسل توريد مجمّعة. |
| **إدارة المشتريات / مختص مشتريات** | عروض أسعار؛ أوامر شراء؛ موردين؛ مقارنات أسعار | تقرير مشتريات؛ تقرير عروض أسعار؛ تقرير التزامات مشتريات. |
| **المخازن / مختص مخازن** | مخزون؛ جرد؛ حركة مواد؛ مواقع تخزين | تقرير مخزون؛ تقرير جرد؛ تقرير حركة مخازن. |
| **النقل البحري / موانئ** | شحن بحري؛ حاويات؛ موانئ؛ حركة بضائع | تقرير شحن بحري؛ تقرير موانئ؛ تقرير حاويات. |
| **النقل الجوي** | شحن جوي؛ أوزان؛ تواريخ | تقرير شحن جوي. |
| **لوجستيات** | شحنات؛ أوزان؛ مسافات؛ تكاليف نقل؛ تواريخ تسليم | تقرير شحنات؛ تقرير تكاليف نقل؛ تقرير أوزان ووحدات. |
| **إداري / دوائر سوقية/تسويقية** | مؤشرات إدارية؛ تقارير سوقية؛ ظهور | تقارير إدارية؛ تقارير تسويقية. |
| **إدارة وهندسة** | مشاريع؛ مواصفات؛ تكامل أنظمة | تقارير هندسية؛ تقارير مشاريع. |
| **تاجر** | متجري؛ منتجاتي؛ إشعارات؛ تسويق؛ مؤشرات | تقرير مبيعات (عند التفعيل)； تقرير عروض ومنتجات. |

## 1.3 التقارير الاحترافية للمصانع (أمثلة)

| التقرير | المحتوى | التوقيت |
|---------|---------|---------|
| **تقرير مشتريات المصنع** | أوامر شراء، موردين، كميات، أسعار، تواريخ تسليم | شهري / ربع سنوي |
| **تقرير توريد حسب المورد** | أداء المورد، التزام بالمواصفات، التأخير، الجودة | شهري |
| **تقرير مخزون سكراب/معادن** | كميات حسب النوع والموقع؛ قيمة مخزون؛ دوران | أسبوعي / شهري |
| **تقرير جودة ومواصفات** | مطابقة HS Code؛ شهادات؛ عينات | حسب الطلب |
| **تقرير تكلفة التوريد** | تكلفة وحدة؛ تكلفة نقل؛ إجمالي التوريد | شهري |

# الجزء الثاني — الشركات والمستخدمين والصلاحيات

## 2.1 الشركة (Organization)

| البند | الوصف |
|--------|--------|
| **الشركة** | كيان واحد (مصنع، شركة تجارية، مؤسسة) — لها عدد محدود من المستخدمين حسب الاشتراك. |
| **عدد المستخدمين لكل شركة** | حد أقصى يُحدَّد حسب الباقة (مثلاً: 5، 10، 50، غير محدود) — يُطبَّق عند إضافة مستخدم جديد. |
| **ربط المستخدم بالشركة** | كل مستخدم يُربط بشركة واحدة؛ يمكن أن يكون له أدوار متعددة (مشتريات + لوجستيات). |

## 2.2 المستخدم والصلاحيات

| البند | الوصف |
|--------|--------|
| **المستخدم** | شخص مسجّل — له بريد وجوال وكلمة مرور؛ مرتبط بشركة؛ له دور أو أكثر. |
| **الصلاحيات (permissions)** | حسب الدور: عرض، إضافة، تعديل، حذف، تقارير، إعدادات، إدارة مستخدمين (للمسؤول). |
| **أدوار (roles)** | مسؤول الشركة، مشتريات، مختص مشتريات، مخازن، مختص مخازن، نقل بحري، نقل جوي، موانئ، لوجستيات، إداري، دوائر سوقية/تسويقية، إدارة وهندسة، مالية، إنتاج، جودة، مستشار، باحث، تاجر، **قطاع حكومي، شركة حكومية** — أو مخصص. |
| **قسم أو أكثر** | المستخدم يمكن أن يُعيَّن لأكثر من قسم (مثال: مشتريات + لوجستيات) — فيظهر له ما يخص كل قسم حسب الصلاحيات. |

## 2.3 مصفوفة صلاحيات (مثال)

| الصلاحية | مسؤول الشركة | مشتريات | لوجستيات | مستشار | تاجر |
|----------|---------------|----------|-----------|---------|------|
| عرض الداشبورد | ✓ | ✓ | ✓ | ✓ | ✓ |
| تقارير احترافية | ✓ | ✓ (مشتريات) | ✓ (لوجستيات) | ✓ | ✓ (منتجاتي) |
| أوامر شراء / عروض | ✓ | ✓ | — | عرض | — |
| إدارة مستخدمين الشركة | ✓ | — | — | — | — |
| إعدادات الشركة / التكامل | ✓ | — | — | — | — |
| منتجاتي / عروض المتجر | — | — | — | — | ✓ |

# الجزء الثالث — الداشبورد ونظام شيخه ERP مقابل تكامل ERP الشركة

## 3.1 الخياران

| الخيار | الوصف | التفعيل |
|--------|--------|---------|
| **الداشبورد ونظام شيخه ERP** | الشركة تستخدم داشبورد شيخه ونظام ERP شيخه (إدارة مشتريات، توريد، مخزون، تقارير) داخل المنصة. | اشتراك «شيخه ERP» — واجهات احترافية للداشبورد والتقارير حسب نوع المستخدم. |
| **تكامل مع ERP الشركة** | الشركة لديها نظام ERP خاص (SAP، Oracle، Microsoft، غيرها) — التكامل عبر Sheikha Partner API لمزامنة البيانات (عروض، أوامر، كميات، أسعار). | تفعيل التكامل — API Key، Endpoints؛ مزامنة ثنائية الاتجاه حسب العقد. |

## 3.2 ما يُفعَّل في كل خيار

| البند | شيخه ERP | تكامل ERP الشركة |
|--------|-----------|-------------------|
| الداشبورد | واجهات احترافية كاملة حسب المستخدم (مصنع، مستشار، مشتريات، إلخ) | عرض ملخص مزامنة؛ سجلات تكامل؛ تنبيهات. |
| التقارير | تقارير احترافية للمصانع والمشتريات واللوجستيات والمستشارين | تصدير بيانات مزامنة؛ تقارير حسب ما يرد من ERP. |
| المستخدمون والصلاحيات | إدارة مستخدمين الشركة؛ صلاحيات حسب الدور والقسم | نفس الصلاحيات أو ربط مع أدوار ERP إن وُجد. |
| عدد المستخدمين | حد حسب الباقة (5، 10، 50، غير محدود) | نفس الحد أو حسب العقد. |

# الجزء الرابع — التطبيق في الواجهات (احترافية فخمة)

## 4.1 مبادئ الواجهة الاحترافية

| المبدأ | التطبيق |
|--------|---------|
| **وضوح حسب المستخدم** | الداشبورد يعرض فقط ما يخص دور المستخدم وأقسامه — بدون ضوضاء. |
| **تقارير احترافية** | تقارير قابلة للطباعة والتصدير؛ جداول منسقة؛ تواريخ ومراجع واضحة. |
| **مظهر فخم واحترافي** | ألوان متناسقة (أخضر/ذهبي/أغمق)； خطوط واضحة؛ ظلال خفيفة؛ تباعد مريح؛ بطاقات منظمة. |
| **دعم CISCC والمستشارين** | لغة ومصطلحات سلاسل إمداد وتوريد؛ مؤشرات KPI؛ تحليلات واضحة. |
| **دعم المصانع** | تقارير مشتريات، توريد، مخزون، جودة — بصيغة احترافية. |

## 4.2 الداشبورد الاحترافي

- **شريط علوي (header):** شعار شيخه، اسم الشركة، اسم المستخدم، الدور (مثل: مستشار CISCC، إدارة المشتريات)، خيار الخروج.
- **شريط جانبي (sidebar):** أقسام واضحة — متجري، إشعارات، منتجاتي، إضافة منتج، تسويق، مؤشرات، تقارير (للمصانع/المستشارين)، إعدادات، خريطة الموقع، خروج.
- **المحتوى الرئيسي:** بطاقات (cards) منظمة؛ عناوين واضحة؛ جداول منسقة؛ أزرار ذات ألوان متناسقة.

## 4.3 الموقع الرئيسي الاحترافي

- **هيدر:** شريط علوي أنيق؛ شعار؛ تنقل واضح؛ أزرار تسجيل متجر / عرض بضائع / تسجيل الدخول / لوحة التحكم.
- **أقسام الصفحة الرئيسية:** أهداف الموقع؛ دخول السوق؛ دورة التدوير؛ لماذا شيخه؛ المتاجر والعروض — بتصميم متناسق واحترافي.

**ملخص:** نظام صلاحيات ومستخدمين وشركات؛ واجهات وتقارير احترافية حسب المستخدم (مصنع، مستشار بأي اعتماد، دكتوراه/باحث، مشتريات، أقسام أخرى؛ جامعات وباحثون ومتخصصون)؛ حد مستخدمين لكل شركة؛ خيار الداشبورد ونظام شيخه ERP أو تكامل مع ERP الشركة؛ اشتراكات جامعات وباحثين ومتخصصين؛ قاعدة معرفة وذكاء الأفضل بالكون وتغذية الذكاء وأساسات البيئة المتكاملة — واجهات فخمة واحترافية للداشبورد والموقع.
