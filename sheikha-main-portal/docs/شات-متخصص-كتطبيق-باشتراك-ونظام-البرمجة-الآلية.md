# شات متخصص كتطبيق باشتراك — ونظام البرمجة الآلية

## النطاق

- **شات متخصص** — تطبيق (App) يعمل باشتراك؛ بأفضل التقنيات؛ قابل للدمج بأي بيئة.
- **نظام البرمجة الآلية** — تفعيل الأكواد وإنتاج السكربتات للبرمجة عند التطوير — بأفضل التقنيات.
- **التكامل** — إمكانية دمج الشات المتخصص بأي بيئة (سوق شيخه، ERP، منصات، أنظمة خارجية).

---

# الجزء الأول — الشات المتخصص كتطبيق باشتراك

## 1.1 التعريف

| البند | الوصف |
|--------|--------|
| **شات متخصص** | تطبيق محادثة (Chat) متخصص في مجال سلاسل الإمداد والتوريد، المعادن والسكراب، اللوجستيات، المصاهر، النقل، الموانئ، التجارة الرقمية، والمواضيع ذات العلاقة. |
| **كتطبيق** | يعمل كتطبيق مستقل (ويب، موبايل، أو مدمج في منصة) — واجهة محادثة مع ذكاء اصطناعي متخصص. |
| **باشتراك** | الاشتراك يُحدد مستوى الوصول: عدد الاستعلامات، عمق التحليل، تصدير سكربتات، تكامل مع بيئات — حسب الباقة. |

## 1.2 أفضل التقنيات

| المكون | الوصف |
|--------|--------|
| **ذكاء اصطناعي و LLM** | نماذج لغة كبيرة (LLM) متخصصة أو مُحسَّنة لسياق سلاسل الإمداد والمعادن والسكراب واللوجستيات؛ مع قاعدة معرفة وغذاء مستمر. |
| **قاعدة المعرفة** | قاعدة بيانات الأفضل بالكون في التخصص — مصطلحات، أوصاف، استعلامات، بحوث — لتغذية الشات. |
| **سياق المستخدم** | حسب دور المستخدم (مستشار، مصنع، مشتريات، حكومة، باحث) — إجابات وتوصيات مخصّصة. |
| **أمان وخصوصية** | تشفير، عدم تخزين بيانات سرية دون موافقة؛ ضوابط الشريعة والمحتوى الحلال. |

## 1.3 الدمج بأي بيئة

| البند | الوصف |
|--------|--------|
| **دمج بأي بيئة** | الشات المتخصص يمكن دمجه بأي بيئة — سوق شيخه، داشبورد تاجر/مصنع/حكومي، ERP، منصات طرف ثالث، مواقع، تطبيقات. |
| **واجهات التكامل** | APIs (REST، WebSocket)؛ مكوّنات جاهزة (Widget، iframe، SDK) للدمج في الويب أو التطبيقات. |
| **Single Sign-On** | إمكانية ربط الدخول بحساب المستخدم في المنصة (نفاذ، حسابات شيخه، حسابات الشركة). |

# الجزء الثاني — نظام البرمجة الآلية وتفعيل الأكواد وإنتاج السكربتات

## 2.1 التعريف

| البند | الوصف |
|--------|--------|
| **نظام البرمجة الآلية** | نظام يسمح بتحويل توصيات الشات أو أوامر المستخدم إلى **أكواد قابلة للتنفيذ** و**سكربتات برمجية** — لأتمتة المهام، التكامل، أو التطوير. |
| **تفعيل الأكواد** | إمكانية تفعيل (تنفيذ) الأكواد أو السكربتات المُنتَجة في بيئة آمنة — أو تصديرها للتشغيل في بيئة المستخدم. |
| **إنتاج السكربتات** | الشات أو النظام ينتج سكربتات جاهزة (Python، JavaScript، SQL، إلخ) حسب طلب المستخدم — لأتمتة تقارير، تكامل APIs، معالجة بيانات. |

## 2.2 أفضل التقنيات للتطوير

| المكون | الوصف |
|--------|--------|
| **لغات وبيئات** | دعم أفضل التقنيات — Python، JavaScript/TypeScript، Node.js، SQL، وأدوات CI/CD، حزم معيارية — انظر: نظام-شيخه-للغات-البرمجة.md. |
| **قوالب سكربتات** | قوالب جاهزة لأغراض شائعة: جلب بيانات من API، تصدير تقارير، مزامنة مع ERP، تحليل بيانات. |
| **تنفيذ آمن** | عند تفعيل الأكواد داخل المنصة — بيئة رملية (sandbox)، حدود زمنية وموارد، عدم الوصول لبيانات غير مصرح بها. |
| **تصدير واستخدام خارجي** | المستخدم يمكنه تصدير السكربت المُنتَج واستخدامه في بيئته (خادمه، ERP، أداة تحليل). |

# الجزء الثالث — التفعيل والاشتراكات

## 3.1 مستويات الاشتراك (أمثلة)

| المستوى | ما يُقدَّم |
|---------|------------|
| **أساسي** | عدد محدود من المحادثات/الاستعلامات؛ إجابات عامة؛ بدون تصدير سكربتات. |
| **متقدم** | استعلامات أكثر؛ سياق حسب الدور (مستشار، مصنع، مشتريات)؛ تصدير سكربتات قوالب. |
| **احترافي** | استعلامات غير محدودة (أو حد مرتفع)؛ تفعيل أكواد في بيئة آمنة؛ إنتاج سكربتات مخصصة؛ دمج بأي بيئة (APIs، Widget). |
| **شركات / حكومي** | نفس الاحترافي + تكامل مع بيئة الشركة أو الجهة الحكومية؛ SSO؛ تقارير استخدام. |

## 3.2 التفعيل في المنصة

| العنصر | الحالة / الإجراء |
|--------|-------------------|
| **الشات المتخصص** | مُفعّل — كصفحة/تطبيق (مساعد شيخه)؛ التوسع ليكون تطبيقاً مستقلاً باشتراك عند الجاهزية. |
| **نظام البرمجة الآلية** | مُفعّل — إنتاج سكربتات وقوالب؛ تفعيل أكواد في بيئة آمنة عند التوفر. |
| **الدمج بأي بيئة** | عبر APIs ومكوّنات؛ توثيق للتكامل مع سوق شيخه، ERP، منصات، النظام السعودي. |
