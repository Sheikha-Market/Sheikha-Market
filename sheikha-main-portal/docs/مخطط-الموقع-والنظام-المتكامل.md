# مخطط الموقع والنظام المتكامل — شيخه

## 1. أهداف الموقع (ما يظهر للزائر)

| # | الهدف | الوصف الموجز |
|---|--------|----------------|
| 1 | **بلا ضرر ولا ضرار** | لا إضرار بالبيئة أو الصحة أو البشر أو المخلوقات. |
| 2 | **حب العلامة وأمان** | العلامة = أمان وجودة؛ لا استغلال ولا خداع. |
| 3 | **محافظة على البيئة والصحة والبشر** | إعادة تدوير المعادن والسكراب للمحافظة — وقاية وجودة. |

هذه الثلاثة تُعرَض في الصفحة الرئيسية (قسم واضح) وفي الفوتر؛ باقي الأهداف تعمل في الخلفية (سياسات، تصميم، تشغيل).

## 2. مخطط الموقع — الصفحات الأساسية مقابل الخلفية

### 2.1 الصفحات الأساسية (في واجهة الموقع — القائمة الرئيسية والفوتر)

يُقصد بها: ما يراه المستخدم أولاً ويحتاجه للتعامل مع السوق.

| الصفحة | المسار | الغرض |
|--------|--------|--------|
| الرئيسية | `index.html` | البوابة، الأهداف، دخول السوق، المتاجر والعروض. |
| تسجيل متجر | `تسجيل-تاجر.html` | فتح متجر في السوق. |
| عرض بضائع | `عرض-بضائع.html` | إضافة عروض للمتاجر. |
| تسجيل الدخول | `تسجيل-الدخول.html` | دخول التاجر/المستخدم. |
| لوحة التحكم | `لوحة-التحكم.html` | إدارة المتجر والمنتجات والإعدادات (بعد الدخول). |
| خريطة الموقع | `خريطة-الموقع.html` | روابط كل الصفحات — أساسية وداعمة. |

**التنقل الرئيسي يُقتصر على:** الرئيسية · المتاجر والعروض · تسجيل متجر · عرض بضائع · تسجيل الدخول / لوحة التحكم · خريطة الموقع.

### 2.2 الصفحات الداعمة (في الخلفية — من خريطة الموقع والفوتر فقط)

لا تظهر في الشريط العلوي الرئيسي؛ تُوصَل من الفوتر أو من صفحة «خريطة الموقع».

| الصفحة | المسار | الغرض |
|--------|--------|--------|
| الهوية والبيئة والجودة | `الهوية-والبيئة-والجودة.html` | الهوية، الأهداف الأخلاقية، البيئة. |
| اللوائح والقوانين والإجراءات | `اللوائح-والقوانين-والاجراءات.html` | الإطار القانوني والإجرائي. |
| آلية البيع | `آلية-البيع.html` | كيف تتم عملية البيع. |
| تكامل | `تكامل.html` | التكامل مع الأنظمة والشركات. |
| مساعد شيخه | `مساعد-شيخه.html` | المساعد الذكي. |
| سلسلة التوريد (رسم) | `سلسلة-التوريد-رسم.html` | رسم سلسلة التوريد. |
| تجربة العميل | `تجربة-العميل.html` | رحلة العميل. |
| تحليل العملية التجارية | `تحليل-العملية-التجارية.html` | تحليل العمليات. |
| نظام المتابعة والتحليل | `نظام-متابعة-وتنفيذ-وتحليل.html` | المتابعة والتحليل. |
| تجربة منظومة البرمجة | `تجربة-منظومة-البرمجة.html` | تجربة المنظومة التقنية. |
| تخطيط كل مرحلة | `تخطيط-كل-مرحلة.html` | تخطيط المراحل. |
| تفعيل النموذج | `تفعيل-النموذج.html` | تفعيل النماذج. |
| رسم سلسلة التوريد | `رسم-سلسلة-التوريد.html` | رسم السلسلة. |
| رسومات هندسية وفنية | `رسومات-هندسية-وفنية.html` | رسومات. |
| ريادة الأعمال | `ريادة-الأعمال.html` | ريادة الأعمال. |
| سوق سلسلة التوريد | `سوق-سلسلة-التوريد.html` | السوق وسلسلة التوريد. |
| نظام البرمجة | `نظام-البرمجة.html` | نظام البرمجة. |
| تطبيقات (مستخدم/تاجر/أدمن) | `user-app.html`, `scrap-specialist.html`, `admin-app.html` | تطبيقات متقدمة. |

## 3. النظام المتكامل — المكونات والعلاقات

```
┌─────────────────────────────────────────────────────────────────┐
│                    واجهة المستخدم (الموقع)                        │
│  الرئيسية · تسجيل متجر · عرض بضائع · لوحة التحكم · خريطة الموقع   │
└───────────────────────────┬─────────────────────────────────────┘
                            │
┌───────────────────────────▼─────────────────────────────────────┐
│                     الخادم (Node.js)                             │
│  API · تسجيل/دخول · متاجر · عروض · أسعار · مؤشرات · عمليات       │
└───────────────────────────┬─────────────────────────────────────┘
                            │
┌───────────────────────────▼─────────────────────────────────────┐
│                     البيانات والأرشفة                            │
│  data/ · عمليات · أرشفة ضرورية · سحابة (عند التفعيل)             │
└───────────────────────────┬─────────────────────────────────────┘
                            │
┌───────────────────────────▼─────────────────────────────────────┐
│                     الذكاء والتكامل (مستقبلاً)                    │
│  LLM · تكامل ERP/سحابة · مؤشرات تسويق وأداء                      │
└─────────────────────────────────────────────────────────────────┘
```

| الطبقة | المكونات | الوظيفة |
|--------|-----------|---------|
| **الواجهة** | HTML/CSS/JS، صفحات أساسية + خريطة موقع | عرض المحتوى، تسجيل، عرض بضائع، لوحة تحكم. |
| **الخادم** | Express، WebSocket، JWT، APIs | مصادقة، متاجر، عروض، أسعار، تسجيل عمليات. |
| **البيانات** | ملفات JSON، مجلد عمليات، أرشفة | تخزين محلي، سجلات، دفع سحابي عند التفعيل. |
| **التكامل** | مؤشرات، حفظ سحابي، تكامل خارجي | قياس أداء، أرشفة، ربط أنظمة. |

## 4. مبدأ العرض — لا رموز غامضة

- **الخطوات (دخول السوق):** أرقام واضحة (1، 2، 3) مع نص قصير (تسجيل، عرض، تواصل).
- **لماذا شيخه:** عناوين واضحة مع جملة واحدة تحت كل عنوان؛ دون رموز هندسية غير مفهومة.
- **لوحة التحكم:** أرقام أو تسميات قصيرة (متجري، إشعارات، منتجاتي، …) بدون رموز غير معيارية.

- صفحة **خريطة الموقع** (`خريطة-الموقع.html`) تعكس هذا التقسيم (أساسي / داعم) وتُربط بكل الصفحات.

- **مخطط لوحة التحكم والموقع والتطبيقات والتكامل وأماكن النشر:** انظر: مخطط-لوحة-التحكم-والموقع-والتطبيقات-والتكامل-وأماكن-النشر.md — لوحة التحكم الرئيسية (الأدمن)، لوحات المستخدمين، مخطط الموقع (ملخص + مرجع لهذا المستند)، التطبيقات الإلكترونية (أجهزة وأنظمة تشغيل)، أماكن النشر (App Store، Google Play، Microsoft Store، ويب/PWA)، مخطط ERP/SCM والتكامل وأماكن ظهورها.
