# 💰 آلية تحقيق الأرباح التجارية + رقمنتها (حلال)
## منظومة وسوق شيخة — 2026

> **مبدأ شرعي حاكم:** الربح مقابل **منفعة مباحة** أو **جهد معلوم** أو **مخاطرة مشروعة**.  
> ممنوع: الربا، الغرر الفاحش، الغش، بيع ما لا يملك.

---

## 1) مصادر الربح (Revenue Streams)

### 1.1 عمولة الوساطة (Marketplace Commission) — **حلال**
- **الوصف:** أجر وساطة/سمسرة على إتمام صفقة داخل السوق.
- **الشرط:** معلومة مسبقاً (نسبة أو مبلغ ثابت) + تظهر في الفاتورة.
- **الرقمنة:** تُحتسب تلقائياً عند انتقال الطلب إلى “مكتمل”.

### 1.2 اشتراكات التجار (Subscriptions) — **حلال**
- **الوصف:** باقات شهرية مقابل مزايا (لوحة تحكم، تحليلات، أدوات تسويق، API).
- **الشرط:** منفعة واضحة + إمكانية الإلغاء + عدم ربطها بقرض/فائدة.
- **الرقمنة:** فواتير دورية + تفعيل/تعطيل تلقائي للمزايا.

### 1.3 رسوم الخدمات التشغيلية (Service Fees) — **حلال**
أمثلة:
- **رسوم توثيق/فحص جودة** (مختبرات/أجهزة)  
- **رسوم تخزين** (بالطن/اليوم/الشهر)  
- **رسوم مناولة/تحميل/تفريغ**  
- **رسوم تجهيز حاوية**  
- **رسوم تخليص جمركي**  

**الرقمنة:** كل خدمة تتحول إلى “أمر خدمة” مرتبط بالطلب + تسعير آلي + مستندات.

### 1.4 خدمات المؤسسات (Enterprise) — **حلال**
- **الوصف:** عقود B2B مع مصانع/شركات كبيرة (SLA + تكاملات ERP).
- **الرقمنة:** فواتير شهرية + قياس SLA + تقارير امتثال.

## 2) آلية توزيع المنفعة (Profit Sharing) — بلا ضرر ولا ضرار

### 2.1 قواعد عامة
- لا تُخصم عمولة/رسوم **إلا** بعد وضوحها وقبول الأطراف.
- عند النزاع: تجميد المبلغ محل النزاع مؤقتاً (إجراء إداري) + لجنة نزاعات + توثيق.

### 2.2 نموذج محاسبي رقمي (Ledger)
لكل عملية مالية في المنظومة سجل (Ledger Entry):
```yaml
transaction_id: TX-2026-0001
order_id: ORD-2026-0101
type: commission | subscription | service_fee | payout | refund
amount: 0
currency: SAR
counterparty: buyer | seller | carrier | warehouse | lab | sheikha
status: pending | settled | disputed | refunded
sharia_flags:
  - none
documents:
  - invoice_id
  - delivery_proof
  - inspection_report
timestamps:
  created_at:
  settled_at:
```

## 3) رقمنة الأرباح: “محرك الأرباح” (Profit Engine)

### 3.1 مكونات المحرك
1) **Pricing Engine**: قواعد تسعير عمولة/خدمات/اشتراكات  
2) **Commission Engine**: احتساب العمولة تلقائياً وفق الباقة  
3) **Billing Engine**: إصدار فواتير (خدمات/اشتراكات)  
4) **Payout Engine**: تحويل مستحقات الأطراف بعد الإغلاق  
5) **Refund Engine**: استرداد/تعويض وفق سياسة واضحة  
6) **Audit Engine**: تتبع وتدقيق + تقارير + منع الغش  

### 3.2 قواعد احتساب نموذجية (قابلة للتعديل)
```yaml
commission:
  free: 0.05
  basic: 0.03
  pro: 0.02
service_fees:
  storage_per_ton_month: 45-120
  inspection_per_batch: 500-3000
  customs_per_shipment: 1000-3000
subscriptions:
  free: 0
  basic: 500
  pro: 1500
  enterprise: contract
```

## 4) لوحات قياس الأرباح (Dashboards)

### 4.1 مؤشرات يومية
- GMV (إجمالي قيمة التداول)
- صافي العمولة
- صافي رسوم الخدمات
- عدد الطلبات المكتملة
- معدل الاسترداد/النزاعات

### 4.2 مؤشرات شهرية
- MRR (إيراد اشتراكات)
- CAC/LTV (لتسويق التجار)
- هامش الخدمة لكل خدمة
- التزام شرعي (0 مخالفات)

---

## 5) ضوابط شرعية رقمية (Sharia-by-Design)

### 5.1 قواعد آلية تمنع الشبهات
- منع أي “فوائد/غرامات ربوية” في السداد.
- عقود واضحة: الثمن/المثمن/التسليم/الضمانات/التحكيم.
- منع بيع ما لا يملك: لا يتم نشر منتج إلا مع “دليل ملكية/حيازة” أو عقد توريد مشروع.
- توثيق وزن/صور/تسليم لتقليل الغرر والنزاع.

## 6) مسار تنفيذ سريع (14 يوم)

1) تعريف جداول التسعير (عمولة/اشتراك/خدمة) كملف إعدادات موحد

3) ربط الفاتورة/المستندات بالعملية  
4) إنشاء تقارير يومية/شهرية (حتى لو CSV كبداية)  
5) تدقيق شرعي على نصوص الشروط والرسوم
