حماية تامة للملكية الفكرية، وتوثيق وتشفير للمعلومات، للأمن والسريّة والأمان — وفق الكتاب والسنة.

> «وَأَدِّ الأَمَانَةَ إِلَى مَنِ ائْتَمَنَكَ» — الأنفال:٢٧  
> «وَلَا تَجَسَّسُوا» — الحجرات:١٢

## ٢. حماية الملكية الفكرية

### ٢.١ ما يُحمى

| النوع | الوصف | الإجراء |
|------|-------|---------|
| **الكود المصدري** | server.js، lib/، config/ | لا نشر بدون إذن المالك |
| **البيانات** | users، traders، listings | تشفير + صلاحيات |
| **التصاميم** | واجهات، شعارات، أصول | علامة مائية + توثيق |
| **الوثائق** | سياسات، عقود، تقارير | hash + timestamp |
| **الأسرار** | JWT، API keys | .env فقط — لا Git |

### ٢.٢ إشعار الملكية

كل مخرجات المنظومة تحمل إشعاراً:

```
© 2026 سلمان أحمد بن سلمان الراجح — SHEIKHA
جميع الحقوق محفوظة — لا تعديل ولا نسخ بدون إذن صريح
```

### ٢.٣ التوثيق

- كل وثيقة: hash (SHA-256) + timestamp
- كل معاملة: سجل لا يُحذف
- كل وصول حساس: يُسجّل (من، متى، ماذا)

---

## ٣. التشفير والأمان

### ٣.١ التشفير عند التخزين (At Rest)

| البيانات | الخوارزمية | الملاحظة |
|----------|------------|----------|
| كلمات المرور | SHA-256 + salt | لا تُخزّن نصوصاً |
| بيانات حساسة | AES-256-GCM | مفتاح من .env |
| JWT | HS256 | مفتاح ≥ 32 حرف |

### ٣.٢ التشفير عند النقل (In Transit)

- TLS 1.3 على الإنتاج
- HTTPS إلزامي للواجهات
- لا إرسال أسرار في query params

### ٣.٣ المتغيرات المطلوبة

```bash
JWT_SECRET=          # ≥ 32 حرف
PASSWORD_SALT=      # فريد لكل بيئة
SHEIKHA_ENCRYPTION_KEY=  # 32 بايت لـ AES-256 (اختياري للبيانات الإضافية)
```

## ٤. السرية والخصوصية

### ٤.١ تصنيف البيانات

| المستوى | أمثلة | الوصول |
|---------|-------|--------|
| **Critical** | JWT_SECRET، API keys | المالك فقط |
| **High** | بيانات المستخدمين، عقود | صلاحيات محدودة |
| **Internal** | إعدادات تشغيل | فريق التشغيل |
| **Public** | واجهة السوق، معلومات عامة | الجميع |

### ٤.٢ قواعد

- لا طباعة أسرار في logs
- لا مشاركة مفاتيح عبر قنوات غير آمنة
- أقل صلاحية ممكنة لكل مفتاح
- تدوير الأسرار الحرجة كل ٣٠ يوماً

---

## ٥. التوثيق والتدقيق

### ٥.١ سجل الوصول

كل وصول لبيانات حساسة يُسجّل:

- المستخدم/الدور
- الوقت
- العملية
- النتيجة (نجاح/فشل)

### ٥.٢ hash الوثائق

- كل وثيقة مُصدّقة: hash + timestamp
- التحقق: إعادة حساب hash ومقارنته

---

## ٦. بلا ضرر ولا ضرار

- لا حذف بيانات دون نسخة احتياطية
- لا تغيير منطق تشغيلي دون اختبار
- لا كشف أسرار في أي مخرجات
- استجابة آمنة عند الأخطاء (لا تسريب معلومات)
