# نظام معرفة الكميات الكاملة — حسب المتجر والنوع والمنطقة الجغرافية

## النطاق

- **معرفة الكميات كاملة** — من المعادن والسكراب في **كل متجر**، و**المجموع لكل نوع معين من السكراب** له نفس الخواص (نفس التصنيف، مرحلة، نوع عرض).
- **الكميات حسب المنطقة الجغرافية** — لتنظيم عمليات البيع حسب المنطقة الجغرافية آلياً.
- **وصف القيمة** — بالسعر، العدد، الوزن، عدد الوحدات، وكل وصف ذي علاقة.
- **تنظيم عمليات البيع حسب المنطقة آلياً** — وتنظيم عمليات البيع — كل ما ينظم عمليات البيع آلياً.

---

# الجزء الأول — الكميات في كل متجر

## 1.1 ما يُقاس لكل متجر

| البند | الوصف |
|-------|--------|
| **المتجر (التاجر)** | معرف التاجر، اسم المتجر، المنطقة، التخصص. |
| **عدد العروض** | عدد قوائم البضائع (عروض) في هذا المتجر. |
| **الكميات حسب النوع** | لكل نوع سكراب/معدن (نفس الخواص: category/hsCode، supplyChainStage، scrapType): كمية، وحدة، وزن، سعر، عملة، عدد وحدات (عدد العروض من هذا النوع). |
| **المجموع الكلي للمتجر** | إجمالي الكمية (إن أمكن تجميع وحدات متجانسة)، إجمالي الوزن، إجمالي القيمة (سعر × كمية حيث ممكن)، عدد العروض. |

## 1.2 وصف القيمة (لكل متجر وعرض)

| الوصف | الوحدة / المعنى |
|-------|------------------|
| **السعر** | سعر الوحدة (ريال/طن، دولار/كيلو، إلخ). |
| **العدد** | عدد العروض أو عدد الوحدات (قطع، متر، إلخ). |
| **الوزن** | الوزن (طن، كيلو) — إن وُجد في العرض. |
| **عدد الوحدات** | كمية معبّرة بعدد وحدات (طن، كيلو، قطعة، متر). |
| **القيمة النقدية** | سعر × كمية (إن كانت الكمية رقمية) — تقدير قيمة العرض. |
| **كل وصف ذو علاقة** | وصف البضاعة، نوع السكراب، مرحلة السلسلة، نوع العرض (تجميع، صهر، نقل، شاحنات). |

# الجزء الثاني — المجموع لكل نوع معين من السكراب (نفس الخواص)

## 2.1 تعريف «نوع معين — نفس الخواص»

| الخواص | الوصف |
|--------|--------|
| **التصنيف (category / hsCode)** | نفس فئة المعادن/السكراب (حديد، نحاس، ألمنيوم، ستانلس، مختلط، إلخ) — أو نفس HS Code. |
| **مرحلة السلسلة (supplyChainStage)** | نفس المرحلة (مصدر، تجميع، نقل، مصهر، تصنيع، طرف نهائي). |
| **نوع عرض السكراب (scrapType)** | نفس نوع العرض (تجميع، صهر، نقل، شاحنات). |

العروض التي تتشارك **نفس التصنيف + نفس المرحلة + نفس نوع العرض** تُجمَع في **نوع واحد** — ويُحسب لها **مجموع الكميات، الوزن، القيمة، عدد العروض، عدد المتاجر**.

## 2.2 ما يُقاس لكل نوع (نفس الخواص)

| البند | الوصف |
|-------|--------|
| **النوع** | category/hsCode، supplyChainStage، scrapType (واسم النوع بالعربي). |
| **عدد العروض** | عدد العروض من هذا النوع في السوق. |
| **عدد المتاجر** | عدد المتاجر (تجار) التي لديها عرض من هذا النوع. |
| **مجموع الكمية** | مجموع الكميات (إن أمكن تجميع وحدات متجانسة — مثلاً كلها طن)؛ أو توزيع حسب الوحدة (طن، كيلو، قطعة). |
| **مجموع الوزن** | مجموع الأوزان المعرّفة في العروض (إن وُجدت). |
| **وصف القيمة** | متوسط السعر أو نطاق السعر؛ إجمالي القيمة التقديرية (سعر × كمية حيث ممكن)； عدد الوحدات. |
| **حسب المنطقة** | نفس النوع مُجزّأ حسب منطقة المتجر (storeRegion) — لتنظيم البيع حسب المنطقة. |

# الجزء الثالث — الكميات حسب المنطقة الجغرافية

## 3.1 تجميع حسب المنطقة

| البند | الوصف |
|-------|--------|
| **المنطقة (storeRegion)** | منطقة المتجر أو العرض (مثال: الخبر، الرياض، جدة، المنطقة الشرقية، إلخ). |
| **الكميات حسب المنطقة** | لكل منطقة: عدد المتاجر، عدد العروض، مجموع الكميات (أو توزيع حسب النوع)، مجموع الأوزان، وصف القيمة (سعر، عدد، وزن، وحدات). |
| **حسب النوع داخل المنطقة** | داخل كل منطقة: نفس البنية «نوع معين (نفس الخواص)» — مجموع الكميات والوزن والقيمة وعدد العروض والمتاجر لهذا النوع في هذه المنطقة. |

## 3.2 تنظيم عمليات البيع حسب المنطقة آلياً

| البند | التطبيق |
|-------|---------|
| **عرض حسب المنطقة** | واجهات (API، داشبورد، سوق) تسمح بفلترة العروض والمتاجر حسب المنطقة — المشتري يختار منطقه أو يرى العروض الأقرب. |
| **تنظيم آلي** | النظام يجمع الكميات والعروض حسب المنطقة ويوفّرها لـ API وواجهات البيع — تنظيم عمليات البيع حسب المنطقة آلياً. |
| **تنظيم عمليات البيع** | عمليات البيع (عرض، طلب، تواصل) تُنظّم حسب المنطقة: عروض حسب المنطقة، كميات حسب المنطقة، قيمة حسب المنطقة — كل ما ينظم عمليات البيع. |

# الجزء الرابع — وصف القيمة (كامل)

## 4.1 عناصر وصف القيمة

| العنصر | الوصف |
|--------|--------|
| **السعر** | سعر الوحدة (ريال، دولار، درهم) — لكل عرض أو متوسط/نطاق للنوع أو المنطقة. |
| **العدد** | عدد العروض أو عدد الوحدات (قطع، صناديق، إلخ). |
| **الوزن** | الوزن (طن، كيلو) — لكل عرض أو مجموع للنوع أو المنطقة. |
| **عدد الوحدات** | كمية معبّرة بعدد وحدات (طن، كيلو، قطعة، متر) — لكل عرض أو مجموع. |
| **القيمة النقدية** | سعر × كمية (تقدير) — لكل عرض أو مجموع للنوع أو المنطقة. |
| **كل وصف ذي علاقة** | اسم البضاعة، نوع السكراب، مرحلة السلسلة، نوع العرض، منطقة، متجر — لوصف كامل للقيمة. |

## 4.2 استخدام وصف القيمة

- **في كل متجر:** وصف قيمة عروض المتجر (سعر، عدد، وزن، وحدات، قيمة تقديرية).
- **لكل نوع (نفس الخواص):** وصف قيمة مجمّع للنوع (مجموع كميات، أوزان، قيمة، عدد عروض ومتاجر).
- **حسب المنطقة:** وصف قيمة مجمّع للمنطقة (مجموع كميات، أوزان، قيمة، عدد عروض ومتاجر — واختياراً حسب النوع داخل المنطقة).

# الجزء الخامس — التفعيل في البوابة (API وداشبورد)

## 5.1 واجهات API مقترحة

| المسار | الوصف |
|--------|--------|
| **GET /api/quantities-by-store** | الكميات في كل متجر: لكل تاجر، عدد العروض، كميات حسب النوع، مجموع (كمية، وزن، قيمة)، وصف القيمة. |
| **GET /api/quantities-by-type** | المجموع لكل نوع معين (نفس الخواص): category + stage + scrapType — عدد عروض، عدد متاجر، مجموع كمية، وزن، قيمة، وصف القيمة؛ واختياراً حسب المنطقة. |
| **GET /api/quantities-by-region** | الكميات حسب المنطقة الجغرافية: لكل منطقة، عدد متاجر، عدد عروض، مجموع كميات/أوزان/قيمة؛ وتفصيل حسب النوع داخل المنطقة. |
| **GET /api/resource-summary** | ملخص موحّد: كميات حسب المتجر، حسب النوع، حسب المنطقة — مع وصف القيمة (سعر، عدد، وزن، وحدات) — لتنظيم عمليات البيع وداشبورد إدارة الموارد. |

## 5.2 الداشبورد وإدارة الموارد

- **داشبورد الأدمن / إدارة الموارد:** عرض كميات حسب المتجر، حسب النوع، حسب المنطقة — مع وصف القيمة — لتنظيم عمليات البيع واتخاذ القرار.
- **داشبورد التاجر:** عرض كميات متجري فقط (عروضي، حسب النوع، وصف القيمة).
- **تنظيم عمليات البيع آلياً:** واجهات البيع (سوق، عروض) تفلتر وتعرِض العروض حسب المنطقة والنوع والقيمة — تنظيم آلي لعمليات البيع.
