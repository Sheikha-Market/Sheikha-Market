# نظام المنطق ومنظومة المنطق ومحرك المنطق

## 1) الهدف

جعل الموقع **منطقياً ومفهوماً وواضحاً**، مع تفعيل **نظام للمنطق** و**منظومة المنطق** تعملان بالخلفية، و**محرك المنطق** يقوم بتحليل المنظومة كاملة.

---

## 2) التعريفات

| العنصر | الوصف |
|--------|--------|
| **نظام المنطق** | نظام يعمل بالخلفية لضمان اتساق المحتوى والتنقل والبنية — الموقع منطقي ومفهوم وواضح. |
| **منظومة المنطق** | مجموعة المكونات (البنية، المحتوى، الأنظمة، واجهة المستخدم) التي تخضع للمنطق والوضوح. |
| **محرك المنطق** | مكوّن يحلل المنظومة كاملة (الصفحات، الروابط، الأنظمة، البيانات) لضمان الاتساق والوضوح والمنطقية. |

---

## 3) التفعيل في المنصة

| العنصر | الحالة |
|--------|--------|
| **نظام المنطق** | مُفعّل — يعمل بالخلفية. |
| **منظومة المنطق** | تعمل بالخلفية — البنية والمحتوى والأنظمة منظّمة منطقياً. |
| **محرك المنطق** | مُفعّل — يحلل المنظومة كاملة بالخلفية. |

يتم عرض الحالة عبر واجهة **الأنظمة النشطة** (زر ⚙️ في الصفحة الرئيسية) وفي صفحة **ريادة الأعمال** (لوحة حالة الأنظمة)، وكذلك في التذييل: «منطقي · مفهوم · واضح — نظام المنطق ومنظومة المنطق ومحرك المنطق يعملان بالخلفية».

---

## 4) ما يحققه المنطق في الموقع

- **منطقي:** ترتيب المحتوى والروابط والخدمات وفق منطق واضح (رئيسية → خدمات → سوق → تواصل، وريادة الأعمال منفصلة ومرتبطة).
- **مفهوم:** لغة واضحة، عناوين محددة، عدم تعقيد غير ضروري.
- **واضح:** تنقل واضح، أزرار دعوة واضحة (ابدأ الآن، أضف عرض، تواصل)، واتساب ورقم تواصل ظاهر.

---

## 5) واجهة API

الخادم يعيد في `GET /api/systems-status`:

- `mantiq`: نظام المنطق — `running: 'بالخلفية'`
- `mantiqEngine`: محرك المنطق — `running: 'يحلل المنظومة كاملة'`

الصفحات التي تعرض حالة الأنظمة (مثل ريادة الأعمال) تعرض هذين النظامين مع ريادة الأعمال ومسرعات وحاضنات الأعمال.
