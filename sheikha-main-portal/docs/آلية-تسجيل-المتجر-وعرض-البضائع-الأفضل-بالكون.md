# آلية وشكل تسجيل المتجر وعرض البضائع — الأفضل بالكون

## الهدف

- **آلية وشكل تسجيل المتجر وعرض البضائع** تكون **الأفضل بالكون** — أفضل من أي سوق للمتاجر.
- **طريقة العرض والتسجيل والعرض والشكل العام والتجربة** — وضوح، سرعة، جمال، وسهولة لا تُضاهى في أسواق المتاجر.

---

# الجزء الأول — تسجيل المتجر (الأفضل بالكون)

## 1.1 المبادئ

| المبدأ | التطبيق |
|--------|---------|
| **وضوح المسار** | خطوة واحدة واضحة: بيانات التاجر + التخصص (الفئة/مرحلة السلسلة) — بدون تعقيد. |
| **أقل حقول ضرورية أولاً** | الاسم، الجوال، التخصص (مطلوب) — ثم المنطقة، البريد، اسم المؤسسة (اختياري). |
| **التخصص = هوية المتجر** | اختيار الفئة (مصدر، جامع، ناقل، مصهر، معادن، شاحنات) يحدد مرحلة السلسلة ووصف المتجر — مع توضيح مرئي (مرحلة السلسلة + وصف التخصص). |
| **تجربة فورية** | بعد التسجيل: رسالة نجاح واضحة + زر «عرض بضائع الآن» و«السوق» — الانتقال مباشرة إلى عرض البضائع أو استكشاف السوق. |
| **الشكل العام** | تصميم احترافي: ألوان الهوية (ذهبي، أخضر، تركواز)، بطاقات واضحة، تركيز بصري، متجاوب، حلال (لا صور ذوات أرواح). |

## 1.2 الشكل والعرض

| العنصر | المواصفة |
|--------|----------|
| **الهيدر** | رابط العودة للرئيسية، عنوان الصفحة، لغة (عربي/إنجليزي). |
| **النموذج** | حقول في بطاقة واحدة أو مجمّعة منطقياً (بياناتك → التخصص → تفاصيل إضافية)؛ زر تسجيل بارز. |
| **التخصص** | قائمة منسدلة مع مرحلة السلسلة ووصف التخصص عند الاختيار — حتى يفهم التاجر أين هو في السلسلة. |
| **النجاح** | رسالة نجاح + اسم المتجر/التخصص + أزرار: «عرض بضائع»، «السوق»، «الرئيسية». |

## 1.3 لماذا أفضل من أي سوق للمتاجر

| الميزة | الفرق |
|--------|-------|
| **تخصص واحد واضح** | ليس سوقاً عاماً — معادن وسكراب فقط؛ التخصص (مصدر، جامع، ناقل، مصهر، إلخ) يربط المتجر بمرحلة السلسلة فوراً. |
| **مسار واحد قصير** | تسجيل في صفحة واحدة بدون خطوات متعددة مربكة — ثم عرض بضائع أو دخول السوق. |
| **ربط بالتكامل** | إشارة للشركات (ERP/SAP/Oracle) إلى صفحة التكامل — لا ننسى B2B والتكامل الرقمي. |
| **حلال وشفاف** | محتوى حلال؛ لا صور محرمة؛ شفافية الهدف والخطوات. |

# الجزء الثاني — عرض البضائع (الأفضل بالكون)

## 2.1 المبادئ

| المبدأ | التطبيق |
|--------|---------|
| **ربط بالتاجر فوراً** | اختيار التاجر من القائمة (إن كان مسجّلاً) أو إدخال رقم الجوال — حتى يظهر العرض تحت متجر التاجر الصحيح. |
| **مرحلة السلسلة ونوع العرض** | اختيار مرحلة (مصدر، تجميع، نقل، مصهر، تصنيع، طرف نهائي) ونوع عرض السكراب (تجميع، صهر، نقل، شاحنات) — ليظهر العرض في المكان الصحيح في السوق. |
| **الحقول الأساسية** | اسم البضاعة (مطلوب)، كمية، وحدة، سعر، عملة — ثم وصف، وزن، موقع إن وجد. |
| **تجربة بعد الإضافة** | رسالة نجاح + زر «عرض بضائع أخرى» و«عرض السوق» — لرؤية العرض في السوق أو إضافة عرض جديد. |
| **الشكل العام** | نفس مستوى الجودة: بطاقات، ألوان الهوية، وضوح، متجاوب، حلال. |

## 2.2 الشكل والعرض

| العنصر | المواصفة |
|--------|----------|
| **ربط التاجر** | قائمة منسدلة للتجار المسجّلين أو حقل جوال — مع توضيح «إن لم تختر، أدخل رقم جوالك المسجّل». |
| **البضاعة** | اسم البضاعة، كمية، وحدة، سعر، عملة — في قسم واضح. |
| **المرحلة ونوع العرض** | قائمتان: مرحلة السلسلة (من API)، نوع عرض السكراب (تجميع، صهر، نقل، شاحنات) — لتصنيف العرض في السوق. |
| **النجاح** | «تم تسجيل المنتج وعرض البضاعة بنجاح» + أزرار: «عرض بضائع أخرى»، «عرض السوق»، «الرئيسية». |

## 2.3 لماذا أفضل من أي سوق للمتاجر

| الميزة | الفرق |
|--------|-------|
| **تصنيف حسب السلسلة ونوع السكراب** | كل عرض مرتبط بمرحلة سلسلة التوريد ونوع العرض (تجميع، صهر، نقل، شاحنات) — السوق يعرض حسب المرحلة والنوع؛ أفضل من متاجر عامة بدون هذا التصنيف. |
| **ربط فوري بالمتجر** | التاجر أو الجوال يربط العرض بالمتجر — العروض تظهر في متجر التاجر وفي السوق الموحّد. |
| **مسار واحد قصير** | نموذج واحد واضح؛ بعد الإضافة خيارات واضحة (أخرى، السوق، الرئيسية). |
| **متخصص 100%** | معادن وسكراب فقط — واجهة وجداول مبنية لهذا التخصص. |

# الجزء الثالث — الشكل العام والتجربة (الأفضل بالكون)

## 3.1 الشكل العام

| البند | التطبيق |
|-------|---------|
| **الهوية** | ألوان شيخه (ذهبي، أخضر، تركواز، داكن)؛ خط Tajawal؛ زخارف هندسية حلال. |
| **البطاقات** | خلفية بطاقة واضحة، حدود ناعمة، ظل خفيف؛ تركيز بصري على النموذج. |
| **الأزرار** | زر رئيسي بارز (تسجيل، إضافة عرض)؛ أزرار ثانوية للروابط (عرض بضائع، السوق، الرئيسية). |
| **الاستجابة** | متجاوب 100% — الموبايل والتابلت والسطح. |
| **اللغة** | عربي أولاً؛ إنجليزي للتواصل الدولي (تسجيل المتجر يدعم التبديل). |

## 3.2 التجربة

| البند | التطبيق |
|-------|---------|
| **وضوح الخطوات** | المستخدم يعرف فوراً: تسجيل متجر → عرض بضائع → تواصل وبيع. |
| **ردود فورية** | تحقق من الاسم والبريد في الواجهة؛ رسالة نجاح أو خطأ واضحة بعد الإرسال. |
| **انتقال سلس** | بعد التسجيل أو إضافة عرض — أزرار واضحة للخطوة التالية (عرض بضائع، السوق). |
| **حلال وأمان** | لا صور محرمة؛ تجربة آمنة وواضحة — بلا ضرر ولا ضرار. |
