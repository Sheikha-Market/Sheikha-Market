# خطة بناء أكبر شبكة في الكون للسكراب والمعادن

## الرؤية

**بناء أكبر شبكة في الكون للسكراب والمعادن** — منصة واحدة تربط المصادر والجامعين والناقلين والمصاهر والمصنّعين والتجار في كل سوق، بدءاً بالسكراب (تخصصاً ومنتجاً) ثم التوسع بالمعادن إن شاء الله.

---

## المبدأ: السكراب أولاً — تخصص ومنتج

| الأولوية | المعنى | التطبيق |
|----------|--------|----------|
| **1. تخصص** | أن تكون المنصة **متخصصة في السكراب** قبل أي شيء — هوية، محتوى، عروض، طلبات، تقارير. | كل التسجيل والعروض والطلبات والسوق تُصنَّف وتُعرض حسب تخصص السكراب (مصدر، جامع، ناقل، مصهر، تاجر معادن، شاحنات ومعدات). |
| **2. منتج** | أن يكون **المنتج الأول** واضحاً: **سكراب** (حديد، نحاس، ألومنيوم، رصاص، زنك، إلخ) — أنواع ومواصفات وكميات. | نماذج العروض والطلبات والتقارير مبنية على «منتج سكراب» (نوع، مواصفات، كمية، سعر، موقع). المعادن كمنتج نهائي أو وسيط تأتي لاحقاً. |
| **3. التوسع بالمعادن** | بعد ترسيخ السكراب — إضافة منتجات المعادن (مواد خام، منتجات وسيطة، منتجات نهائية) للعرض والطلب. | مرحلة لاحقة: توسيع فئات المنتجات لتشمل معادن جاهزة للتصنيع أو للبيع، مع الإبقاء على السكراب كقلب المنصة. |

---

## المرحلة الأولى: السكراب فقط (تخصص + منتج)

### الأهداف

1. **السيادة في السكراب** — أن تصبح شيخه المرجع الأول لعرض وطلب السكراب في السوق المستهدف.
2. **شبكة كاملة للسكراب** — مصدر → تجميع → نقل → مصهر → تصنيع → طرف نهائي، كلها على المنصة.
3. **منتج واحد واضح** — السكراب بأنواعه ومواصفاته (تجميع، صهر، نقل، شاحنات) دون خلط مبكر مع منتجات المعادن الجاهزة.

### ما يُبنى أولاً (تقنياً وتشغيلياً)

| البند | الوصف |
|-------|--------|
| **تسجيل التجار** | حسب التخصص: مصدر، جامع سكراب، ناقل، مصهر، تاجر معادن، شاحنات ومعدات. |
| **عروض السكراب** | نوع (تجميع / صهر / نقل / شاحنات)، صنف معدني (حديد، نحاس، ألومنيوم، …)، كمية، مواصفات، سعر، موقع. |
| **طلبات الشراء** | مصاهر ومصنّعون وتجار يطلبون سكراب حسب النوع والصنف والكمية. |
| **السوق حسب المرحلة** | عرض العروض والطلبات حسب مرحلة سلسلة التوريد (مصدر، تجميع، نقل، مصهر، تصنيع، طرف نهائي). |
| **السوق حسب نوع السكراب** | فلترة حسب: سكراب تجميع، سكراب صهر، سكراب نقل، شاحنات. |
| **لوحة التحكم** | عروضي، طلباتي، حاويات (للجامع/الناقل)، تقارير (عروض، طلبات، مبيعات، مشتريات). |
| **التكامل (API / ERP)** | عروض، طلبات، تقارير — للربط مع أنظمة العملاء (SAP, Oracle, ERP). |

### الجغرافيا: السعودية أولاً ثم العالم

| الترتيب | السوق | الإجراء |
|---------|--------|----------|
| 1 | السعودية | تفعيل كامل: لغة عربية، ريال سعودي، مصاهر ومشترون سعوديون، تكامل مع رؤية 2030 والاقتصاد الدائري. |
| 2 | الخليج | توسع العروض والطلبات (الإمارات، البحرين، الكويت، عمان، قطر). |
| 3 | الشرق الأوسط وشمال أفريقيا | حسب الطلب والشراكات. |
| 4 | العالم | أوروبا، آسيا، أمريكا — شريحة Enterprise والمصاهر العالمية. |

## المرحلة الثانية: التوسع بالمعادن (إن شاء الله)

بعد استقرار شبكة السكراب ووضوح المنتج الأول:

| البند | الوصف |
|-------|--------|
| **فئات منتجات إضافية** | مواد خام من المصاهر، منتجات وسيطة، منتجات نهائية (معادن جاهزة للتصنيع أو البيع). |
| **نفس السلسلة** | مصدر → تجميع → نقل → مصهر → تصنيع → طرف نهائي — مع إضافة «منتج معادن» بجانب «منتج سكراب». |
| **قائمة المصاهر والمشترين** | الاستفادة من هيكل قائمة المصاهر والمشترين والكميات والحجم العالمي (الموجود في الوثائق) — تحديث وإثراء مع نمو المنصة. |
| **تقارير عالمية** | عرض وطلب سكراب ومعادن حسب الدولة/المنطقة، مع إحصائيات مجمّعة (بدون أسماء أطراف إلا بموافقة). |

## خطوات التنفيذ التفصيلية

### 1. خطوات تقنية (المنصة الحالية)

| الخطوة | الوصف | الحالة / الرابط في المشروع |
|--------|--------|-----------------------------|
| **1.1** | تفعيل تسجيل التجار حسب التخصص (مصدر، جامع، ناقل، مصهر، تاجر معادن، شاحنات ومعدات) | صفحة تسجيل-تاجر.html، API: POST /api/traders/register، GET /api/trader-specialties |
| **1.2** | نموذج عرض السكراب: نوع (تجميع/صهر/نقل/شاحنات)، صنف معدني (حديد، نحاس، ألومنيوم، …)، كمية، مواصفات، سعر، موقع | عرض-بضائع.html، API: GET/POST /api/listings، GET /api/scrap-display-types |
| **1.3** | سجل الطلبات: مصاهر ومصنّعون يطلبون سكراب (نوع، صنف، كمية) | تفعيل أو إنشاء API للطلبات (orders) وربطها بلوحة التحكم |
| **1.4** | السوق حسب المرحلة ونوع السكراب | GET /api/market-by-stage?stage=...، GET /api/market-by-scrap-type?scrapType=... |
| **1.5** | لوحة التحكم: عروضي، طلباتي، حاويات (للجامع/الناقل)، تقارير | لوحة-التحكم.html، تصدير CSV/JSON للعروض والطلبات |
| **1.6** | التكامل: Sheikha Partner API — عروض، طلبات، تقارير للربط مع ERP | وثيقة تخطيط-تسجيل-المستخدمين-لكل-مرحلة، APIs الموجودة في server.js |
| **1.7** | اللغة والعملة: عربي + ريال سعودي كافتراضي للسوق السعودي | i18n.js، إعدادات المنطقة في التسجيل والعروض |

### 2. خطوات تشغيلية (إطلاق المنتج الأول)

| الخطوة | الوصف |
|--------|--------|
| **2.1** | الصفحة الرئيسية تعرض بوضوح: «دخول السوق — المنتج الأول (بيع السكراب)» و«تسجيل تاجر» و«عرض بضائع» — دون إرباك المستخدم بأقسام أخرى. |
| **2.2** | تجربة أولى: تسجيل 3–5 تجار (مصدر، جامع، مصهر) وإضافة عروض سكراب حقيقية أو تجريبية. |
| **2.3** | تفعيل سجل الطلبات والصفقات: عند اكتمال صفقة، تسجيلها (عمولة لاحقاً) وتقارير للمستخدم. |
| **2.4** | قائمة المصاهر والمشترين: البدء بملء السجل من هيكل الوثيقة (هيكل-قائمة-المصاهر-والمشترين) — سعودي أولاً، ثم تحديث من تقارير BIR و USGS وغيرها. |

### 3. خطوات تسويقية (السعودية أولاً)

| الخطوة | الوصف |
|--------|--------|
| **3.1** | المحتوى والهوية: «سوق شيخه — أكبر شبكة سكراب في الكون»؛ لغة عربية، ريال سعودي، رؤية 2030 والاقتصاد الدائري. |
| **3.2** | استهداف: مصاهر سعودية، جامعي سكراب، ناقلين، تجار معادن — دعوة للتسجيل وعرض العروض والطلبات. |
| **3.3** | قنوات: محركات بحث، شبكات اجتماعية، محتوى (مقالات، فيديوهات عن السكراب وإعادة التدوير)، شراكات مع اتحادات القطاع. |
| **3.4** | بعد الاستقرار في السعودية: توسع إلى الخليج (الإمارات، البحرين، الكويت، عمان، قطر) ثم الشرق الأوسط والعالم. |

### 4. معالم النجاح (Milestones)

| المعلم | الوصف | مؤشر |
|--------|--------|------|
| **M1** | المنتج الأول يعمل | تسجيل تاجر + عرض سكراب + ظهور في السوق + لوحة تحكم تعمل |
| **M2** | طلبات وصفقات | مصهر/مصنع يضع طلب شراء سكراب؛ اكتمال صفقة واحدة على الأقل عبر المنصة |
| **M3** | شبكة سكراب سعودية | 20+ تاجر مسجّل، 50+ عرض سكراب، 5+ طلبات/صفقات شهرية |
| **M4** | تكامل أول عميل | عميل واحد (مصهر أو تاجر) متصل عبر API أو استيراد/تصدير ملفات |
| **M5** | التوسع بالخليج | تفعيل لغة/عملة إضافية أو منطقة؛ تجار من دولة خليجية ثانية |
| **M6** | التوسع بالمعادن | إضافة فئة «منتج معادن» (خام/وسيط/نهائي) مع الإبقاء على السكراب كأساس |

## ملخص التنفيذ

1. **الآن:** التركيز 100% على **السكراب** — تخصص (هوية ومتاجر وتصنيفات) ومنتج (عروض وطلبات سكراب فقط).
2. **السوق:** السعودية أولاً، ثم الخليج، ثم الشرق الأوسط والعالم.
3. **لاحقاً:** إضافة منتجات **المعادن** (خام، وسيط، نهائي) مع الإبقاء على السكراب كأساس المنصة.
4. **الهدف:** أكبر شبكة في الكون للسكراب والمعادن — شيخه.
