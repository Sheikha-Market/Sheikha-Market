# الجزء الأول — حجم المشروع

## 1.1 تعريف حجم المشروع

| البند | الوصف |
|--------|--------|
| **حجم المشروع** | نطاق العمل المطلوب لبناء وتشغيل المنظومة — مكونات (واجهة، خادم، APIs، بيانات، ذكاء، تكامل)، صفحات، ميزات، مستخدمين مستهدفين، مناطق. |
| **المراجع** | مخطط-الموقع-والنظام-المتكامل.md؛ مخطط-إدارة-المشروع-الكامل-المبتكر.md؛ تعريف-شيخه-الكامل-القوانين-ورحلة-العميل-والداشبوردات.md. |

## 1.2 تقدير الحجم — قبل التشغيل

| المكون | الوصف التقديري | ملاحظات |
|--------|-----------------|----------|
| **الواجهة** | صفحات أساسية (رئيسية، تسجيل، عرض بضائع، لوحة تحكم، خريطة موقع) + صفحات داعمة (هوية، لوائح، تكامل، إلخ) — عشرات الصفحات | مُنجَز جزئياً؛ يُحدَّث عند إضافة صفحات. |
| **الخادم والـ APIs** | Node.js، APIs (تسجيل، عروض، متاجر، كميات، مؤشرات، عمليات، إلخ) — عشرات المسارات | مُنجَز جزئياً؛ يُحدَّث عند إضافة APIs. |
| **البيانات** | ملفات JSON (تجار، عروض، حاويات، عمليات) — قابلة للترقية إلى قاعدة بيانات | حجم يزداد مع المستخدمين والعروض. |
| **الهوية والعلامة** | ألوان، خطوط، شعار، صفحات هوية — انظر: هوية-شيخه-المتكاملة-والعلامة-التجارية.md | مُطبَّق في الواجهة. |
| **الحوكمة والوثائق** | مخططات، وثائق، مراجعات — انظر: نظام-تخطيط-المنظومة-إداريا-وفنيا-وهندسيا-وعلميا-وحسابيا.md | عشرات الوثائق المرجعية. |

## 1.3 تقدير الحجم — خلال مراحل التشغيل

| المرحلة | نطاق الحجم (تقديري) | المخرجات |
|---------|----------------------|-----------|
| **1 — التأسيس والوضوح** | واجهة أساسية، خريطة موقع، أهداف واضحة | صفحات، مخططات، توثيق |
| **2 — التشغيل والمستخدم** | تسجيل، عروض، لوحة تحكم، APIs | تشغيل الخادم؛ مستخدمون ومتاجر |
| **3 — البيانات والأرشفة** | سجلات عمليات، أرشفة، سحابة (عند التفعيل) | بيانات موثوقة؛ نسخ احتياطي |
| **4 — الذكاء والتكامل** | مؤشرات، تكامل ERP/سحابة، LLM (مستقبلاً) | داشبوردات؛ تكامل شركاء |
| **5 — النمو والقياس** | توسع متاجر، تقارير، تحسين تجربة | نمو مستخدمين؛ استدامة |

## 1.4 تقدير الحجم — قمة المنظومة

| البند | الوصف التقديري |
|--------|-----------------|
| **قمة المنظومة** | المنظومة عند اكتمال المراحل الخمس وتفعيل الذكاء والتكامل والنمو — واجهة كاملة، خادم وAPIs كاملة، بيانات وأرشفة وسحابة، مؤشرات وتقارير، تكامل ERP/SCM، قاعدة معرفة وذكاء، تعاون مع الدولة والشركات الحكومية — انظر: قاعدة-المعرفة-والذكاء-والاشتراكات-وأساسات-البيئة-المتكاملة.md؛ نظام-متكامل-للسعودية-والأهداف-والشركات-والمنصات-والقوانين-وبرامج-الدولة.md. |
| **الحجم النسبي** | أكبر من مرحلة التأسيس بعدة أضعاف — ميزات إضافية، مستخدمون أكثر، مناطق أوسع، تكامل أعمق. |

# الجزء الثاني — حساب التكلفة وتكلفة المنظومة (تقدير رسمي موثوق)

## 2.1 مبدأ التقدير وحساب التكلفة

| البند | الوصف |
|--------|--------|
| **تقدير رسمي موثوق** | أرقام تقديرية قابلة للمراجعة؛ تُحدَّث عند تغيير النطاق أو الأسعار أو المراحل. |
| **وحدات التكلفة** | يمكن التعبير بالريال السعودي (ر.س) أو أي عملة معتمدة؛ التقدير يُحدَّث حسب أسعار السوق والموارد. |
| **التصنيف** | تكلفة قبل التشغيل؛ تكلفة خلال المراحل (تشغيل، صيانة، تطوير، تسويق)؛ تكلفة عند قمة المنظومة (تشغيل مستمر، توسع). |

## 2.2 تقدير التكلفة — قبل التشغيل

| البند | وصف التكلفة (تقديري) | ملاحظات |
|--------|------------------------|----------|
| **التطوير الأولي** | تطوير الواجهة، الخادم، APIs، البيانات الأولية — جهد تطوير (وقت، موارد بشرية أو أدوات). | يعتمد على الفريق والأدوات؛ يمكن تقديره بعدد الساعات أو المبالغ. |
| **الهوية والعلامة** | تصميم الهوية، الشعار، الصفحات المرجعية — جهد تصميم وتوثيق. | مُنجَز جزئياً في الوثائق والواجهة. |
| **البنية التحتية (اختياري)** | استضافة، نطاق، شهادات SSL — تكلفة دورية (شهرية/سنوية). | عند النشر الفعلي (غير localhost). |
| **التوثيق والمخططات** | وثائق، مخططات، مراجعات — جهد توثيق. | عشرات الوثائق مُنجَزة. |

**ملاحظة:** الأرقام الفعلية (بالريال أو العملة) تُدرج عند توفرها — هذا الإطار يُملأ بتقديرات محددة من قيادة المشروع أو المسؤول المالي.

## 2.3 تقدير التكلفة — خلال مراحل التشغيل

| المرحلة | بنود التكلفة (تقديري) | ملاحظات |
|---------|------------------------|----------|
| **المرحلة 1** | توثيق، واجهة، خريطة موقع — جهد تطوير وتوثيق. | مُنجَز معظمه. |
| **المرحلة 2** | تشغيل الخادم، صيانة، دعم مستخدمين — استضافة، وقت إداري/تقني. | جاري. |
| **المرحلة 3** | تخزين، أرشفة، سحابة (عند التفعيل) — تكلفة تخزين ونسخ احتياطي. | يزداد مع حجم البيانات. |
| **المرحلة 4** | ذكاء (APIs خارجية إن وُجدت)، تكامل — تكلفة خدمات أو تراخيص. | يعتمد على مزودي الذكاء والتكامل. |
| **المرحلة 5** | تسويق، توسع، تحسين تجربة — تكلفة تسويق وتطوير إضافي. | يعتمد على استراتيجية النمو. |

## 2.4 تقدير التكلفة — قمة المنظومة

| البند | الوصف التقديري |
|--------|-----------------|
| **التشغيل المستمر** | استضافة، صيانة، تحديثات، دعم — تكلفة دورية شهرية/سنوية. |
| **التوسع** | ميزات جديدة، مناطق جديدة، تكامل إضافي — تكلفة تطوير وتشغيل إضافية. |
| **الذكاء والبيانات** | حفظ سحابي، LLM، مؤشرات — تكلفة خدمات سحابية أو تراخيص. |
| **الامتثال والحوكمة** | مراجعات، امتثال، تقارير — جهد إداري وتوثيق. |

**النتيجة:** التقدير الرسمي الموثوق يُحدَّث دورياً (مثلاً ربع سنوي) ويُربط بخطة الموازنة ونظام الاقتصاد الرقمي — انظر: نظام-الاقتصاد-الرقمي-وتخطيطه-وتنفيذه-وهندسته-وإدارته.md.
