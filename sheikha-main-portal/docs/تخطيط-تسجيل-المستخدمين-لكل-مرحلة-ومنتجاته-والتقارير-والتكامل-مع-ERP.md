# تخطيط تسجيل المستخدمين لكل مرحلة — المنتجات، تجربة المستخدم، تكامل ERP (SAP / Oracle)، التقارير، وحركة المواد والمعلومات

هذه الوثيقة تضع **التخطيط التفصيلي** لتسجيل المستخدمين حسب المرحلة (مصدر، تجميع، نقل، مصهر، تصنيع، طرف نهائي)، مع تحديد **المنتجات والخدمات** التي نقدمها لكل مرحلة، **تجربة المستخدم** بالتفصيل، **سهولة تكامل ERP** (SAP، Oracle)، **نوعية وحجم وأهمية التقارير** لكل عميل، و**حركة المواد والمعلومات والعمليات**، مع **أفضل نموذج عمل مبتكر** لكل مرحلة.

---

## 1. المرحلة الأولى — مصدر (Source)

### 1.1 من هو العميل؟

مصنع، منجم، فرد، أو تاجر يملك أو يورد سكراباً للبيع — المرحلة الأولى في السلسلة.

### 1.2 المنتجات والخدمات التي نقدمها للمصدر

| المنتج/الخدمة | الوصف | الأهمية |
|----------------|--------|----------|
| **تسجيل كمصدر** | إنشاء متجر «متجر مصدر سكراب» مرتبط بمرحلة المصدر | أساسي |
| **عرض كميات للبيع** | إضافة عروض (حديد، نحاس، ألومنيوم، إلخ) مع الكمية، الوحدة، السعر، الموقع | أساسي |
| **ظهور في السوق** | ظهور العروض في سوق شيخه للمشترين (جامعين، مصاهر، تجار) دون إظهار اسم المصدر إن رُغب | أساسي |
| **اقتراح أسعار** | اقتراح سعر مبني على أسعار السوق المرجعية | مساعد |
| **تقارير المبيعات والعروض** | تقرير بعدد العروض، المشاهدات، الطلبات الواردة (إن وُجدت) | مهم |

### 1.3 تجربة المستخدم (رحلة المصدر)

1. **التسجيل:** يختار «مصدر سكراب» في نموذج التسجيل → يُربط تلقائياً بمرحلة «مصدر» في السلسلة.
2. **لوحة التحكم:** يرى ملخص عروضه، إشعارات الطلبات، روابط «عرض بضائع» و«إضافة عرض».
3. **إضافة عرض:** نموذج واحد (اسم البضاعة، نوع السكراب، الكمية، الوحدة، السعر، الموقع) → يُحفظ ويظهر في السوق.
4. **استقبال الطلبات:** إشعار «لديك طلب جديد عبر سوق شيخه» (بدون اسم المشتري إن رُفض الإظهار) → يرد أو يقبل من لوحة التحكم.
5. **التقارير:** تقرير شهري/أسبوعي بعدد العروض، الطلبات، وحجم الكميات المعروضة.

### 1.4 تكامل ERP (SAP / Oracle) — سهولة ويُسر

| الجانب | التطبيق |
|--------|----------|
| **واجهة موحدة** | Sheikha Partner API (REST/JSON) — نفس الواجهة لجميع العملاء؛ SAP و Oracle يتصلان عبر HTTP/HTTPS. |
| **مزامنة العروض** | المصدر يصدّر من ERP قائمة منتجات/كميات/أسعار → نستقبلها عبر `POST /api/listings` أو ملف CSV/Excel مرفوع (لاحقاً). |
| **استيراد الطلبات** | الطلبات الواردة من سوق شيخه تُصدّر إلى ERP عبر `GET /api/orders` أو webhook عند إنشاء طلب. |
| **سهولة التكامل** | لا حاجة لتثبيت برنامج خاص على أرض العميل؛ الاتصال من SAP/Oracle إلى عنوان URL لسوق شيخه + مفتاح API. |
| **الأمان** | OAuth2 أو API Key؛ بيانات عبر HTTPS فقط. |

### 1.5 التقارير — نوعها، حجمها، أهميتها للمصدر

| التقرير | النوع | الحجم التقريبي | الأهمية |
|---------|--------|-----------------|----------|
| ملخص العروض | جدول (عروضي، الحالة، الكمية، السعر) | صفحة واحدة | عالية |
| الطلبات الواردة | قائمة طلبات مع التاريخ والحالة | حسب العدد | عالية |
| تقرير مبيعات شهري | إجمالي الكميات المباعة، عدد الصفقات | 1–2 صفحة | متوسطة |
| تصدير لـ ERP | CSV/JSON للعروض والطلبات | حسب الحجم | عالية عند التكامل |

### 1.6 حركة المواد والمعلومات والعمليات

- **المواد:** خروج سكراب من موقع المصدر → نحو المشتري (جامع/مصهر/تاجر). المنصة لا تتحكم بالشحن الفعلي؛ تُسجّل العروض والطلبات.
- **المعلومات:** دخول (عروض المصدر، مواصفات، أسعار) → خروج (طلبات واردة، تقارير). تدفق من المصدر إلى السوق ومن السوق إلى المصدر.
- **العمليات:** تسجيل → إضافة عرض → استقبال طلب → تأكيد/رفض → (لاحقاً) توثيق تسليم ودفع.

### 1.7 أفضل نموذج عمل مبتكر للمصدر

- **البيع باسم شيخه:** المصدر يظهر كـ «عرض من سوق شيخه» في واجهة المشتري؛ الثقة من العلامة الموحدة.
- **تسعير ذكي:** اقتراح سعر من أسعار السوق يقلل الجهد ويسرّع البيع.
- **تكامل ERP بدون تعقيد:** مصدر يستخدم SAP أو Oracle يربط نظامه مرة واحدة ويستقبل الطلبات ويصدّر العروض تلقائياً.

## 2. المرحلة الثانية — تجميع (Collection / جامع سكراب)

### 2.1 من هو العميل؟

جامع سكراب أو مزود تجميع — يجمع من مصادر متعددة، يفرز، ويجهز للشحن أو البيع.

### 2.2 المنتجات والخدمات التي نقدمها لجامع التجميع

| المنتج/الخدمة | الوصف | الأهمية |
|----------------|--------|----------|
| **تسجيل كجامع** | متجر «متجر تجميع سكراب» مرتبط بمرحلة التجميع | أساسي |
| **عرض كميات مجمعة للبيع** | عروض سكراب مجمّع ومرتب (حديد، نحاس، مختلط، إلخ) مع المواصفات | أساسي |
| **شراء من المصادر** | تصفح عروض المصادر في السوق وطلب كميات للتجميع | أساسي |
| **إدارة حاويات/مواقع** | تسجيل حاويات أو مواقع تجميع (إن وُجدت) | مساعد |
| **تقارير التجميع والمشتريات** | تقرير كميات مجمعة، مشتريات من المصادر، مبيعات للمصاهر/التجار | مهم |

### 2.3 تجربة المستخدم (رحلة الجامع)

1. **التسجيل:** يختار «جامع / مزود تجميع» → مرحلة «تجميع».
2. **لوحة التحكم:** عروضه للبيع + عروض السوق (مصادر) للشراء.
3. **شراء من السوق:** يبحث عن عروض مرحلة «مصدر» أو «تجميع»، يرسل طلب شراء عبر المنصة.
4. **بيع للمصاهر/التجار:** عروضه تظهر للمصاهر والتجار؛ يستقبل طلبات ويُنهي الصفقات.
5. **التقارير:** كميات مشتراة، كميات مباعة، هوامش (إن رُبطت بالأسعار).

### 2.4 تكامل ERP (SAP / Oracle)

| الجانب | التطبيق |
|--------|----------|
| **مزامنة المشتريات** | الطلبات الصادرة من الجامع (شراء من مصادر) تُسجّل في شيخه وتُصدّر إلى ERP كأوامر شراء. |
| **مزامنة المبيعات** | عروض الجامع تُستورد من ERP (مخزون مجمّع) أو تُدخل يدوياً؛ الطلبات الواردة تُصدّر إلى ERP. |
| **التقارير** | تقارير شيخه (مشتريات، مبيعات، حركة كميات) قابلة للتصدير CSV/JSON لربطها بـ SAP/Oracle. |

### 2.5 التقارير — نوعها، حجمها، أهميتها للجامع

| التقرير | النوع | الحجم | الأهمية |
|---------|--------|-------|----------|
| حركة المشتريات | قائمة طلبات شراء من مصادر | حسب العدد | عالية |
| حركة المبيعات | قائمة طلبات واردة من مصاهر/تجار | حسب العدد | عالية |
| ملخص الكميات (مجمّع / مباع) | إجماليات حسب النوع والمرحلة | 1–2 صفحة | عالية |
| تقرير شهري للتكامل مع ERP | ملخص للاستيراد إلى SAP/Oracle | صفحة واحدة | عالية عند التكامل |

### 2.6 حركة المواد والمعلومات والعمليات

- **المواد:** دخول سكراب من مصادر → فرز وتجميع → خروج للمصاهر أو التجار.
- **المعلومات:** عروض المصادر ← الجامع؛ عروض الجامع → المصاهر/التجار؛ طلبات ذهاباً وإياباً.
- **العمليات:** شراء من السوق → استلام → فرز → إضافة عرض بيع → بيع → تسليم.

### 2.7 أفضل نموذج عمل مبتكر للجامع

- **منصة واحدة للشراء والبيع:** الجامع يشتري من مصادر ويبيع لمصاهر من نفس الواجهة؛ تقليل التشتت.
- **تقارير موحدة لـ ERP:** تقرير واحد يلخص المشتريات والمبيعات جاهز للاستيراد في SAP/Oracle.
- **تسمية «عبر شيخه»:** التعامل يظهر باسم سوق شيخه مما يعزز الثقة مع المصادر والمشترين.

## 3. المرحلة الثالثة — نقل (Transport)

### 3.1 من هو العميل؟

ناقل، شركة نقل، أو مالك شاحنات/حاويات يقدم خدمة نقل السكراب أو يؤجر معدات.

### 3.2 المنتجات والخدمات التي نقدمها للناقل

| المنتج/الخدمة | الوصف | الأهمية |
|----------------|--------|----------|
| **تسجيل كناقل أو شاحنات/معدات** | متجر نقل أو متجر شاحنات ومعدات | أساسي |
| **عرض خدمات النقل** | عرض خدمة نقل (من–إلى، سعر، توفر) أو عرض حاويات/شاحنات للإيجار | أساسي |
| **إدارة الحاويات** | تسجيل حاويات (صغيرة، متوسطة، رول أوف) وحالتها (متاحة، قيد الاستخدام) | أساسي |
| **استقبال طلبات النقل** | طلبات من بائعين/مشترين تحتاج نقل → عرض سعر أو قبول | مهم |
| **تقارير الرحلات والحاويات** | تقرير بعدد الرحلات، الحاويات المستخدمة، الإيرادات (إن رُبطت) | مهم |

### 3.3 تجربة المستخدم (رحلة الناقل)

1. **التسجيل:** يختار «ناقل» أو «شاحنات ومعدات» → مرحلة «نقل».
2. **لوحة التحكم:** عروضه (خدمات نقل، حاويات)، طلبات النقل الواردة، إدارة الحاويات.
3. **إضافة عرض نقل أو حاوية:** نموذج خدمة نقل (من، إلى، سعر، وحدة) أو إضافة حاوية (نوع، سعة، حالة).
4. **طلبات النقل:** إشعار «طلب نقل جديد عبر سوق شيخه» → عرض سعر أو قبول.
5. **التقارير:** عدد الرحلات، الحاويات، الإيرادات (إن وُجدت).

### 3.4 تكامل ERP (SAP / Oracle)

| الجانب | التطبيق |
|--------|----------|
| **مزامنة الطلبات** | طلبات النقل تُصدّر إلى ERP كأوامر خدمة أو أوامر شحن. |
| **الحاويات والمعدات** | سجل الحاويات في شيخه قابل للتصدير (CSV/JSON) لمطابقة مع مخزون المعدات في SAP/Oracle. |
| **التكامل اللوجستي** | إذا كان العميل يستخدم SAP TM أو Oracle Logistics، يمكن ربط الطلبات والمواقع عبر API. |

### 3.5 التقارير — نوعها، حجمها، أهميتها للناقل

| التقرير | النوع | الحجم | الأهمية |
|---------|--------|-------|----------|
| طلبات النقل | قائمة طلبات مع الحالة والتاريخ | حسب العدد | عالية |
| حركة الحاويات | حاويات متاحة / قيد الاستخدام / معبأة | صفحة واحدة | عالية |
| تقرير شهري (رحلات، إيرادات) | ملخص للاستيراد في ERP | 1–2 صفحة | متوسطة–عالية |

### 3.6 حركة المواد والمعلومات والعمليات

- **المواد:** نقل فيزيائي للسكراب من موقع إلى آخر؛ المنصة تسجّل الطلب والاتفاق ولا تتحكم بالشحن.
- **المعلومات:** طلبات نقل، عروض أسعار نقل، حالة الحاويات؛ تدفق بين البائع/المشتري والناقل.
- **العمليات:** استقبال طلب نقل → عرض سعر → اتفاق → تنفيذ → (لاحقاً) إغلاق الطلب.

### 3.7 أفضل نموذج عمل مبتكر للناقل

- **سوق نقل موحد:** كل طلبات النقل تمر عبر شيخه؛ الناقل يحصل على طلبات من كل المراحل (مصدر، تجميع، مصهر).
- **إدارة حاويات في السحابة:** الحاويات مسجّلة في المنصة وقابلة للمزامنة مع ERP العميل.
- **تقارير جاهزة لـ SAP/Oracle:** تقرير شهري بحجم مناسب للاستيراد دون تعقيد.

## 4. المرحلة الرابعة — مصهر (Smelter)

### 4.1 من هو العميل؟

مصهر أو معيد تدوير — يشتري سكراباً، يصهره، ويخرج مادة خام أو منتج وسيط.

### 4.2 المنتجات والخدمات التي نقدمها للمصهر

| المنتج/الخدمة | الوصف | الأهمية |
|----------------|--------|----------|
| **تسجيل كمصهر** | متجر «متجر مصهر» مرتبط بمرحلة الصهر | أساسي |
| **عرض طلبات شراء (طلب سكراب)** | المصهر يعلن عن احتياج كميات ونوع (حديد، نحاس، ألومنيوم، مواصفات) | أساسي |
| **تصفح عروض البائعين** | عروض من مصادر وجامعين حسب النوع والمرحلة | أساسي |
| **اقتراح أسعار للشراء** | اقتراح سعر شراء مبني على السوق | مساعد |
| **تقارير المشتريات والكميات** | كميات مشتراة، حسب النوع، التكلفة، المورد (معرف داخلي أو «عبر شيخه») | عالية |

### 4.3 تجربة المستخدم (رحلة المصهر)

1. **التسجيل:** يختار «مصهر / معيد تدوير» → مرحلة «مصهر».
2. **لوحة التحكم:** طلبات الشراء التي أضافها، العروض المطابقة من السوق، الطلبات المُنفذة.
3. **إضافة طلب شراء:** يحدد النوع (حديد، نحاس، إلخ)، الكمية، المواصفات، السعر الأقصى (اختياري) → يظهر للمصادر والجامعين.
4. **استقبال عروض البائعين:** بائع يرسل عرضاً مطابقاً أو المصهر يختار من قائمة العروض ويرسل طلب شراء.
5. **إتمام الصفقة:** اتفاق عبر المنصة (الكل باسم سوق شيخه) → تقرير مشتريات للتصدير إلى ERP.

### 4.4 تكامل ERP (SAP / Oracle) — سهولة ويُسر للمصهر

| الجانب | التطبيق |
|--------|----------|
| **استيراد احتياجات الشراء** | المصهر يصدّر من SAP/Oracle قائمة احتياجات (نوع، كمية، مواصفات) → نستقبلها عبر API أو ملف → ننشئ طلبات شراء في شيخه. |
| **تصدير أوامر الشراء** | الصفقات المكتملة في شيخه تُصدّر كأوامر شراء إلى ERP للمطابقة مع المخزون والمالية. |
| **التقارير** | تقرير مشتريات شهري (مورد، كمية، قيمة) بحجم مناسب لـ SAP MM / Oracle Purchasing. |
| **سهولة التكامل** | REST API + API Key؛ لا حاجة لبرنامج إضافي على السيرفر الداخلي للمصهر. |

### 4.5 التقارير — نوعها، حجمها، أهميتها للمصهر

| التقرير | النوع | الحجم | الأهمية |
|---------|--------|-------|----------|
| طلبات الشراء المفتوحة | قائمة طلبات مع العروض المطابقة | حسب العدد | عالية |
| المشتريات المُنفذة | صفقات مكتملة (كمية، نوع، قيمة، تاريخ) | حسب العدد | عالية جداً |
| تقرير شهري للمواد الواردة | ملخص للاستيراد في SAP/Oracle (Materials Management) | 1–3 صفحات | عالية جداً |
| تحليل تكلفة الشراء حسب النوع | متوسط سعر الشراء حسب نوع السكراب | صفحة واحدة | متوسطة |

### 4.6 حركة المواد والمعلومات والعمليات

- **المواد:** دخول سكراب إلى المصهر → صهر → خروج مادة خام/منتج وسيط.
- **المعلومات:** طلبات شراء المصهر → السوق؛ عروض البائعين → المصهر؛ اتفاق → تقرير مشتريات → ERP.
- **العمليات:** نشر طلب شراء → استقبال عروض → تفاوض/قبول → اتفاق → استلام (فعلي) → إغلاق في المنصة وتصدير إلى ERP.

### 4.7 أفضل نموذج عمل مبتكر للمصهر

- **شراء موحد عبر شيخه:** كل مشتريات السكراب من منصة واحدة؛ الموردون يظهرون كـ «مورد عبر شيخه» أو معرف داخلي فقط.
- **تكامل ERP بلا تعقيد:** مصهر SAP أو Oracle يربط مرة واحدة؛ استيراد احتياجات، تصدير أوامر شراء وتقارير شهرية.
- **تقارير بحجم وأهمية مناسبة:** تقرير المشتريات مُصمّم ليلائم وحدات الشراء والمخزون في ERP.

## 5. المرحلة الخامسة — تصنيع (Manufacturing)

### 5.1 من هو العميل؟

مصنع أو وحدة تصنيع تستخدم معادن أو منتجات وسيطة من المصاهر.

### 5.2 المنتجات والخدمات التي نقدمها للمصنع

| المنتج/الخدمة | الوصف | الأهمية |
|----------------|--------|----------|
| **تسجيل كتصنيع** | متجر مرتبط بمرحلة التصنيع | أساسي |
| **طلب شراء مواد (معادن، منتجات وسيطة)** | نشر احتياج معادن أو منتجات من المصاهر | أساسي |
| **تصفح عروض المصاهر والتجار** | عروض مواد جاهزة للتصنيع | أساسي |
| **تقارير المشتريات والاستهلاك** | مواد مشتراة، ربط مع أوامر الإنتاج (إن وُجد في ERP) | مهم |

### 5.3 تجربة المستخدم، التكامل، التقارير

- **التجربة:** مشابهة للمصهر مع تركيز على «مواد للتصنيع» بدل «سكراب للصهر».
- **التكامل مع SAP/Oracle:** استيراد احتياجات الإنتاج (BOM أو احتياجات مواد) → طلبات شراء في شيخه؛ تصدير أوامر الشراء المكتملة إلى ERP.
- **التقارير:** تقرير مشتريات مواد تصنيع (نوع، كمية، مورد، قيمة) بحجم يلائم وحدة المشتريات وMRP.

### 5.4 أفضل نموذج عمل مبتكر للتصنيع

- **ربط الطلب بالإنتاج:** طلب مواد من شيخه مرتبط برقم أمر إنتاج أو مشروع في ERP (لاحقاً).
- **تقارير جاهزة لـ SAP PP/Oracle Manufacturing:** تقرير مواد وارد للتصنيع بحجم وأهمية مناسبة.

## 6. المرحلة السادسة — طرف نهائي (End / تاجر معادن)

### 6.1 من هو العميل؟

تاجر معادن، مشتري بالجملة، أو طرف يشتري للمرة الأخيرة (إعادة بيع أو استهلاك).

### 6.2 المنتجات والخدمات التي نقدمها للطرف النهائي

| المنتج/الخدمة | الوصف | الأهمية |
|----------------|--------|----------|
| **تسجيل كتاجر معادن/مشتري نهائي** | متجر «متجر معادن» | أساسي |
| **تصفح وشراء من السوق** | عروض من كل المراحل (مصادر، جامعين، مصاهر، مصنعين) | أساسي |
| **طلب شراء** | إرسال طلب شراء لعرض معين أو لمجموعة عروض | أساسي |
| **تقارير المشتريات** | قائمة مشتريات، قيمة، مورد (معرف أو «عبر شيخه») | مهم |

### 6.3 تجربة المستخدم، التكامل، التقارير

- **التجربة:** تسجيل → تصفح السوق → اختيار عرض → إرسال طلب → متابعة الحالة → استلام.
- **التكامل:** تصدير قائمة المشتريات إلى ERP (SAP/Oracle) للمطابقة مع الحسابات والمخزون.
- **التقارير:** تقرير مشتريات شهري بحجم وأهمية مناسبة للتجارة والمالية.

### 6.4 أفضل نموذج عمل مبتكر للطرف النهائي

- **سوق واحد لجميع المراحل:** يرى عروض المصادر والجامعين والمصاهر ويختار الأفضل سعراً وجودة.
- **الشراء باسم شيخه:** لا يظهر اسم البائع إن رُفض؛ التعامل موحد تحت العلامة.

## 7. ملخص التكامل مع ERP (SAP / Oracle) — يسر ونوعية التقارير

| المرحلة | سهولة التكامل | نوع التقارير الأهم | حجم تقريبي | أهميتها للعميل |
|---------|----------------|---------------------|------------|-----------------|
| مصدر | API واحد، مفتاح API | عروضي، طلبات واردة، مبيعات شهرية | صغير–متوسط | عالية |
| تجميع | نفس API، استيراد/تصدير مشتريات ومبيعات | مشتريات، مبيعات، حركة كميات | متوسط | عالية |
| نقل | نفس API، حاويات وطلبات نقل | طلبات نقل، حركة حاويات، إيرادات | صغير–متوسط | عالية |
| مصهر | استيراد احتياجات، تصدير أوامر شراء | مشتريات منفذة، تقرير شهري للمواد | متوسط–كبير | عالية جداً |
| تصنيع | ربط احتياجات إنتاج، تصدير أوامر شراء | مشتريات مواد تصنيع | متوسط | عالية |
| طرف نهائي | تصدير قائمة مشتريات | تقرير مشتريات | صغير | متوسطة–عالية |

## 8. حركة المواد والمعلومات والعمليات — نظرة عامة

- **المواد:** تدفق فيزيائي من مصدر → تجميع → نقل → مصهر → تصنيع → طرف نهائي. المنصة لا تنقل المواد؛ تسجّل العروض والطلبات والاتفاقات.
- **المعلومات:** تدفق ثنائي الاتجاه — عروض، طلبات، تقارير — بين المستخدم وسوق شيخه، وبين سوق شيخه وـ ERP العميل (عند التكامل).
- **العمليات:** تسجيل حسب المرحلة → إضافة عروض/طلبات → تطابق → اتفاق → (تنفيذ فعلي خارج المنصة) → إغلاق وتقارير وتصدير لـ ERP.
