# مؤشرات التسويق والأداء — داشبورد المؤشرات الكامل

## النطاق

- **مؤشرات التسويق كاملة** — عدد زوار، مشاهدات صفحات، تحويلات، تسجيلات، عروض — لكل الصفحات والمسارات.
- **مؤشرات الأداء** — عمليات، أرشفة، سحابة، استجابة، جودة — في داشبورد واحد.
- **وضع مؤشرات الأداء في الداشبورد** — عرض كل المؤشرات في لوحة تحكم (أدمن وتاجر) — نظرة واحدة شاملة.

---

# الجزء الأول — مؤشرات التسويق (كاملة)

## 1.1 مؤشرات الزوار والمشاهدات

| المؤشر | الوصف | المصدر (اقتراح) |
|--------|--------|------------------|
| **إجمالي الزوار** | عدد الزوار الفريدين (حسب الجلسة أو المعرف) في الفترة (يوم، أسبوع، شهر). | تسجيل دخول الصفحة الرئيسية وكل الصفحات؛ تجميع حسب الفترة. |
| **زوار الصفحة الرئيسية** | عدد زيارات الصفحة الرئيسية (index.html أو /). | عداد أو سجل طلبات GET /. |
| **زوار تسجيل المتجر** | عدد زيارات صفحة تسجيل المتجر. | طلبات GET /تسجيل-تاجر.html. |
| **زوار عرض البضائع** | عدد زيارات صفحة عرض البضائع. | طلبات GET /عرض-بضائع.html. |
| **زوار تسجيل الدخول** | عدد زيارات صفحة تسجيل الدخول. | طلبات GET /تسجيل-الدخول.html. |
| **زوار لوحة التحكم** | عدد زيارات لوحة التحكم. | طلبات GET /لوحة-التحكم.html. |
| **زوار السوق / سلسلة التوريد** | عدد زيارات صفحات السوق وسلسلة التوريد. | طلبات GET ذات الصلة. |
| **زوار التكامل** | عدد زيارات صفحة التكامل. | طلبات GET /تكامل.html. |
| **زوار الهوية والبيئة والجودة** | عدد زيارات صفحة الهوية والبيئة والجودة. | طلبات GET /الهوية-والبيئة-والجودة.html. |
| **زوار كل صفحة** | عدد مشاهدات كل صفحة (صفحة رئيسية، تسجيل، عرض بضائع، إلخ) — تفصيل لكل مسار. | سجل طلبات أو عداد لكل مسار. |

## 1.2 مؤشرات التحويل والنشاط

| المؤشر | الوصف |
|--------|--------|
| **تسجيلات تجار (متاجر)** | عدد تسجيلات التجار الناجحة في الفترة. |
| **عروض مضافة** | عدد العروض (قوائم بضائع) المضافة في الفترة. |
| **تسجيلات مستخدمين** | عدد تسجيلات المستخدمين (دخول بحساب) في الفترة. |
| **دخول مسجّل** | عدد مرات تسجيل الدخول الناجحة في الفترة. |
| **نسبة التحويل (تسجيل بعد الزيارة)** | (تسجيلات تجار / زوار تسجيل المتجر) أو مشابه — نسبة من زار صفحة التسجيل وسجّل. |
| **نسبة التحويل (عرض بعد الزيارة)** | (عروض مضافة / زوار عرض البضائع) أو مشابه. |

## 1.3 مؤشرات السوق والإنتاج

| المؤشر | الوصف |
|--------|--------|
| **عدد المتاجر (تجار)** | إجمالي التجار المسجّلين. |
| **عدد العروض** | إجمالي العروض (قوائم البضائع) النشطة. |
| **عدد المستخدمين** | إجمالي المستخدمين المسجّلين. |
| **عمليات اليوم / الأسبوع / الشهر** | عدد العمليات المُسجّلة (تسجيل، إضافة عرض، تعديل، حذف) في الفترة. |

# الجزء الثاني — مؤشرات الأداء

## 2.1 مؤشرات العمليات والأرشفة

| المؤشر | الوصف |
|--------|--------|
| **إجمالي العمليات المُسجّلة** | عدد العمليات في سجل العمليات (من نظام حفظ العمليات). |
| **عمليات ضرورية** | عدد العمليات المصنّفة كضرورية (تسجيل تجار، مستخدمين، عروض كبيرة). |
| **حالة الطابور السحابي** | طول طابور الرفع السحابي؛ آخر رفع ناجح. |
| **حالة الأرشفة** | آخر تدوير للسجل؛ حجم الأرشيف. |

## 2.2 مؤشرات الاستجابة والجودة

| المؤشر | الوصف |
|--------|--------|
| **زمن استجابة API** | زمن استجابة واجهات API (مثلاً متوسط أو P95). |
| **معدل نجاح الطلبات** | نسبة الطلبات الناجحة (2xx) من إجمالي الطلبات. |
| **مؤشر الجودة** | مؤشر مركّب (إن وُجد) — جودة البيانات، الالتزام بالمعايير. |

## 2.3 مؤشرات الذكاء والسحابة

| المؤشر | الوصف |
|--------|--------|
| **حالة الحفظ السحابي** | مفعّل/غير مفعّل؛ آخر مزامنة. |
| **حالة المساعد (LLM)** | مفعّل/غير مفعّل؛ عدد الطلبات في الفترة (إن وُجد). |

# الجزء الثالث — وضع المؤشرات في الداشبورد

## 3.1 داشبورد الأدمن

| القسم | المؤشرات المعروضة |
|-------|---------------------|
| **نظرة عامة** | إجمالي الزوار، زوار الصفحة الرئيسية، زوار تسجيل المتجر، زوار عرض البضائع؛ عدد المتاجر، عدد العروض، عدد المستخدمين؛ عمليات اليوم/الأسبوع. |
| **مؤشرات التسويق** | جدول أو بطاقات: زوار كل صفحة (رئيسية، تسجيل متجر، عرض بضائع، تسجيل دخول، لوحة تحكم، تكامل، هوية وبيئة، إلخ)؛ تسجيلات تجار، عروض مضافة، تسجيلات مستخدمين؛ نسب تحويل (إن وُجدت). |
| **مؤشرات الأداء** | عمليات مُسجّلة، عمليات ضرورية، طابور سحابي، أرشفة؛ زمن استجابة، معدل نجاح؛ حالة الحفظ السحابي، حالة المساعد. |
| **روابط** | لوحات تحكم أخرى، أنظمة إدارية، تقارير. |

## 3.2 داشبورد التاجر (المستخدم)

| القسم | المؤشرات المعروضة |
|-------|---------------------|
| **متجري** | اسم المتجر، المنطقة، عدد المنتجات. |
| **نشاطي** | عدد عروضي، آخر إضافة، إشعارات. |
| **مؤشرات أداء (للتاجر)** | عدد مشاهدات متجري أو عروضي (إن وُجد تتبع)؛ عدد العروض النشطة — حسب الصلاحيات والبيانات المتاحة. |

## 3.3 مصدر البيانات

| المؤشر | المصدر (تقني) |
|--------|----------------|
| **زوار / مشاهدات** | عداد في الخادم عند كل طلب GET لصفحة (أو middleware يسجّل المسار)؛ تخزين في ملف أو قاعدة بيانات (مثلاً data/analytics/views.json أو صفحة: عدد). |
| **تسجيلات / عروض** | من البيانات الحية: TRADERS.length، LISTINGS.length، USERS.length؛ ومن سجل العمليات: عدد عمليات trader_register، listing_add، user_register في الفترة. |
| **عمليات وأرشفة** | من نظام حفظ العمليات: OPERATIONS_STATS، GET /api/operations/status. |
| **استجابة** | قياس في الخادم (وقت معالجة الطلب) أو من طبقة أمامية (إن وُجد). |

# الجزء الرابع — التفعيل في البوابة

## 4.1 API مقترحة

| المسار | الوصف |
|--------|--------|
| **GET /api/marketing-indicators** | إرجاع مؤشرات التسويق: زوار إجمالي، زوار لكل صفحة، تسجيلات تجار، عروض مضافة، مستخدمين، عمليات الفترة — للداشبورد. |
| **GET /api/performance-indicators** | إرجاع مؤشرات الأداء: عمليات، أرشفة، سحابة، استجابة — للداشبورد. |
| **GET /api/dashboard** | يضم أو يربط بمؤشرات التسويق والأداء (إحصائيات، marketing، performance). |

## 4.2 تسجيل الزوار (اقتراح)

- عند كل طلب GET لصفحة (/, /تسجيل-تاجر.html, /عرض-بضائع.html, إلخ): زيادة عداد اليوم والصفحة في ملف أو كيوي ثم حفظ دوري (مثل data/analytics/page-views.json أو بنية تاريخية).
- عدم تخزين بيانات شخصية دون موافقة؛ العدّ فقط (عدد الطلبات لكل مسار وتاريخ).
