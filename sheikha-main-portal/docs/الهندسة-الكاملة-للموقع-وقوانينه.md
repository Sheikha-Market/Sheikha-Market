# الهندسة الكاملة لموقع شيخه — رسم هندسي، خواص، وقوانين (إدارية وهندسية)

## 1. الرؤية والهدف

| البند | الوصف |
|-------|--------|
| **الاسم** | شيخه — سوق المعادن والسكراب |
| **الشعار** | أفضل تقنية في بيع السكراب بالكون |
| **الهدف** | منصة واحدة واضحة: تربط البائع بالمشتري، تسجيل التجار وعرض البضائع، وتكامل مع الأنظمة والذكاء. |
| **المستخدم المستهدف** | تاجر سكراب، بائع، مشترٍ، مستثمر. |

### مكونات الموقع (حسب الأولوية للوضوح)

| المكون | المسار | الوظيفة |
|--------|--------|----------|
| **الصفحة الرئيسية** | `/` أو `index.html` | رسالة واحدة واضحة + مسار بيع بثلاث خطوات + CTA واحد. |
| **تسجيل تاجر** | `/تسجيل-تاجر.html` | نموذج: اسم، جوال، منطقة، شركة — يرسل إلى `/api/traders/register`. |
| **عرض بضائع** | `/عرض-بضائع.html` | نموذج: تاجر (هاتف/معرف)، اسم بضاعة، كمية، وحدة، سعر — يرسل إلى `/api/listings`. |
| **عروض السوق** | في الرئيسية أو `/api/market-listings` | عرض البضائع المعروضة من المتاجر والتجار. |
| **تواصل** | واتساب ثابت + رقم واحد | 966554942904+ — رابط واحد في الهيدر والفوتر. |

## 3. الخواص — الوظائف والواجهات

### 3.1 واجهات API (نظام المتاجر والتجار)

| الطريقة | المسار | الوظيفة | المدخلات المطلوبة |
|---------|--------|----------|-------------------|
| POST | `/api/traders/register` | تسجيل تاجر جديد | `name`, `phone` (اختياري: email, region, company) |
| GET | `/api/traders` | قائمة التجار | — |
| POST | `/api/listings` | إضافة عرض بضاعة | `name` (اختياري: traderId أو traderPhone، quantity، unit، price، currency) |
| GET | `/api/stores` | قائمة المتاجر مع بضائعها | — |
| GET | `/api/market-listings` | كل العروض (سوق مفتوح) | اختياري: `?type=iron` لفلترة حسب النوع |
| GET | `/api/scrap-categories` | تصنيفات السكراب | — |

### 3.2 تكامل الأنظمة والذكاء

| النظام | الحالة | التكامل مع الموقع |
|--------|--------|-------------------|
| نظام المنطق / محرك المنطق | يعمل في الخلفية | يُستدعى من الخادم أو مهام مجدولة؛ لا يعقّد واجهة المستخدم. |
| نظام ريادة الأعمال | يعمل | صفحة ريادة-الأعمال.html؛ رابط من الرئيسية إن رُغب. |
| نظام المتاجر | يعمل | تسجيل تاجر + عرض بضائع + عرض العروض في السوق. |
| نظام البرمجة / اللغات | يعمل | صفحة نظام-البرمجة.html وتجربة-منظومة-البرمجة.html. |
| إدارة المحتوى | محتوى ديناميكي من الخادم | hero ونصوص من `/api/content` أو ثابتة واضحة. |

**قانون التكامل:** لا تظهر للمستخدم تفاصيل أنظمة داخلية إلا عبر صفحات مخصصة (مثل "ريادة الأعمال"، "نظام البرمجة"). الصفحة الرئيسية تبقى بسيطة: بيع السكراب + مسار واضح + تواصل.

## 4. القوانين الإدارية

| الرقم | القانون | التفاصيل |
|-------|----------|----------|
| أ-1 | **وضوح الرسالة** | الصفحة الرئيسية تعرض رسالة واحدة: "أفضل تقنية في بيع السكراب بالكون" ومساراً واحداً (ثلاث خطوات). |
| أ-2 | **مسار واحد للبيع** | الخطوات الثلاث: (1) سجّل وتعرّف على السوق، (2) اعرض بضاعتك، (3) اتّفق وبيع. لا مسارات متوازية مربكة. |
| أ-3 | **تواصل واحد** | رقم واتساب واحد (966554942904+) في كل من: الهيدر، الهيرو، قسم التواصل، الفوتر. |
| أ-4 | **المحتوى حلال** | لا صور لذوات أرواح؛ زخارف هندسية ونخيل ونجوم وتدرجات فقط (وفق تصميم-وهوية-شيخه). |
| أ-5 | **لغة الموقع** | العربية الفصحى مع إمكانية إنجليزية للشعار أو العلامة عند الحاجة. |
| أ-6 | **التجارة والبيانات** | بيانات التجار والعروض تُخزّن في البيانات التابعة للمنصة؛ لا تُنقل لأطراف ثالثة دون موافقة. |

## 5. القوانين الهندسية

| الرقم | القانون | التفاصيل |
|-------|----------|----------|
| ه-1 | **الهيكل الثابت للصفحة الرئيسية** | ترتيب ثابت: هيدر → هيرو (رسالة واحدة) → مسار 3 خطوات → لماذا شيخه (اختياري مختصر) → تواصل → فوتر. |
| ه-2 | **الهيدر** | يحتوي على: الشعار، روابط (الرئيسية، كيف تبيع، المتاجر، تواصل)، زر واتساب، وزر قائمة للموبايل. |
| ه-3 | **التسلسل الهرمي للنص** | عنوان رئيسي واحد (H1) في الهيرو؛ عناوين الأقسام H2؛ عناوين البطاقات/الخطوات H3. حجم خط واضح وقابل للقراءة (لا أقل من 16px للنص الأساسي). |
| ه-4 | **الألوان والخط** | ألوان الهوية: ذهبي #d4af37، أخضر #006633، خلفية داكنة. خط Tajawal. تباين كافٍ للنص (لا نص رمادي فاتح على خلفية فاتحة). |
| ه-5 | **الاستجابة** | الموقع متجاوب؛ الهيدر يتحول إلى قائمة همبرغر على الشاشات الصغيرة. |
| ه-6 | **الروابط الخارجية** | روابط واتساب تفتح في تاب جديد (`target="_blank"` مع `rel="noopener"`). |
| ه-7 | **الأداء** | تقليل الطلبات؛ خط واحد (Tajawal)؛ صور إن وُجدت مُحسّنة. |
| ه-8 | **الخادم والبيانات** | الخادم يعمل على المنفذ 8080؛ مجلد `data/` يُنشأ تلقائياً؛ الملفات traders.json و listings.json. |

## 6. معايير الوضوح والاستخدام (UX)

| المعيار | التطبيق |
|---------|----------|
| **شاشة واحدة = فكرة واحدة** | الهيرو يعرض فقط "أفضل تقنية في بيع السكراب" وزرّين (ابدأ، واتساب). |
| **الإجراء الأساسي واضح** | الزر الرئيسي: "ابدأ بيع السكراب" أو "تسجيل تاجر" حسب الصفحة. |
| **التسميات بالعربية المباشرة** | "تسجيل تاجر"، "عرض بضائع"، "عروض السوق"، "تواصل" — بدون اختصارات غامضة. |
| **لا ازدحام** | لا أكثر من 3–4 روابط رئيسية في الهيدر؛ لا أكثر من 3 خطوات في المسار. |
| **التغذية الراجعة** | بعد تسجيل تاجر أو إضافة بضاعة تظهر رسالة نجاح واضحة. |
| **الوصول للمتاجر** | من الرئيسية يوجد رابط واضح إلى "تسجيل تاجر" و"عرض بضائع" و/أو "عروض السوق". |

## 7. ملخص التكامل مع الأنظمة

- **نظام المتاجر:** يعمل عبر `/api/traders/*` و `/api/listings` و `/api/stores` و `/api/market-listings`؛ واجهات المستخدم: تسجيل-تاجر.html، عرض-بضائع.html، وقسم العروض في الرئيسية.
- **الذكاء والمنطق:** يعمل في الخلفية (الخادم/مهام) دون تعقيد الواجهة.
- **ريادة الأعمال والبرمجة:** صفحات مستقلة مع روابط من الهيدر أو الفوتر عند الحاجة.

هذه الوثيقة هي المرجع الهندسي والإداري لبناء وتطوير الموقع؛ أي تغيير في الهيكل أو الواجهات يُقيّس بهذه القوانين والخواص.
