# آلية البيع — بالموقع فقط

## الهدف

أن يستخدم المستخدم **أدوات سوق شيخه داخل الموقع فقط** لإتمام البيع: عرض منتجات، عرض شاحنات/حاويات، والبيع لموقع سكراب — كل ذلك من خلال الموقع دون الحاجة لخارج المنصة.

---

## الأدوات المتاحة للمستخدم (بالموقع فقط)

| الأداة | الوصف | أين في الموقع |
|--------|--------|----------------|
| **عرض منتجات / بضائع** | إضافة عروض سكراب (حديد، نحاس، ألومنيوم، إلخ) مع الكمية والسعر والموقع | [عرض بضائع](public/عرض-بضائع.html) — نموذج إضافة عرض ثم ظهوره في السوق |
| **عرض شاحنات / حاويات** | تسجيل شاحنات أو حاويات للنقل أو التأجير — تظهر في السوق للمشترين | لوحة التحكم → حاوياتي، أو عبر API `/api/containers` (بعد تسجيل الدخول) |
| **بيع لموقع سكراب** | طلب بيع كمية سكراب لجهة (مصهر، جامع، إلخ) — يتم عبر الطلبات داخل الموقع | لوحة التحكم → عروضي، طلباتي؛ أو ربط عرض بطلب من السوق |

---

## التفعيل الحالي

1. **عرض بضائع:** صفحة `عرض-بضائع.html` + API `POST /api/listings` — المستخدم يضيف عرضاً (ربط بالهاتف أو التاجر) ويظهر في `/api/market-listings` والسوق.
2. **شاحنات / حاويات:** بعد تسجيل الدخول، إدارة الحاويات عبر `GET/POST /api/containers` وواجهة لوحة التحكم.
3. **التسوق والطلبات:** المستخدم يتصفح العروض من الصفحة الرئيسية أو سوق سلسلة التوريد، ويتواصل (واتساب أو عبر المنصة) — الطلبات الرسمية داخل المنصة تُربط لاحقاً بجدول طلبات وإشعارات.

---

## ما يُفعّل لاحقاً (لإكمال الآلية)

- **طلبات موحدة داخل الموقع:** عند النقر «اشتري» أو «اطلب» على عرض، إنشاء سجل طلب مرتبط بالبائع والمشتري (مع إخفاء الهوية حسب خطة «مبيعات باسم شيخه»).
- **إشعار للبائع:** إشعار داخل الموقع + اختياري بريد/واتساب «لديك طلب جديد عبر سوق شيخه».
- **توثيق الصفقة:** حفظ حالة الصفقة (معلق، مؤكد، منفذ، ملغي) باسم سوق شيخه.

---

## خلاصة

الآلية الحالية تسمح بـ **عرض المنتجات** و**عرض الشاحنات/الحاويات** و**التصفح والاتصال** بالكامل من داخل الموقع. إكمال **سجل الطلبات والصفقات** مع **إظهار العمليات باسم سوق شيخه** يتم وفق خطة «مبيعات باسم شيخه».
