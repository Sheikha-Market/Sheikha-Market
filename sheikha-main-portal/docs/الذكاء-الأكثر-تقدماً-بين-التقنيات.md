# الذكاء الأكثر تقدماً بين التقنيات — خارطة شيخه

## الهدف والرؤية (إن شاء الله)

- **تحقيق الذكاء الأكثر تقدماً والأفضل بالكون** — دمج أفضل التقنيات العالمية (OpenAI، Claude، Gemini، DeepSeek، نماذج محلية) مع سياق كامل: سوق المعادن والسكراب، إدارة البرمجة، التكامل الرقمي، إدارة العمليات، والذكاء الصناعي والتقنية ذات العلاقة.
- **تحقيق اللغة الكبيرة (LLM) الأفضل والأكثر تقدماً بالكون** — حفظ اللغة وحركتها، سياق إدارة البرمجة وبرمجيات ذات العلاقة، والتكامل الرقمي — لبناء وتغذية أفضل LLM وأكثرها تقدماً إن شاء الله.

---

هذه الوثيقة تضع **أحدث تقنيات الذكاء الاصطناعي** المتاحة عالمياً وكيفية دمجها في سوق شيخه ليكون الموقع مدعوماً بأقوى ذكاء ممكن.

**الحفظ السحابي للذكاء واللغة (LLM):** لنظام وشبكة الحفظ السحابي لوقت الإنتاج الذكاء، سوق شيخه المعادن والسكراب كامل، وحفظ اللغة وحركتها + **سياق إدارة البرمجة والتكامل الرقمي والذكاء الصناعي** — انظر: نظام-وشبكة-الحفظ-السحابي-للذكاء-والسوق-وللغة-LLM.md.

## 2. مكونات الذكاء الأكثر تقدماً في شيخه

### 2.1 روبوت محادثة (Chatbot) — نماذج لغة كبيرة (LLM)

- **الوظيفة:** إجابات عن أسئلة شائعة، توجيه للتسجيل والعروض، شرح أسعار السكراب، شروط الحلال والجودة.
- **التقنية:** تكامل مع **OpenAI Chat Completions API** (أو أي API متوافق: Azure OpenAI، Claude، Gemini عبر واجهة موحدة).
- **الوضع في الكود:** واجهة `POST /api/ai/chat` جاهزة؛ تُفعّل عند ضبط `OPENAI_API_KEY` في `.env`.
- **التطوير لاحقاً:** نواة (intent) لأسئلة متكررة، ربط بإشعارات وتوصيات، أو نموذج عربي/سعودي عند توفره.

### 2.2 اقتراح أسعار ذكي (تطوّر من القواعد إلى الذكاء)

- **الحالي:** قواعد + أسعار السوق الحالية في الموقع (`/api/ai/pricing-suggestion`).
- **الأكثر تقدماً:**
  - إدخال **نص حر** (وصف السكراب، المنطقة، الكمية) → النموذج يقدّر سعراً أو نطاقاً مع تعليل قصير.
  - دمج **بيانات خارجية** (بورصات، LME، أسعار محلية) عبر واجهات ثم تمرير ملخص للنموذج.
  - **تنبؤات أسعار** (سلاسل زمنية أو نماذج تعلم آلي) لاحقاً عند توفر بيانات كافية.

### 2.3 توصيات «من قد يشتري منك» (Recommendation)

- **الوظيفة:** اقتراح مشترين أو متاجر مناسبة حسب تخصص التاجر، المنطقة، ونوع السكراب.
- **التقنيات:**
  - **مرحلة ١:** قواعد + تشابه محتوى (content-based) على بيانات المتاجر والعروض الموجودة.
  - **مرحلة ٢:** نماذج توصية (collaborative filtering أو هجينة) عند توفر تفاعلات كافية (عروض، طلبات، نقرات).
  - **مرحلة ٣:** إثراء التوصيات بنص حر (LLM) لشرح «لماذا هذا المشتري مناسب».

### 2.4 لوجستيات ذكية

- **الوظيفة:** اقتراح تكلفة نقل، مسارات، أو شاحنات حسب وزن، مسافة، ونوع سكراب.
- **التقنيات:** قواعد أولاً؛ ثم نماذج انحدار أو تعلم آلي عند توفر بيانات نقل فعلية؛ وإمكانية استخدام LLM لشرح التقدير أو الإجابة بلغة طبيعية.

### 2.5 إشعارات ذكية وأولوية

- **الوظيفة:** تنبيه عند تغيّر سعر، طلب مطابق، أو فرصة بيع — مع أولوية حسب أهمية المستخدم.
- **التقنيات:** قواعد + WebSocket/SSE حالياً؛ لاحقاً نموذج تصنيف أو تعلم لترتيب أولوية الإشعارات وتقليل الإزعاج.

### 2.6 تحليل وتقارير ذكية

- **الوظيفة:** توقعات أسعار، نشاط السوق، تقارير جاهزة بلغة طبيعية.
- **التقنيات:** تجميع البيانات أولاً؛ ثم نماذج سلاسل زمنية أو تعلم آلي؛ وعرض النتائج عبر LLM كنص مقروء (ملخص، تفسير).

## 3. التكامل التقني في الخادم (Node.js)

### 3.1 متطلبات البيئة

- في `.env`:
  - `OPENAI_API_KEY=sk-...` — لتفعيل المحادثة والاقتراحات النصية.
  - اختياري: `AI_LLM_PROVIDER=openai` أو `azure` أو عنوان قاعدة URL لواجهة متوافقة مع OpenAI (مثل بعض واجهات Claude/Gemini).

### 3.2 واجهات API المضافة/المخططة

| المسار | الوظيفة | التقنية |
|--------|---------|----------|
| `POST /api/ai/chat` | محادثة مع مساعد شيخه (سكراب، أسعار، تسجيل) | LLM (OpenAI أو متوافق) |
| `GET /api/ai/capabilities` | إظهار إمكانات الذكاء (بما فيها المحادثة إن مُفعّلة) | موجود — يُحدَّث ليعكس حالة المفتاح |
| `GET /api/ai/pricing-suggestion` | اقتراح سعر من بيانات السوق (قواعد) | موجود |
| `POST /api/ai/pricing-explain` (لاحقاً) | وصف نصي → اقتراح سعر + تعليل | LLM + بيانات السوق |
| `GET /api/ai/recommend-buyers` (لاحقاً) | توصيات مشترين محتملين | قواعد/تشابه ثم نماذج توصية |

### 3.3 الأمان والحدود

- عدم تمرير بيانات مستخدمين حساسة (كلمات مرور، هويات كاملة) إلى نموذج خارجي.
- تحديد حد أقصى لطول المحادثة وطول الإدخال/الإخراج لتجنب استهلاك زائد.
- تسجيل الاستدعاءات (بدون محتوى حساس) للمراجعة والتكلفة.

## 4. الواجهة الأمامية (تجربة المستخدم)

- **زر أو رابط «اسأل الذكاء» أو «مساعد شيخه»** في الهيدر أو الفوتر يفتح نافذة/صفحة محادثة.
- **صفحة محادثة** (`/مساعد-شيخه.html` أو مدمجة في لوحة التحكم): حقل إدخال + سجل رسائل + استدعاء `POST /api/ai/chat`.
- عند عدم تفعيل المفتاح: إظهار رسالة واضحة «المساعد الذكي سيُفعّل قريباً» أو عرض إجابات من قائمة أسئلة شائعة (fallback).

## 6. خلاصة

- **الذكاء الأكثر تقدماً** في شيخه يبدأ بـ **LLM للمحادثة والشرح** (OpenAI أو أي API متوافق) ثم يتوسع إلى **اقتراح أسعار نصي، توصيات، لوجستيات، وإشعارات ذكية**.
- البنية الحالية (قواعد + أسعار السوق + واجهات جاهزة) تبقى أساساً؛ إضافة المفتاح وواجهة المحادثة تجعل الموقع **مدعوماً بأحدث تقنيات الذكاء** القابلة للتطوير لاحقاً إلى نماذج أخرى (Claude، Gemini، DeepSeek، أو محلية سعودية) دون تغيير جذري في المنطق.

هذه الوثيقة هي المرجع لـ **الذكاء الأكثر تقدماً بين التقنيات** في سوق شيخه.
