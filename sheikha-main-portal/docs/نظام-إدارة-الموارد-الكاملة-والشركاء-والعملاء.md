# نظام إدارة الموارد الكاملة — وعلاقة الشركاء والعملاء

## النطاق

- **نظام لإدارة الموارد كاملة** — إدارة كل الموارد المرتبطة بسوق شيخه: متاجر، عروض، كميات، أسعار، مناطق، سلاسل توريد، شركاء، عملاء.
- **علاقة الشركاء والعملاء** — ربط الموارد بالشركاء (تجار، مصاهر، ناقلون، مدمجو ERP) والعملاء (مشترون، مستهلكون نهائيون) — لتنظيم العمليات والبيع والتكامل.

---

# الجزء الأول — إدارة الموارد الكاملة

## 1.1 ما تُدار من موارد

| المورد | الوصف |
|--------|--------|
| **المتاجر (التجار)** | كل تاجر مسجّل: الاسم، المنطقة، التخصص، المتجر، العروض المرتبطة. |
| **العروض (قوائم البضائع)** | كل عرض: اسم البضاعة، كمية، وحدة، وزن، سعر، عملة، مرحلة سلسلة، نوع سكراب، منطقة. |
| **الكميات** | كميات المعادن والسكراب في كل متجر؛ المجموع لكل نوع (نفس الخواص)؛ حسب المنطقة الجغرافية. |
| **القيمة** | وصف القيمة: بالسعر، العدد، الوزن، عدد الوحدات — وكل وصف ذي علاقة (انظر: نظام معرفة الكميات). |
| **المناطق الجغرافية** | تجميع حسب منطقة المتجر/العرض — لتنظيم عمليات البيع حسب المنطقة. |
| **سلاسل التوريد** | مراحل السلسلة (مصدر، تجميع، نقل، مصهر، تصنيع، طرف نهائي) — ربط العروض والمتاجر بالمرحلة. |

## 1.2 علاقة الشركاء والعملاء

| الطرف | العلاقة مع الموارد |
|-------|---------------------|
| **الشركاء — تجار** | كل تاجر = متجر؛ الموارد المرتبطة: عروضه، كمياته، منطقه، تخصصه. |
| **الشركاء — مصاهر / ناقلون** | مصاهر وناقلون كأطراف في السلسلة؛ الموارد: عروض مرتبطة بمرحلة مصهر/نقل، مناطق. |
| **الشركاء — مدمجو ERP** | شركاء مرتبطون عبر Sheikha Partner API؛ الموارد: مزامنة عروض، طلبات، كميات. |
| **العملاء — مشترون** | مشترون يبحثون عن عروض؛ الموارد المعروضة لهم: حسب المنطقة، النوع، السعر، الكمية. |
| **العملاء — مستهلكون نهائيون** | طرف نهائي في السلسلة؛ الموارد: عروض مرحلة «طرف نهائي» حسب المنطقة والنوع. |

## 1.3 تنظيم العمليات

| البند | التطبيق |
|-------|---------|
| **تنظيم عمليات البيع** | عمليات البيع تُنظّم حسب المنطقة الجغرافية (عروض حسب المنطقة)، حسب النوع (معادن/سكراب بنفس الخواص)، وحسب القيمة (سعر، عدد، وزن، وحدات). |
| **تنظيم آلي** | النظام يجمع الكميات والعروض حسب المتجر، حسب النوع، حسب المنطقة — ويوفّر واجهات (API، داشبورد) لاستعراضها وتنظيم البيع آلياً. |
| **علاقة الشركاء والعملاء** | الشركاء (تجار، مصاهر، ناقلون) يغذون الموارد (عروض، كميات)؛ العملاء (مشترون، مستهلكون) يستفيدون منها — والنظام يربط الطرفين بالموارد والمناطق والنوع والقيمة. |

# الجزء الثاني — الربط مع نظام معرفة الكميات والمنطقة

- **معرفة الكميات كاملة** — في كل متجر، والمجموع لكل نوع معين من السكراب (نفس الخواص)، والكميات حسب المنطقة الجغرافية — موثّقة في وثيقة «نظام معرفة الكميات الكاملة حسب المتجر والنوع والمنطقة».
- **وصف القيمة** — بالسعر، العدد، الوزن، عدد الوحدات، وكل وصف ذي علاقة — جزء من نظام الكميات والعروض.
- **تنظيم عمليات البيع حسب المنطقة آلياً** — يعتمد على تجميع الكميات والعروض حسب المنطقة وتوفيرها للشركاء والعملاء عبر API وداشبورد.
