# تفعيل أدوات وأتمتة عملية البيع ونظام المبيعات وآليته وتخطيطه بالمخطط العام

## النطاق والهدف

- **أدوات البيع:** أدوات المستخدم داخل الموقع فقط — عرض منتجات، عرض شاحنات/حاويات، طلبات شراء، إشعارات، واتساب عند التفعيل.
- **أتمتة عملية البيع:** أتمتة خطوات البيع (عرض → طلب → قبول/رفض → تأكيد → إشعار/فاتورة) داخل المنصة دون تدخل يدوي خارجي.
- **نظام المبيعات:** آلية المبيعات، تخطيطها، وتفعيلها — مدمجة في **المخطط العام** للموقع والنظام المتكامل (مخطط-الموقع-والنظام-المتكامل.md).
- **تفعيل المخطط العام:** ربط الفوتر في **كل** الصفحات بخريطة الموقع وأهداف الموقع؛ إضافة قسم قياس الاحترافية في لوحة التحكم؛ ربط هذه الوثيقة بمخطط-عام-تام-وشامل-للتنفيذ.md.

## 1. ربط أدوات البيع بالمخطط العام

### 1.1 المخطط العام — مرجع

| المستند | المحتوى |
|---------|---------|
| **مخطط-الموقع-والنظام-المتكامل.md** | أهداف الموقع، الصفحات الأساسية والداعمة، الطبقات (واجهة، خادم، بيانات، تكامل). |
| **خريطة-الموقع.html** | روابط كل الصفحات — أساسية (رئيسية، تسجيل متجر، عرض بضائع، تسجيل الدخول، لوحة التحكم) وداعمة (آلية البيع، التكامل، الهوية، اللوائح). |
| **نظام-خريطة-الموقع-الأفضل-ونظام-قياس-الاحترافية-وتمام-التجارة.md** | قياس شمولية الموقع، تمام الاحترافية، تمام التجارة والتجارة الرقمية. |

### 1.2 أدوات البيع ضمن المخطط

| الأداة | الصفحة / الواجهة | API / الخلفية | المخطط العام |
|--------|-------------------|----------------|---------------|
| **عرض منتجات / بضائع** | `عرض-بضائع.html`، لوحة التحكم → منتجاتي / إضافة منتج | `POST /api/listings`، `GET/POST /api/products` | صفحة أساسية في المخطط؛ مذكورة في خريطة الموقع. |
| **عرض شاحنات / حاويات** | لوحة التحكم → حاوياتي (عند التفعيل) | `GET/POST /api/containers`، `GET /api/my-containers` | جزء من لوحة التحكم في المخطط. |
| **طلبات شراء** | لوحة التحكم → طلباتي / عروضي؛ الرئيسية → المتاجر والعروض → «اطلب» | `POST /api/orders`، `GET /api/my-orders` (عند التفعيل) | تفعيل لاحق — يكمّل دورة التجارة في المخطط. |
| **آلية البيع** | `آلية-البيع.html` | — | صفحة داعمة في المخطط؛ مرتبطة بالفوتر وخريطة الموقع. |
| **إشعارات واتساب** | لوحة التحكم → إشعارات؛ إعدادات واتساب | تكامل واتساب (عند التفعيل) | جزء من التكامل والإشعارات في المخطط. |

### 1.3 تفعيل الربط (تم)

- **الفوتر:** **كل** صفحات الموقع (26 صفحة HTML) تحتوي على رابط **خريطة الموقع** وأهداف الموقع (بلا ضرر ولا ضرار · حب العلامة وأمان · محافظة على البيئة والصحة والبشر). الصفحات التي كانت تفتقد الفوتر أو رابط الخريطة تم تحديثها.
- **لوحة التحكم:** تحتوي على رابط **خريطة الموقع** في القائمة الجانبية؛ وقسم **قياس الاحترافية** كامل (لوحة HTML مع مؤشرات: شمولية، تمام احترافية، امتياز، تمام التجارة، تمام رقمي؛ وقائمة تحقق التمام).
- **صفحة آلية البيع:** مرتبطة من الرئيسية والفوتر وخريطة الموقع؛ توجّه المستخدم إلى عرض بضائع، تسجيل الدخول، لوحة التحكم، المتاجر والعروض.

## 2. أتمتة عملية البيع

### 2.1 خطوات العملية (الحالية والمستقبلية)

| الخطوة | الحالة | الوصف |
|--------|--------|--------|
| 1. عرض البضائع | مفعّل | التاجر يضيف عروضاً عبر عرض-بضائع أو لوحة التحكم — تُخزَن وتظهر في السوق. |
| 2. تصفح المشتري | مفعّل | المشتري يرى العروض في الرئيسية (المتاجر والعروض) أو عرض البضائع. |
| 3. تواصل / طلب | جزئي | حالياً: تواصل (واتساب أو اتصال). لاحقاً: زر «اطلب» → إنشاء طلب داخل المنصة. |
| 4. قبول/رفض البائع | مخطط | عند تفعيل الطلبات: البائع يرى الطلب في لوحة التحكم ويقبل أو يرفض. |
| 5. تأكيد البيع | مخطط | عند القبول: تأكيد الصفقة، إشعار الطرفين، اختياري: فاتورة باسم سوق شيخه. |
| 6. إشعارات | جزئي | إشعارات داخل لوحة التحكم؛ واتساب عند التفعيل. |

### 2.2 أتمتة مقترحة

- **تلقائي:** عند إضافة عرض → يظهر في السوق فوراً؛ عند إنشاء طلب (لاحقاً) → إشعار فوري للبائع.
- **تلقائي:** عند قبول البائع → إشعار للمشتري وتحديث حالة الطلب.
- **تلقائي:** إصدار رقم عملية/فاتورة باسم «سوق شيخه» دون كشف أسماء الأطراف (انظر: خطة-مبيعات-باسم-شيخه.md، آلية-البيع-بالموقع-وخطة-مبيعات-باسم-شيخه.md).
- **اختياري:** أتمتة واتساب — إرسال رابط العرض أو تأكيد الطلب عبر واتساب عند التفعيل.

## 3. نظام المبيعات — آلية وتخطيط وتفعيل

### 3.1 آلية المبيعات

| البند | الوصف |
|--------|--------|
| **العمليات باسم شيخه** | الفواتير والتأكيدات والإشعارات تظهر باسم «سوق شيخه»؛ لا يُكشف اسم البائع للمشتري ولا اسم المشتري للبائع في الواجهة (أو يُعرض «بائع/مشتري عبر شيخه»). |
| **البيانات الداخلية** | إدارة المنصة لديها معرفة الأطراف للدعم والامتثال وحل النزاعات. |
| **المراجع** | آلية-البيع-بالموقع-وخطة-مبيعات-باسم-شيخه.md؛ خطة-مبيعات-باسم-شيخه.md؛ آلية-البيع-بالموقع-فقط.md. |

### 3.2 تخطيط نظام المبيعات

| المرحلة | الإجراء |
|---------|---------|
| **الآن** | أدوات العرض مفعّلة (عرض بضائع، لوحة تحكم، مؤشرات)؛ صفحة آلية البيع والفوتر وخريطة الموقع مرتبطة بالمخطط العام؛ قياس الاحترافية في لوحة التحكم. |
| **قريب** | إخفاء اسم التاجر في واجهة السوق (استبدال بـ «سوق شيخه» أو «بائع عبر شيخه»)؛ تفعيل طلبات الشراء داخل الموقع (اطلب → قبول/رفض → تأكيد). |
| **لاحقاً** | فواتير وإيصالات آلية باسم سوق شيخه؛ أتمتة واتساب للإشعارات والتأكيدات. |

### 3.3 التفعيل بالمخطط العام

| الإجراء | المخطط العام |
|---------|----------------|
| **ربط الصفحات بالفوتر** | تم: الرئيسية، تسجيل تاجر، عرض بضائع، تسجيل الدخول، آلية البيع، الهوية والبيئة والجودة، اللوائح، التكامل — تحتوي على رابط خريطة الموقع وأهداف الموقع. |
| **لوحة التحكم** | تم: رابط خريطة الموقع؛ قسم قياس الاحترافية (شمولية، تمام احترافية، امتياز، تمام التجارة والرقمية). |
| **خريطة الموقع** | تعكس كل الصفحات الأساسية والداعمة؛ مرتبطة من الفوتر في كل الصفحات المحدّثة. |
| **آلية البيع** | صفحة آلية-البيع.html توضّح الأدوات (عرض بضائع، شاحنات/حاويات، بيع لموقع سكراب) ومرتبطة من المخطط وخريطة الموقع. |
| **نظام المبيعات** | موثّق في آلية-البيع-بالموقع-وخطة-مبيعات-باسم-شيخه.md وخطط التفعيل أعلاه؛ مدمج في تخطيط الموقع والنظام المتكامل. |
