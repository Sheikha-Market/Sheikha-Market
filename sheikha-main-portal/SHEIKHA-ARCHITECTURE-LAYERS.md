## 🔌 الطبقة الثانية: واجهات API

### (API Endpoints - الجسر بين الواجهات والخلفية)

### مجموعات الـ API:

#### 1️⃣ **وكلاء إسلاميون** (`/api/islamic-agents/*`)

```
POST   /api/islamic-agents/activate-all           → تفعيل جميع الوكلاء
POST   /api/islamic-agents/generate-maqsad        → إنشاء وكيل حسب المقصد
POST   /api/islamic-agents/generate-custom        → إنشاء وكيل مخصص
POST   /api/islamic-agents/execute-task           → تنفيذ مهمة إسلامية
GET    /api/islamic-agents/stats                  → إحصائيات النظام
GET    /api/islamic-agents/list                   → قائمة الوكلاء النشطة
GET    /api/islamic-agents/:agentId               → تفاصيل وكيل معين
```

#### 2️⃣ **المصادقة** (`/api/auth/*`)

```
POST   /api/auth/login                            → تسجيل الدخول
POST   /api/auth/register                         → إنشاء حساب
POST   /api/auth/logout                           → تسجيل الخروج
GET    /api/auth/verify-token                     → التحقق من JWT
POST   /api/auth/refresh-token                    → تجديد الرمز
```

#### 3️⃣ **السوق** (`/api/market/*`)

```
POST   /api/market/rfq                            → طلب عرض سعر
POST   /api/market/listing                        → نشر عرض جديد
GET    /api/market/listings                       → قائمة العروض
GET    /api/market/rates                          → أسعار المعادن الحالية
POST   /api/market/order                          → إنشاء أمر شراء
```

#### 4️⃣ **المستخدمون** (`/api/users/*`)

```
GET    /api/users/profile                         → ملف المستخدم الشخصي
PUT    /api/users/profile                         → تحديث البيانات
POST   /api/users/wallet                          → عمليات المحفظة
GET    /api/users/transactions                    → السجل المالي
```

#### 5️⃣ **الشركات** (`/api/companies/*`)

```
POST   /api/companies/register                    → تسجيل شركة جديدة
GET    /api/companies/:id                         → معلومات الشركة
PUT    /api/companies/verify                      → التحقق من الشركة
GET    /api/companies/branches                    → فروع الشركة
```

## 🔄 تدفق البيانات - مثال عملي

### سيناريو: طلب فتوى إسلامية 📜

```
1. المستخدم يفتح الوكلاء الإسلاميين
   ↓ [Frontend: الوكلاء-الإسلاميين.html]

2. ينقر على "طلب فتوى عن معاملة"
   ↓ [JavaScript Event]

3. يرسل طلب API:
   POST /api/islamic-agents/execute-task
   {
     "agentId": "agent-deen-preservation-xxx",
     "task": {
       "type": "fatwa_request",
       "question": "هل هذه المعاملة حلال؟"
     }
   }
   ↓ [Server.js ← استقبال الطلب]

4. يذهب للمحرك:
   sheikha-islamic-agents-engine.js
   ├─ البحث عن الوكيل
   ├─ تطبيق قواعد مقصد حفظ الدين
   └─ تجهيز السياق الإسلامي
   ↓

5. يطلب من محرك الذكاء:
   sheikha-lmm-engine.js
   ├─ دمج آيات قرآنية
   ├─ إضافة أحاديث نبوية
   ├─ توليد الفتوى بناءً على المقاصد
   └─ التحقق من التوافق الشرعي
   ↓

6. يحفظ النتيجة:
   data/islamic-agents/agent-deen-preservation-xxx.json
   ↓ [حفظ في الذاكرة الدائمة]

7. يعيد النتيجة للواجهة:
   {
     "success": true,
     "data": {
       "fatwa": "الحكم الشرعي...",
       "references": ["الآية..", "الحديث.."],
       "timestamp": "2026-03-02T10:05:00Z"
     }
   }
   ↓ [API Response ← العودة إلى المتصفح]

8. تحديث الواجهة:
   [JavaScript] ← عرض الفتوى بشكل جميل
   ↓

✅ انتهى! المستخدم يرى الفتوى والمراجع
```

## 🔐 طبقات الأمان

```
┌─────────────────────────────────────┐
│  🔒 Validation Layer               │  ← التحقق من المدخلات
│     (تصفية الأوامر الضارة)         │
├─────────────────────────────────────┤
│  🔐 Authentication Layer            │  ← التحقق من الهوية
│     (JWT + OAuth)                   │
├─────────────────────────────────────┤
│  🛡️ Authorization Layer             │  ← التحقق من الصلاحيات
│     (Role-based Access Control)     │
├─────────────────────────────────────┤
│  🔒 Encryption Layer                │  ← تشفير البيانات
│     (TLS + Password Hashing)        │
├─────────────────────────────────────┤
│  📊 Logging & Monitoring Layer      │  ← تتبع النشاط
│     (Audit Trails)                  │
└─────────────────────────────────────┘
```

## 📈 أداء وقابلية التوسع

### الحد الأقصى للمستخدمين المتزامنين:

- **الآن:** تصميم لـ 1,000+ مستخدم متزامن
- **محسّنات:** WebSocket + Caching + Load Balancing

### سرعة الاستجابة المتوقعة:

- API عادي: **50-200 ميلي ثانية**
- تحليل ذكاء اصطناعي: **1-5 ثواني** (حسب سرعة الـ GPU)
- عمليات السوق: **100-300 ميلي ثانية**

### تخزين البيانات:

- نمو متوقع: **100 MB - 1 GB سنويًا**
- نسخ احتياطي: يومي
- الحفظ: في qubic cluster أو AWS S3
