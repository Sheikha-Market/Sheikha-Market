## ٢. البنية التقنية

### ٢.١ الخادم والواجهات البرمجية

- **Backend:** Node.js + Express.js
- **ملف الخادم:** `server.js` — آلاف الأسطر، أكثر من ٢٠٠ نقطة نهاية API
- **نظام التحميل:** Lazy Loading — ٢١ محركاً حرجاً (L0) تُحمّل عند البدء، والباقي (١١٣+) عند الطلب لتسريع التشغيل
- **الحماية:** ميثاق شيخة مُفعّل (SHEIKHA_CHARTER_ENFORCED)، وبدء تشغيل L0 فقط (STARTUP_L0_ONLY)

### ٢.٢ الواجهة الأمامية

- **التقنيات:** HTML5، CSS3، JavaScript (Vanilla) — بدون أطر عمل خارجية
- **الاتجاه:** RTL كامل للعربية
- **التصميم:** ثيم داكن + ذهبي (#D4AF37) + نحاسي (#B87333) + خط Tajawal
- **PWA:** Service Worker + `manifest.json` — تطبيق ويب تقدمي مع اختصارات (السوق، الشريعة، المجتمع، إلخ)

### ٢.٣ التخزين والمصادقة

- **التخزين:** ملفات JSON في مجلد `data/` (مستخدمون، تجار، شركات، قوائم، إشعارات، حالات الطيار، مؤشرات، إلخ)
- **المصادقة:** JWT + OAuth (Google، Microsoft، Apple، Nafath)
- **الوقت الفعلي:** WebSocket لأسعار لحظية

### ٢.٤ النشر

- **المنصة:** Vercel
- **منفذ التشغيل المحلي:** 8080

## ٦. الأوامر والتشغيل

```bash
npm start          # تشغيل على المنفذ 8080
npm run dev        # تشغيل مع auto-reload (--watch)
vercel --prod      # نشر إنتاج
```

أوامر تشغيلية إضافية (من package.json):

- `ops:readiness` / `ops:readiness:fix` — فحص جاهزية التكامل
- `ops:auth:readiness` — فحص جاهزية المصادقة
- `ops:bg-guard` / `ops:bg-guard:loop` — حراسة العمليات الخلفية
- دعم PM2 لحراسة الخلفية (start/stop)

## ٧. قواعد الكود وبروتوكول التكامل

- تعليقات بالعربية؛ إزاحة ٤ مسافات؛ camelCase؛ فواصل منقوطة؛ single quotes
- كل API يرجع صيغة موحدة: `{ success, data, message, timestamp }`
- معالجة أخطاء ورسائل بالعربية؛ التحقق من المدخلات
- **دور Claude:** تدقيق شرعي، مخاطر، سياسات، مراجعة — لا يكتب UI ولا يفرض refactor ولا يغير منطقاً تشغيلياً دون إذن المالك؛ التنفيذ الفعلي في Cursor وبإذن المالك فقط

## ٨. الخلاصة

منظومة وسوق شيخة مشروع واحد متكامل: خادم Express ضخم مع أكثر من ٢٠٠ API، أكثر من ١٣٠ محركاً في `lib/`، تخزين JSON، مصادقة JWT وOAuth، واجهة عربية RTL بدون أطر عمل، PWA، وطبقة أمان ٩/٩ مع التزام شرعي صريح. الحالة الحالية تُظهر تشغيل البوابة الرئيسية وسوق شامل، لوحات أدمن/مستخدم/شركة مكتملة، ومهام واضحة قيد التنفيذ ومخطط لها (تسجيل حكومي، Event Bus، CRM، ترحيل قاعدة بيانات، تكامل حكومي).
