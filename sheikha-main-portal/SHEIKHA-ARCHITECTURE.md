# معماريّة نظام "شيخة" الذكيّة

## الهدف
إنشاء نظام **هائل وقوي** (ضخم بدون إسراف) يعتمد على **أقوى نماذج الذكاء الاصطناعي** ويدمجها ضمن "هوية شيخة" موحدة عربية أصيلة.  
يدعم التعامل مع أي LLM في العالم ويقدم **لغة DSL عربية** يمكنها التحكم بالبرمجيات وإصدار أوامر.

---

## المكوّنات الرئيسية

1. **طبقة المزودين (LLM Providers)**
   - واجهة موحدة للكائنات: `lib/sheikha/llmProvider.js`
   - تدعم OpenAI, Anthropic, Google, Ollama، إلخ.
   - قابلة للتوسيع بسهولة مع أي مزود جديد.

2. **محرك الشات (ChatSheikha)**
   - `lib/sheikha/chatSheikha.js`
   - يدير الحوار، يتابع التاريخ، يوجّه الطلبات إلى المزود الأنسب.
   - يمكن التدريب (fine-tuning) على مجموعات بيانات مخصصة.

3. **لغة شيخة DSL**
   - قواعد نحوية عربية تسمح بكتابة الأوامر بلغة بسيطة.
   - سيتم تصميم parser باستخدام Nearley أو ANTLR.
   - المترجم يولّد كود JavaScript/ Python/ … وفقا للمضيف.

4. **واجهة التطبيق**
   - يمكن أن تكون CLI أو واجهة ويب/سطح مكتب (Electron أو React+Node).
   - تُعرّف بـ "بوابة شيخة" وتحمل هوية رقمية (OpenID/OAuth).
   - تكامل مع نظام الهوية الرقمية الإسلامية.

5. **بيانات شرعية**
   - مجموعة نصوص من القرآن، الأحاديث، كتب الفقه واللغة.
   - مصدر لتدريب النماذج على الفهم الإسلامي والمنهجية القرآنية.
   - تُحفظ محليًا أو تستخدم لتوجيه تعليم النموذج.

6. **الإدارة والخصوصية**
   - التحكم في إرسال بيانات المستخدم للنماذج.
   - دعم التعلم المحلي (LLMs على الجهاز) لتقليل الإهدار.
   - فك تشفير البيانات محليًا، ومراقبة الأداء.

7. **التوسعة والتكامل**
   - إضافة وحدات (modules) جديدة عبر نظام Plugin.
   - دعم اللغات الأخرى داخل البيئة نفسها (`#python`, `#bash`).
   - تخزين التكوين في JSON/YAML لاستخدامه عبر الخدمات.

---

## استراتيجية النمو (Scale & Efficiency)

- **ضع الأساس بسيطًا ثم وسّعه**: لا تحاول تحميل جميع المزايا دفعة واحدة.
- **استعن بالتخزين المؤقت (caching)** للنتائج الشائعة لتقليل الطلبات للنماذج.
- **استخدم نماذج خفيفة محليًا** (مثل Ollama أو LLaMA) للمهام الروتينية، وانتقل للـ cloud للمهام الثقيلة فقط.
- **قسّم الخدمة إلى microservices** إذا كانت المقاييس كبيرة: شات، تحليل، توجه، تخزين.
- **لوّن بناء الكود بالتصاميم الموجهة للحجم (scalable patterns)** وعدم الإسراف بالاعتماد على خدمات مدفوعة إلا عند الضرورة.

## خطة التطوير السريعة

1. **تهيئة المكتبة الأساسية**
   - إنشاء `package.json` في مجلد `lib/sheikha` أو جذريًا `sheikha-core`.
   - إضافة وحدات `llmProvider` و `chatSheikha` وتصديرها.

2. **بناء نموذج اختبار**
   - كتابة سكربت `examples/simpleChat.js` يستخدم `ChatSheikha` مع مزود واحد.
   - تأكد من تشغيله مع مفتاح OpenAI/Azure.

3. **تصميم اللغة العربية (DSL)**
   - أعد طرح قواعد بسيطة للغة.
   - أنشئ grammar أولي واختبر مع nearley/ANTLR.
   - جعل المترجم يولّد JavaScript نظيف.

4. **إنشاء واجهة أولية**
   - تطبيق ويب بسيط بـ Express + React.
   - شاشات: تسجيل دخول، نافذة محادثة، محرر لغة شيخة.

5. **ضخّم بحذر**
   - أضف تحسينات تدريجية: caching، دعم أكثر للمزودين، تكوين ديناميكي.
   - انتقل إلى بنية الخدمات الصغيرة والموزعة عندما تتطلب الحاجة.

## هل التغيير "ضخم بدون إسراف"؟

- اجعل كل module يُحمل عند الطلب (dynamic import) لتجنب ثقل بدأ التشغيل.
- استخدم بيئة تحكّم بإصدار الموارد (feature flags) حتى لا تُفعّل كل شيء فورًا.
- اختبر الأداء مع حمولات مختلفة قبل إضافة خدمة جديدة. 
- استثمر في تحسين الكود أكثر من إضافة وظائف غير مجدية.
