# 🛡️ سياسة الأمان — Sheikha Security Policy

## 🚨 الإبلاغ عن ثغرة

**لا تفتح Issue عام للثغرات الأمنية.**

## 🔐 قواعد الأمان السبع

| # | القاعدة | المرجع |
|---|---------|--------|
| ١ | لا حذف بدون نسخة احتياطية | «لا ضرر ولا ضرار» |
| ٢ | لا استهلاك أكثر من 80% موارد | ﴿ لَا يُكَلِّفُ اللَّهُ نَفْسًا إِلَّا وُسْعَهَا ﴾ |
| ٣ | لا تغييرات جذرية بدون مراجعة | ﴿ وَلَا تَقْفُ مَا لَيْسَ لَكَ بِهِ عِلْمٌ ﴾ |
| ٤ | لا كشف لبيانات المستخدمين | حماية الخصوصية |
| ٥ | لا كلمات مرور في الكود | أمان المفاتيح |
| ٦ | لا .env في المستودع | حماية الأسرار |
| ٧ | الإصلاح التلقائي للطوارئ فقط | الحكمة والتثبت |

---

## 🔑 إدارة الأسرار

### ✅ مسموح
- GitHub Secrets للـ CI/CD
- Azure Key Vault للإنتاج
- SSH Keys للمصادقة

### ❌ محظور تماماً
- كلمات مرور في الكود (hardcoded)
- ملفات `.env` في المستودع
- Tokens في ملفات نصية
- Credentials في `.git-credentials`

---

## 🖥️ سياسات VSCode

### إعدادات إلزامية
```json
{
  "security.workspace.trust.enabled": true,
  "git.confirmSync": true,
  "git.allowForcePush": false
}
```

### محظور في VSCode
- تشغيل سكربتات من مصادر غير موثوقة
- تثبيت إضافات غير معتمدة (راجع extensions.json)
- نسخ Tokens في terminal مكشوف
- فتح مستودعات خارجية بدون مراجعة

## 🔄 الفحص الأمني التلقائي

يعمل تلقائياً عند كل push وPR:
- ✅ npm audit — فحص ثغرات المتطلبات
- ✅ gitignore check — تأكيد عدم تتبع ملفات حساسة
- ✅ يومياً الساعة 3 ص توقيت الرياض
