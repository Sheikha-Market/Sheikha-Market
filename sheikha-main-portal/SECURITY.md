## 🚨 الإبلاغ عن ثغرة

**لا تفتح Issue عام للثغرات الأمنية — ابلغ بشكل خاص.**

### قنوات الإبلاغ
| القناة | التفاصيل |
|--------|---------|
| 📧 البريد الإلكتروني | `market@sheikha.top` |
| 🔐 تشفير GPG | مفضل للثغرات الحرجة |
| ⏰ وقت الاستجابة | خلال **24 ساعة** |

### ماذا تضمّن في تقريرك
- وصف الثغرة وتأثيرها المحتمل
- خطوات إعادة الإنتاج (PoC إن أمكن)
- الإصدار / البيئة المتأثرة
- أي اقتراحات للإصلاح

## ⏱️ الجدول الزمني للإفصاح المنسق

| المرحلة | الإطار الزمني | الإجراء |
|---------|--------------|---------|
| **إقرار الاستلام** | ≤ 24 ساعة | تأكيد استلام البلاغ |
| **تصنيف الخطورة** | ≤ 72 ساعة | Critical / High / Medium / Low |
| **إصلاح الثغرة** | ≤ 30 يوماً (Critical: 7 أيام) | تطبيق الإصلاح واختباره |
| **إصدار التحديث** | ≤ 45 يوماً | نشر النسخة المُصلَّحة |
| **الإفصاح العام** | ≤ 90 يوماً | نشر CVE/Advisory إن لزم |

## 🔑 إدارة الأسرار

### ✅ مسموح
- GitHub Secrets للـ CI/CD
- Azure Key Vault / Google Secret Manager للإنتاج
- SSH Keys للمصادقة (مفاتيح ed25519 مفضلة)
- Environment variables في وقت التشغيل فقط

### ❌ محظور تماماً
- كلمات مرور في الكود (hardcoded)
- ملفات `.env` في المستودع
- Tokens في ملفات نصية أو تعليقات كود
- Credentials في `.git-credentials`
- مفاتيح خاصة (`*.pem`, `*.key`, `*.p12`) في git
- Service account JSON files في git

## 🖥️ سياسات VSCode

### إعدادات إلزامية
```json
{
  "security.workspace.trust.enabled": true,
  "git.confirmSync": true,
  "git.allowForcePush": false
}
```

### محظور في VSCode
- تشغيل سكربتات من مصادر غير موثوقة
- تثبيت إضافات غير معتمدة (راجع `.vscode/extensions.json`)
- نسخ Tokens في terminal مكشوف
- فتح مستودعات خارجية بدون مراجعة

## 🔄 الفحص الأمني التلقائي

يعمل تلقائياً عند كل push وPR وجدولياً يومياً:

| # | الفحص | الأداة | المستوى |
|---|-------|--------|---------|
| 1 | ثغرات المتطلبات | `npm audit` | moderate+ |
| 2 | ملفات حساسة في git | فحص مخصص | إلزامي |
| 3 | أسرار مسرَّبة | Gitleaks | كل commit |
| 4 | SAST (تحليل ثابت) | CodeQL | security-and-quality |

النتائج تظهر في: **Security → Code scanning** في GitHub.

---

## 📢 عملية CVE والـ Advisory

1. عند اكتشاف ثغرة حرجة — نُنشئ **GitHub Security Advisory** خاصاً
2. نُحدد CVSS score والتأثير
3. بعد الإصلاح — نطلب CVE من MITRE إن لزم
4. ننشر Advisory عاماً ضمن الجدول الزمني أعلاه
