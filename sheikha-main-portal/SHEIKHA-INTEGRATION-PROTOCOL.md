## A) قاعدة التعارف (Project Recognition Handshake)

عند بدء أي جلسة في أي بيئة:
- هذه الجلسة **امتداد** لمشروع SHEIKHA وليست تأسيساً جديداً
- المالك الوحيد: **سلمان أحمد بن سلمان الراجح**
- التنفيذ يتطلب **إذن صريح** من المالك
- العربية RTL + Vanilla JS فقط
- ممنوع refactor واسع أو تغيير بنيوي بدون موافقة

---

## B) بيئات التطوير

| # | البيئة | الدور |
|---|--------|-------|
| 1 | **Cursor IDE** | البيئة الرئيسية — التنفيذ النهائي واعتماد الكود |
| 2 | **Claude Projects** | بيئة التدقيق والتحقق — سياسات + مراجعة حساسة |
| 3 | **ChatGPT / أي Chat** | بيئة مساعدة تشغيلية — صياغة/شرح/كود مقترح |
| 4 | **أي IDE آخر (VS Code / Windsurf / Copilot)** | أدوات تنفيذ تلتزم بنفس القواعد |

## C) توزيع الأدوار (Role Separation) — لمنع التضارب

### 1. Cursor (المنفذ)
- التنفيذ النهائي واعتماد الكود
- إدارة الملفات والبنية
- تطبيق التعديلات بعد إذن المالك فقط
- الناتج النهائي يُدمج ويُعتمد هنا فقط

### 2. Claude (المدقق)
- تدقيق شرعي + مخاطر + وضوح التزامات
- مراجعة سياسات/عقود/شروط/تمويل
- **لا يكتب UI** ولا يفرض refactor ولا يغير منطق
- لا قرار مالي — لا فتوى مستقلة

### 3. ChatGPT/شات (المساعد)
- مساعدة في كتابة كود Vanilla JS/Node مقترح
- مساعدة في UX والنصوص التشغيلية
- **لا فتوى** ولا قرار مالي ولا تغيير قواعد

---

## D) التوجيه الذكي (Routing Logic)

| المهمة | الوجهة |
|--------|--------|
| UI/UX/HTML/CSS/Vanilla JS/Node drafts | ChatGPT (اقتراح) |
| عقود/تمويل/وساطة/شروط/التزامات/غرر/ربا/مخاطر | Claude (تدقيق) |
| أي تغيير بنيوي أو refactor أو نقل ملفات | اقتراح فقط + إذن المالك |
| التنفيذ الفعلي على المشروع | **Cursor فقط وبإذن المالك** |

---

## E) قاعدة "النتيجة الواحدة" (Single Outcome)

- أي مخرجات تُنتجها أي بيئة = **(اقتراح/مراجعة/وثيقة)** وليس تنفيذ تلقائي
- الناتج النهائي يُدمج ويُعتمد داخل **Cursor فقط**
- هوية واحدة + سياق واحد + ناتج واحد مهما تغيّرت الأداة

---

## F) منع كشف الأسرار (Secrets Policy)

ممنوع إرسال/عرض:
- `JWT_SECRET`
- `OPENAI_API_KEY`
- `OAuth secrets` (Google, Microsoft, Apple, Nafath)

أي تشخيص يتطلب أسرار → بصيغة placeholders فقط: `<JWT_SECRET>`

---

## G) صيغة طلب الإذن (Mandatory Approval Prompt)

أي بيئة قبل أي تنفيذ تقول حرفياً:

> **"اقتراح: (وصف مختصر). هل تأذن يا سلمان بتنفيذه؟ نعم/لا"**

ولا يتم التنفيذ بدون **(نعم)** صريحة.
