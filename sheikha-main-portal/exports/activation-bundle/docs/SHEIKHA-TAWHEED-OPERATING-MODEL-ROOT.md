## 1) الجذر الحاكم (Root)

### 1.1 المقصد الأعلى
**توحيد الله تعالى** (ربوبية/ألوهية/أسماء وصفات) + **العدل** + **الأمانة** + **الإتقان** + **نفع الناس** + **منع الضرر**.

### 1.2 المرجعية
1. **القرآن الكريم**
2. **السنة النبوية الصحيحة**

**قاعدة تشغيل حاكمة:** **لا ضرر ولا ضرار**

### 1.3 أوامر تشغيل ملزمة
1. **النية قبل العملية** (بوابة نية)
2. **الشرع قبل التنفيذ** (بوابة شريعة)
3. **الحقوق قبل المصلحة** (بوابة عدل)
4. **الأمن قبل التوسع** (بوابة أمان)
5. **الإتقان قبل السرعة** (بوابة جودة)
6. **التوثيق قبل الاعتماد** (بوابة بيان)

---

## 2) الهيكل العام (Conceptual Architecture)

### 2.1 طبقات النظام (من الأعلى للأسفل)

1. **Charter Layer (الميثاق):** قواعد التوحيد والمرجعية
2. **Governance Layer (الحوكمة):** سياسات + صلاحيات + تدقيق
3. **Domain Layer (المنظومات):** السوق/المجتمع/التعليم/...
4. **Engine Layer (المحركات):** وحدات تنفيذية قابلة للتحميل عند الطلب
5. **Flow Layer (التدفقات):** مسارات بيانات ومعاملات
6. **Evidence Layer (الأثر/السجل):** توثيق شرعي/حقوقي/أمني
7. **Metrics Layer (القياس):** KPIs + جودة + أداء
8. **Improvement Layer (التحسين):** مراجعة دورية + إجراءات تصحيح

---

## 3) الهيكل التنظيمي (Organizational Model)

**القرار النهائي بشري، والنظام أداة تنظيم/تحليل/توثيق.**

1. المالك/المشرف العام
2. مسؤول المرجعية الشرعية
3. مسؤول الحوكمة والامتثال
4. مسؤول الأمن السيبراني
5. مسؤول المعمارية والمنصة
6. مسؤول التشغيل
7. مسؤول السوق والمعاملات
8. مسؤول المجتمع والتعليم
9. مسؤول البيانات والجودة

---

## 4) الهيكل التخطيطي (Planning Model) — جوامع الكلم السبعة

**كل محرك/وظيفة لا بد أن ينتمي إلى جامع واحد** (وقد يشارك نواتج مع غيره عبر Outputs فقط):

### 1️⃣ جامع التوحيد والنية
### 2️⃣ جامع الشريعة والضبط
### 3️⃣ جامع العدل والحقوق
### 4️⃣ جامع الأمان والأمن
### 5️⃣ جامع السوق والمعاملة
### 6️⃣ جامع الإدارة والإتقان
### 7️⃣ جامع البيان والتوثيق والهوية

**قاعدة منع التضخم:**
> **أي محرك بلا جامع واضح = يُوقف → يُراجع → يُصحّح/يُحذف.**

---

## 5) الهيكل التنفيذي (Execution Model)

### 5.1 طبقات التشغيل (L0/L1/L2)

- **L0 Critical:** يبدأ معها الخادم فقط (Startup)
- **L1 Important:** تحميل عند الطلب (Lazy) + مخاطرة متوسطة/عالية
- **L2 Advanced:** تحميل عند الطلب (Lazy) + مخاطرة منخفضة
- **Blocked:** مفقود/غير مطابق → حذف/منع

### 5.2 مبدأ التشغيل الإجباري
- **Startup = L0 فقط**
- كل ما عدا L0 **لا يُحمّل إلا بطلب وظيفي واضح** (Route/Job)

## 6) الهيكل الرقمي (Digital Model) — Root/TLD/Domain/Subdomain

### 6.1 Root Domain (الجذر)
**Root = التوحيد** (حاكم على كل شيء)

### 6.2 TLD مفاهيمي
- `.iman` - التوحيد/النية
- `.sharia` - الشريعة/الضبط
- `.adl` - العدل/الحقوق
- `.amn` - الأمان/الأمن
- `.souq` - السوق/المعاملة
- `.idarah` - الإدارة/الإتقان
- `.bayan` - البيان/التوثيق

### 6.3 Domain (منظوماتك)

#### A) منظومة السوق والتنظيم السوقي
- `market.souq` (الهيكل العام للسوق)
- `metals.souq` (المعادن/السكراب)
- `logistics.souq` (الإمداد/النقل)
- `contracts.adl` (العقود/الحقوق)
- `pricing.adl` (تسعير منضبط)

#### B) منظومة المجتمع والتنظيم المجتمعي
- `community.iman` (المجتمع)
- `ethics.iman` (الآداب/القيم)
- `engagement.idarah` (المبادرات/التفاعل)

#### C) منظومة التعليم
- `learning.iman` (التعليم الشرعي/المعرفي)
- `training.idarah` (برامج تدريب)
- `standards.bayan` (مناهج/معايير)

### 6.4 Subdomain (وظائف دقيقة)
- `orders.market.souq`
- `offers.metals.souq`
- `shipments.logistics.souq`
- `audit.amn`
- `fatwa.sharia`
- `evidence.bayan`
- `kpi.idarah`

## 7) تدفق المعلومات (Information Flow) — الحركة الصحيحة

### 7.1 تدفق الصفقة (مثال السوق)

**نية → شريعة → حقوق → سوق → تشغيل → توثيق → قياس → تحسين**

1. **بوابة نية** (iman)
2. **فحص شرعي** (sharia)
3. **تثبيت الحقوق والعقد** (adl)
4. **تنفيذ السوق والتسليم** (souq)
5. **ضبط الصلاحيات والأمن** (amn)
6. **سجل وتوثيق** (bayan)
7. **مؤشرات قياس** (idarah)
8. **تحسين مستمر** (idarah)

### 7.2 قاعدة مشاركة المحركات بين أكثر من مكان

المحرك قد يخدم أكثر من Domain بشرط:
- لا "ينتقل" ككيان عشوائي
- إنما يصدر **Outputs معيارية** تستهلكها منظومات أخرى

---

## 8) التغذية الراجعة (Feedback) — لتغذية القياس والتطوير

### 8.1 أنواع المخرجات (Outputs) المعيارية

**كل عملية تُنتج حزمة مخرجات ثابتة:**

#### 1️⃣ **شرعي (Sharia Output):**
- الحكم: (مباح/ممنوع/مشتبه) + سبب موجز + مرجع داخلي
- **ممنوع** استخدامه لتدريب/تغذية "ذكاء"؛ يُستخدم للمنع والضبط فقط

#### 2️⃣ **حقوقي (Legal Output):**
- عقد/التزام/شروط/أطراف/موافقة/وقت

#### 3️⃣ **أمني (Security Output):**
- من نفّذ؟ صلاحية؟ أثر؟ سجل تدقيق؟

#### 4️⃣ **تشغيلي (Ops Output):**
- حالة العملية + الأخطاء + زمن التنفيذ

#### 5️⃣ **جودة (Quality Output):**
- هل العمل متقن؟ (نعم/لا) + معيار الإتقان + سبب

#### 6️⃣ **قياس (Metrics Output):**
- مؤشرات: زمن/تكلفة/دقة/رضا/مخاطر

#### 7️⃣ **معرفة (Knowledge Output):**
- درس مستفاد + توصية تحسين + قاعدة تشغيل

### 8.2 حلقة التحسين
**Metrics + Quality + Knowledge → Improvement Backlog → تنفيذ → قياس جديد**

## 9) التفعيل الرقمي في الكود (مبادئ تنفيذية)

### 9.1 مفاتيح تشغيل (Switches)
```javascript
SHEIKHA_CHARTER_ENFORCED = true
STARTUP_L0_ONLY = true
MCP_DENY_BY_DEFAULT = true
```

### 9.2 قاعدة Lazy الصحيحة
- أي `require('./lib/...')` خارج L0 **يُمنع** عند Startup
- التحميل يكون فقط عبر: `loadEngineOnDemand('engine.js', initFn)` عند الطلب

---

## 10) قائمة التنفيذ (Checklist)

### 10.1 التحقق من التكامل التنظيمي ↔ التشغيلي
- [ ] كل Domain له مسؤول تشغيل واضح (وظيفيًا)
- [ ] كل Engine له جامع + طبقة (L0/L1/L2/Blocked)
- [ ] كل Route/Job مربوط بSubdomain صحيح
- [ ] كل عملية تُنتج Outputs السبعة وتوثقها

### 10.2 تشغيل الإنتاج بدون أخطاء
- [ ] Startup = L0 فقط
- [ ] Unknown/Blocked = لا وجود له في startup
- [ ] Deny-by-default للأدوات
- [ ] Audit logs فعّالة
- [ ] KPIs تُسجل بعد كل عملية
