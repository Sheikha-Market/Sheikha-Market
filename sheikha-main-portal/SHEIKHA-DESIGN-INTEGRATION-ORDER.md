## قواعد الدمج

### القاعدة 1: لا يُدمج مكون قبل اكتمال ما يعتمد عليه
```
مثال: نظام الفواتير يعتمد على نظام الطلبات
     → يجب إكمال الطلبات أولاً ثم الفواتير
```

### القاعدة 2: التنفيذ في Cursor فقط
```
Claude → تدقيق واقتراح
ChatGPT → كود مقترح وUX
Cursor → التنفيذ الفعلي والاعتماد
```

### القاعدة 3: كل مكون يُوثّق في SHEIKHA-WORK-LOG.md
```
بعد كل إنجاز → تحديث سجل العمل → أي بيئة تقرأه تعرف الحالة
```

### القاعدة 4: الشريعة أولاً
```
أي مكون جديد يُراجع شرعياً قبل الاعتماد
خصوصاً: العقود، التمويل، الوساطة، التسعير
```
