## 🚨 الإبلاغ عن ثغرة أمنية — Reporting a Vulnerability

### 📧 تواصل معنا — Contact

**لا تفتح Issue عام أبداً للثغرات الأمنية — Never open a public Issue for security vulnerabilities.**

أرسل تفاصيل الثغرة إلى:

- **البريد الإلكتروني:** `market@sheikha.top`
- **العنوان:** `[SECURITY] وصف موجز للثغرة`
- **GitHub Security Advisories:** [أبلغ عبر GitHub](../../security/advisories/new)

### 📋 ما يجب تضمينه — What to Include

1. **وصف الثغرة** — Description of the vulnerability
2. **خطوات إعادة الإنتاج** — Steps to reproduce
3. **التأثير المحتمل** — Potential impact
4. **الإصدار المتأثر** — Affected version
5. **أي مقترحات للإصلاح** — Suggested fix (optional)

## 🔐 نطاق البحث الأمني — Scope

### ✅ في النطاق — In Scope

- `sheikha-main-portal/` — البوابة الرئيسية
- واجهات API (`/api/**`)
- آليات التوثيق والتفويض
- إدارة الجلسات والرموز (tokens)
- حماية البيانات والمعاملات

### ❌ خارج النطاق — Out of Scope

- هجمات الحرمان من الخدمة (DoS/DDoS)
- الهندسة الاجتماعية على أعضاء الفريق
- الثغرات في خدمات الطرف الثالث (Azure, GitHub)
- الثغرات المعروفة في المكتبات المنشورة في Dependabot PRs

## 🌿 المبادئ الأمنية — Security Principles

```
﴿اللَّهُ لَا إِلَٰهَ إِلَّا هُوَ الْحَيُّ الْقَيُّومُ﴾ — البقرة: ٢٥٥
```

منظومة شيخة تلتزم بـ:

- **لا ضرر ولا ضرار** — حماية المستخدمين والبيانات
- **الأمانة** — الشفافية في كشف الثغرات
- **الإتقان** — معالجة جذرية لكل ثغرة
- **الاستدامة** — أمان طويل الأمد بلا تنازلات
